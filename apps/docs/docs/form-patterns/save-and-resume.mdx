# Save & Resume

Government benefit forms can take 30-60 minutes. Users may need to pause. To find a document, take a break, or step away from a difficult section. CivUI supports both client-side and server-side persistence.

## Client-side persistence

Add `persist` to `civ-form` to auto-save field values to `sessionStorage` as the user types (debounced 500 ms). Values restore automatically on page reload until the tab is closed or the form is submitted.

<StoryEmbed id="forms-form-form--persist-draft" />

```html
<civ-form persist="va-form-21-526ez">
  <!-- Field values auto-save as the user types -->
</civ-form>
```

The persist key (`va-form-21-526ez`) scopes the storage so multiple forms on the same domain don't collide. The key is stored under `civ-form:${persist}` in `sessionStorage`.

**Auto-excluded from persistence:**
- Fields with `data-civ-pii` (e.g. `civ-ssn`, `civ-ein`. Masked inputs flagged as PII)
- Fields with `data-persist-exclude` (consumer-applied opt-out)
- `disabled` fields

Persisted state is cleared automatically when the form submits successfully (via `civ-submit`) or when the user calls `form.reset()`.

## Pause action

For multi-step forms, `show-pause` adds a "Save and come back later" link next to the continue button:

<StoryEmbed id="forms-form-form-step--with-pause-and-resume" />

```html
<civ-form-step show-pause>
  <div data-step-label="Service history">
    <!-- fields -->
  </div>
</civ-form-step>
```

The `sensitive` attribute auto-enables pause. Users shouldn't feel trapped in emotionally difficult sections.

## Server-side persistence

For persistence across days or devices, implement a save endpoint and wire it to the pause action:

```html
<civ-form persist="va-21-526ez">
  <civ-form-step show-pause>
    <!-- steps -->
  </civ-form-step>
</civ-form>
```

```js
const form = document.querySelector('civ-form');

// Save to server when user clicks "Save and come back later"
form.addEventListener('civ-step-pause', async (e) => {
  const formData = form.getFormData();
  await fetch('/api/drafts/va-21-526ez', {
    method: 'PUT',
    body: JSON.stringify(formData),
  });
  window.location.href = '/saved-confirmation';
});
```

## Restoring a draft

On page load, hydrate the form from a saved draft by setting `prefillData`. A `Record<fieldName, { value, source, locked? }>` object:

```js
const draft = await fetch('/api/drafts/va-21-526ez').then(r => r.json());

const form = document.querySelector('civ-form');

// Convert your server response into the PrefillData shape.
form.prefillData = Object.fromEntries(
  Object.entries(draft.fields).map(([name, value]) => [
    name,
    { value, source: 'saved' },
  ]),
);
```

`source: 'saved'` flags the value as a restored draft (vs. `'profile'` or `'api'`), which civ-form uses to render a "Restored from a saved draft" notice. Set `locked: true` on any field that the user shouldn't be able to edit inline (server-of-truth values).

## What gets persisted

| Data | `persist` (sessionStorage) | Server-side |
|------|---------------------------|-------------|
| Text input / select / radio / checkbox values | Yes | Yes |
| File uploads | No (Files can't be JSON-serialized) | Yes (separate upload endpoint) |
| Current step index | No (re-enters at step 1 on reload) | Recommended |
| PII fields (`civ-ssn`, `civ-ein`, anything with `data-civ-pii`) | No (auto-excluded) | Up to you |
| Validation errors | No (re-validated on submit) | No |

## Save and finish later button

For forms without `civ-form-step`, add a secondary save button:

```html
<div class="civ-flex civ-gap-3 civ-mt-6">
  <civ-button type="submit" label="Continue"></civ-button>
  <civ-button emphasis="secondary" label="Save and finish later"></civ-button>
</div>
```

## Components used

- [`civ-form`](/components/form). `persist` attribute for sessionStorage
- [`civ-form-step`](/components/form/form-step). `show-pause` for pause action, `sensitive` auto-enables pause
