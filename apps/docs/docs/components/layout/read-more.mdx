# Read More

<!-- contract-link:end -->

`<civ-read-more>`

Two-stage content disclosure: a teaser paragraph stays visible at all times,
and a "Read more" button reveals the rest of the content. Use it when the user
needs enough context to decide whether to read on. Eligibility fine print,
optional definitions, secondary explanations.

The rest region stays in the DOM but carries the `hidden` attribute when
collapsed, so screen readers skip it entirely until the user expands. The
button label toggles between "Read more" and "Read less" so the accessible
name always describes the action that will result from activating it.

## When to Use Read More vs Disclosure

| Pattern | Use it when |
| --- | --- |
| **Read More** (`civ-read-more`) | You have a useful **teaser** the user should see at a glance, with extra detail behind a toggle. The user makes an informed choice about whether to expand. |
| **Disclosure** (`civ-disclosure`) | The content is **entirely optional**. There's nothing useful to show by default. The button label alone tells the user what's behind it. |

If you find yourself authoring an empty teaser just to use `civ-read-more`,
reach for `civ-disclosure` instead.

## Usage

```html
<civ-read-more>
  <p>You may be eligible for a benefit if you meet certain criteria.</p>
  <div data-rest>
    <p>
      Specifically, you must have served at least 24 months of active duty and
      been discharged under conditions other than dishonorable. Some benefits
      also require a documented service-connected disability.
    </p>
  </div>
</civ-read-more>

<!-- Custom labels and chevron icon -->
<civ-read-more more-label="Show details" less-label="Hide details" icon="chevron-down">
  <p>A dependent is a spouse, child, or parent who relies on you financially.</p>
  <div data-rest>
    <p>More specifically: a spouse you are legally married to; an unmarried
       child under 18 (or under 23 if enrolled full-time in school)…</p>
  </div>
</civ-read-more>
```

The teaser is the default slot. Anything marked with `data-rest` is routed
into the hidden region and revealed on toggle.

### Inline mode

For "...read more" patterns where the trigger should flow as the last words
of a paragraph, set the `inline` attribute. The trigger sheds its button
chrome (no background, no top margin, no chevron) and reads as underlined
inline emphasis. Author the teaser as plain text. Block elements inside an
inline container defeat the layout. And wrap the rest content in
`<span data-rest>` so the HTML stays valid inside the surrounding `<p>`:

```html
<p>
  <civ-read-more inline>
    We use this information to verify your identity.
    <span data-rest>
      Specifically, we hash it with a salt and store only the hash.
    </span>
  </civ-read-more>
</p>
```

## Accessibility

- **Real `<button type="button">` trigger.** The button explicitly carries
  `type="button"` so a read-more inside a `<form>` doesn't accidentally
  submit the form when clicked.
- **`aria-expanded` reflects state.** Flips between `"true"` and `"false"`
  on toggle so screen readers announce the new state on the next
  interaction.
- **`aria-controls` points at the rest region.** Generated `id` ensures
  the association is stable across re-renders.
- **`hidden` on the rest region.** When collapsed, the rest region carries
  the native `hidden` attribute so screen readers skip it entirely.
  better than `aria-hidden`, which keeps the content reachable by
  keyboard.
- **Button label toggles.** The accessible name on the button reads
  "Read more" when collapsed and "Read less" when expanded, so the user
  hears what activating it will do, not what state it's currently in.
- **Locale-aware defaults.** The "Read more" / "Read less" strings come
  from `t('readMoreButton')` and `t('readLessButton')`. Override via the
  locale registry or pass `more-label` / `less-label` directly.
- **`civ-toggle` event is local.** Does NOT bubble or compose, so
  `civ-form`'s form-level event listeners don't see read-more toggles as
  form-field changes.
