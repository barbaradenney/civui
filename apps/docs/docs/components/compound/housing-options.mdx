# Housing Options

`<civ-housing-options>`

Compound component for collecting mailing addresses from people who may not have permanent housing. Provides a checkbox to indicate no permanent address, then offers configurable mail delivery options: USPS General Delivery, an alternate address, or alternative contact methods.

## Mail Options

The `mail-options` prop accepts a JSON array of objects to replace the default options:

```typescript
interface MailOption {
  value: string;           // Unique identifier
  label: string;           // Display label
  description?: string;    // Description text below the label
  renders: 'address' | 'contact' | 'none';  // What to show when selected
  addressHint?: string;    // Hint for the address form
  addressLegend?: string;  // Legend for the address form
}
```

- `renders: 'address'` ‚Äî shows a `civ-address` form
- `renders: 'contact'` ‚Äî shows city/state + contact method fields
- `renders: 'none'` ‚Äî renders nothing (consumer handles via events)

### Default options

| Value | Label | Renders |
|-------|-------|---------|
| `general-delivery` | USPS General Delivery | `address` |
| `alternate-address` | Someone else's address | `address` |
| `no-mail` | I can't receive mail | `contact` |

### Custom options example

```html
<civ-housing-options mail-options='[
  {"value":"po-box","label":"PO Box","renders":"address","addressLegend":"PO Box address"},
  {"value":"shelter","label":"Shelter address","renders":"address"},
  {"value":"no-mail","label":"Cannot receive mail","renders":"contact"}
]'></civ-housing-options>
```

## Language principles

- **Never use "homeless"** in labels ‚Äî use "permanent address" or "living situation"
- **Don't make assumptions** ‚Äî "I don't have a permanent address" is factual, not judgmental
- **Explain why we ask** ‚Äî the section intro provides context
- **Provide alternatives** ‚Äî every path leads to a way to reach the person
