# Fieldset

`<civ-fieldset>`

Native fieldset wrapper with legend, hint, and error support for multi-field grouping — several related controls sharing one section heading (e.g. a manually composed address block of street / city / state, where each child input carries its own `label`).

Uses native `<fieldset>` for automatic `disabled` cascade to all child form elements. Absorbs `civ-error-change` events from children so a child control's internal validation state (e.g. `civ-memorable-date` rejecting an unparseable typed value) surfaces in the wrapper's error slot.

Do **not** wrap a single input or a self-contained group component (`civ-radio-group`, `civ-checkbox-group`, `civ-segmented-control`, `civ-yes-no`, `civ-memorable-date`, `civ-date-range-picker`) in this — those already render their own chrome, so you'd get nested fieldsets with double legends. The `lint:fieldsets` CI check enforces this.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `legend` | `string` | `''` | Legend text for the fieldset |
| `hint` | `string` | `''` | Hint text displayed below the legend |
| `error` | `string` | `''` | Error message for the group (auto-updates when a child dispatches `civ-error-change`) |
| `required` | `boolean` | `false` | Whether the fieldset is required |
| `disabled` | `boolean` | `false` | Disable all child form elements via native cascade |
| `tight-hint` | `boolean` | `false` | Pull the hint flush with the controls below by removing the default 16px gap. Useful for compact compounds where the legend + hint read as one stacked header. |
| `heading-level` | `1`–`6` | — | Promote the legend to a heading via `role="heading"` + `aria-level=N`. See [Heading hierarchy](/components/form/form-group#heading-hierarchy). |
| `size` | `'sm' \| 'md' \| 'lg' \| 'xl'` | `'sm'` | Visual size. `sm` = body size; `md`/`lg`/`xl` step up for section/page heading use. |

## Disabled Cascade

Setting `disabled` on `civ-fieldset` uses the native `<fieldset disabled>` behavior, which automatically disables all form controls within it without needing to set `disabled` on each child individually.

## Usage

```html
<!-- Group related fields -->
<civ-fieldset legend="Emergency contact">
  <civ-text-input label="Contact name" name="emergencyName" required></civ-text-input>
  <civ-text-input label="Relationship" name="emergencyRelation" required></civ-text-input>
  <civ-text-input label="Phone number" name="emergencyPhone" type="tel" mask="phone-us" required></civ-text-input>
</civ-fieldset>

<!-- With hint -->
<civ-fieldset
  legend="Previous employment"
  hint="List your most recent employer first."
>
  <civ-text-input label="Employer name" name="employer"></civ-text-input>
  <civ-text-input label="Job title" name="jobTitle"></civ-text-input>
</civ-fieldset>

<!-- Disabled section -->
<civ-fieldset legend="Spouse information" disabled>
  <civ-text-input label="Spouse first name" name="spouseFirst"></civ-text-input>
  <civ-text-input label="Spouse last name" name="spouseLast"></civ-text-input>
</civ-fieldset>
```
