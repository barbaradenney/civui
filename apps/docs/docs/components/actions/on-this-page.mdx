`<civ-on-this-page>` + `<civ-on-this-page-item>`

In-page navigation ("table of contents") — a vertical list of anchor links pointing at section headings in the current page. `IntersectionObserver` tracks which heading is currently in the viewport and flips the matching item's `active` flag so the highlight follows the reader's scroll position.

## Two usage modes

1. **Auto-detect (default)** — leave the default slot empty and the component scans for `h2[id], h3[id]` (configurable via `selector`) within the `scopeSelector` ancestor (default `main, article, body`). For each heading found, an item is created pointing at `#{heading.id}` with the heading's text as the label.
2. **Manual** — slot explicit `<civ-on-this-page-item>` children. Auto-detection is skipped; you control the list. Useful when headings live outside the scope, when you want a different label, or when you want to skip some headings.

## Usage

```html
<!-- Auto-detect mode: scans h2[id], h3[id] inside main/article/body -->
<civ-on-this-page label="On this page"></civ-on-this-page>

<!-- Manual mode -->
<civ-on-this-page label="Sections">
  <civ-on-this-page-item href="#intro" label="Introduction"></civ-on-this-page-item>
  <civ-on-this-page-item href="#usage" label="Usage"></civ-on-this-page-item>
</civ-on-this-page>
```

## Auto-detect mode caveats

- Auto-detection runs **once**, in `firstUpdated`. Headings rendered later (e.g. by an async data fetch) are not picked up. For dynamic content, wait for the data to land and then render the `<civ-on-this-page>` alongside it, or use manual mode.
- Headings without an `id` attribute are skipped silently — `IntersectionObserver` needs the heading element so it can fire when it crosses the viewport top.
- `scopeSelector` first-match wins. If the page has a `<main>` *and* an `<article>` (default selector matches both), only the `<main>` is scanned. Override with a more specific selector when you need the article.

## Accessibility

- Renders as a `<nav>` landmark. Even when `label=""` suppresses the visible heading, the landmark gets a default `aria-label="On this page"` so it stays distinguishable from other navs.
- The active item uses `aria-current="location"` so the user's reading position is announced.
- Honors `prefers-reduced-motion: reduce` — the scroll-to-section jump is instant for users who opted out of animations.
- Clicks update `location.hash` via `history.pushState` so deep linking and back-button work. The `href` is sanitized via `sanitizeHref` before writing to history.
- Click activations dispatch `civ-analytics` from the host so consumer-level tracking can listen to in-page navigation events.

## When to use a different component

- For navigation between *pages* (not sections of one page), use `<civ-side-nav>`.
- For top-of-page primary navigation, use `<civ-nav>`.
- For breadcrumb trails, use `<civ-breadcrumb>`.
