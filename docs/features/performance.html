Performance

Minimal footprint, maximum capability. Every byte is earned.

Bundle Sizes

CivUI packages are compiled with TypeScript and minified with esbuild. These are the JavaScript output sizes for each package.

Package Raw (tsc) Minified Gzipped (est.)
@civui/forms 656K 113K ~35K
@civui/core 528K 90K ~28K
@civui/actions 30K 5K ~2K
@civui/overlays 10K 2K ~1K
@civui/layout 8K 1K ~0.5K
@civui/feedback 52K 10K ~4K
@civui/navigation 48K 7K ~2K

CSS is an additional 46K unminified (8K gzipped). A typical form using text-input, select, checkbox, and button imports only what it uses via tree-shaking — you never pay for components you don't use.

Build Pipeline

CivUI uses a 3-step build pipeline that balances developer experience with production performance.

1 tsc -b TypeScript compilation with project references
2 postbuild esbuild minification (44% reduction), preserving file structure
3 build:cdn Optional bundled ESM for CDN delivery

Step 2 preserves the individual file structure so that bundlers in consuming applications can tree-shake effectively. Step 3 is optional and produces self-contained bundles for CDN delivery without a build step.

CDN usage requires a single script tag — no bundler, no build step:

<script type="module" src="https://cdn.example.com/civui.js"></script>

Tree-Shaking

CivUI is designed from the ground up for effective tree-shaking. Bundlers like Rollup, Vite, and webpack can eliminate unused code at build time.

Sub-path Exports

Import only the component you need. Each component has its own entry point, so your bundler never pulls in code for components you don't use.

// Import only text-input (not all 15 form components)
import '@civui/forms/text-input';

Side Effects Configuration

Each package.json declares sideEffects arrays that mark component files (which register custom elements) as side effects, while pure utilities remain fully tree-shakeable. Bundlers use this information to safely eliminate unused exports.

Barrel Exports

For convenience, barrel exports are available when you need multiple components. Modern bundlers will still tree-shake unused exports from barrel imports.

import { CivTextInput, CivSelect } from '@civui/forms';

CSS Optimization

CivUI uses Tailwind CSS with aggressive optimization to minimize stylesheet size.

  • Tailwind CSS with content purging — only classes used in components are generated
  • Full CSS output: 46K unminified, 8K gzipped
  • @layer civ prevents specificity conflicts with consumer styles
  • @apply bundles 5–10 utilities into semantic classes, reducing HTML size
  • Design tokens as CSS custom properties — one source of truth, no duplication

Runtime Performance

Beyond bundle size, CivUI components are optimized at runtime to minimize allocations, reflows, and unnecessary work.

Intl.DateTimeFormat Caching

Formatters are cached per locale, not created per render. Eliminates 42+ allocations per calendar view.

isRtl() WeakMap Cache

getComputedStyle() is called once per element lifetime, not per keydown event.

Batched querySelectorAll

Group components query children once per update cycle, not per sync method.

Debounced Announcements

Textarea character count announced at 1-second intervals, not per keystroke.

Click-Outside Utility

Document listener is only active when a dropdown is open, not permanently attached.

Event Listener Cleanup

All listeners are removed in disconnectedCallback to prevent memory leaks.

requestAnimationFrame Cleanup

RAF IDs are tracked and cancelled in live-region cleanup to avoid orphaned callbacks.

Bundle Size Monitoring

CivUI tracks bundle sizes in CI to prevent regressions.

  • CI checks bundle sizes on every pull request
  • Warns if the forms package exceeds 300K
  • Bundle report appears in the GitHub step summary
  • pnpm build:cdn generates a metafile.json for visual analysis at esbuild.github.io/analyze

What CivUI Doesn't Ship

CivUI follows a zero-dependency philosophy. Every kilobyte must justify its existence.

No Icon Fonts or SVG Sprites

Icons are drawn with CSS pseudo-elements. 0KB additional payload, zero network requests.

No Runtime CSS-in-JS

Tailwind generates static CSS at build time. No JavaScript runs to compute styles.

No Framework Wrappers

Native web components work everywhere — React, Vue, Angular, vanilla HTML.

No Polyfills Required

Targets ES2022 browsers. No compatibility shims shipped.

Lit is the only external runtime dependency at ~16K gzipped. Everything else is built in-house or handled at build time.
