Quick Start

npm install @civds/forms

# Use in HTML
<script type="module">
  import '@civds/forms';
</script>

<civds-text-input
  label="Email address"
  name="email"
  type="email"
  required
></civds-text-input>

Why CivDS?

Accessibility First

Light DOM means external labels, ARIA IDREFs, and focus management work without Shadow DOM workarounds. Every component is WCAG 2.1 AA compliant.

Native Form Participation

ElementInternals enables custom elements to participate in native HTML forms. No hidden inputs or JavaScript hacks.

Design Tokens

W3C DTCG format tokens as the single source of truth. Auto-generated CSS custom properties, Tailwind preset, JS constants, and React Native values.

Cross-Platform

Web components for browsers plus a React Native companion package with the same API. Build once, deploy everywhere.

Packages

@civds/tokens

Design tokens in W3C DTCG format. Outputs CSS custom properties, Tailwind preset, JS constants, and React Native values.

@civds/core

Base classes (CivdsBaseElement, CivdsFormElement), accessibility utilities, and shared helpers.

@civds/forms

Form components: text-input, textarea, select, checkbox, radio, combobox, date-input, file-upload, and more.

@civds/react-native

React Native companion with matching API. TextInput, Textarea, Select, Checkbox, Radio, and useForm() hook.

@civds/cli

Developer CLI for generating components, building, testing, validating, and releasing.

@civds/test-utils

Shared test utilities for component testing with Vitest and jsdom.

Form Example

<form>
  <civds-text-input label="Full name" name="name" required></civds-text-input>
  <civds-text-input label="Email" name="email" type="email" required></civds-text-input>
  <civds-select label="State" name="state" .options="${JSON.stringify([
    { value: '', label: 'Select a state' },
    { value: 'CA', label: 'California' },
    { value: 'NY', label: 'New York' },
  ])}"></civds-select>
  <civds-textarea label="Message" name="message" hint="Max 500 characters"></civds-textarea>
  <civds-checkbox label="I agree to the terms" name="terms" required></civds-checkbox>
  <button type="submit">Submit</button>
</form>

CLI

# Generate a new component
civds generate component my-widget

# Build packages
civds build all

# Run tests
civds test --unit

# Validate everything
civds validate

# Release
civds release patch --dry-run
