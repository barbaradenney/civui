One-Line Locale Setup

Switching CivUI to another language takes a single function call. Set it once at app startup and every component picks it up automatically.

import { setLocaleStrings } from '@civui/core';

setLocaleStrings({
  required: '(obligatorio)',
  selectEmpty: '- Seleccionar -',
  fieldRequired: '{label} es obligatorio',
  comboboxNoResults: 'Sin resultados',
});

Only override what you need — everything else falls back to English defaults. The merge is shallow: your partial object is spread over the full default string set, so untouched keys keep their English values.

RTL Support

CivUI uses logical CSS properties throughout, so components render correctly in both LTR and RTL writing modes without any additional configuration.

  • Logical CSS properties throughout: margin-inline-start not margin-left, inset-inline-start not left, border-inline-start not border-left.
  • Arrow key navigation respects direction: ArrowRight and ArrowLeft swap behavior in RTL via the isRtl() utility.
  • Used in: radio groups, segmented controls, and date picker calendar navigation.
  • Toggle positioning: the toggle thumb uses inset-inline-start for correct positioning in RTL.

How to enable RTL

Set dir="rtl" on a parent element or the <html> tag. CivUI detects this automatically and adjusts all layout and navigation behavior.

<html dir="rtl" lang="ar">
  <civ-text-input label="البريد الإلكتروني" required></civ-text-input>
</html>

Locale-Aware Dates

The date picker and memorable date components use Intl.DateTimeFormat to produce locale-appropriate month names and day-of-week headers.

  • Pass a locale prop to get localized month and day names: <civ-date-picker locale="fr-FR">
  • formatDateLong() produces locale-appropriate long date strings for screen reader announcements (e.g., "15 janvier 2026" in French).
  • Locale data is cached per locale for performance — subsequent calls reuse the formatter.
<!-- French locale: month/day names render in French -->
<civ-date-picker
  label="Date de rendez-vous"
  locale="fr-FR"
></civ-date-picker>

<!-- Japanese locale -->
<civ-memorable-date
  label="生年月日"
  locale="ja-JP"
></civ-memorable-date>

Component-Level Overrides

Props always win over locale defaults. If a component has a prop for a translatable string, setting that prop overrides whatever the locale registry provides. This lets you customize individual instances without changing the global locale.

<!-- Uses locale default for empty label -->
<civ-select label="State"></civ-select>

<!-- Overrides with custom text -->
<civ-select label="Estado" empty-label="- Elegir -"></civ-select>

Full Spanish Example

Here is a complete locale setup for a Spanish-language government form. This demonstrates how all i18n features work together — locale strings, date formatting, and component labels.

import { setLocaleStrings } from '@civui/core';

setLocaleStrings({
  required: '(obligatorio)',
  selectEmpty: '- Seleccionar -',
  fieldRequired: '{label} es obligatorio',
  fieldInvalid: '{label} no es válido',
  formErrorSingular: 'Hay 1 error en este formulario',
  formErrorPlural: 'Hay {count} errores en este formulario',
  fileUploadDragText: 'Arrastra archivos aquí o',
  fileUploadBrowseText: 'elige desde carpeta',
  datePickerChooseDateLabel: 'Elegir fecha',
  memorableDateMonthLabel: 'Mes',
  memorableDateDayLabel: 'Día',
  memorableDateYearLabel: 'Año',
});
<civ-form>
  <civ-text-input
    label="Nombre completo"
    name="name"
    required
    required-message="Ingrese su nombre completo"
  ></civ-text-input>

  <civ-text-input
    label="Correo electrónico"
    name="email"
    type="email"
    required
    hint="Por ejemplo: nombre@agencia.gov"
  ></civ-text-input>

  <civ-memorable-date
    label="Fecha de nacimiento"
    name="dob"
    locale="es-ES"
    hint="Por ejemplo: 15 enero 1990"
  ></civ-memorable-date>

  <civ-select
    label="Estado"
    name="state"
  ></civ-select>

  <button type="submit">Enviar</button>
</civ-form>
