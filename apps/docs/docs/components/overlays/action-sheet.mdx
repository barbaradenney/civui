`<civ-action-sheet>`

A shared popup/overlay component that renders as an absolute dropdown on desktop and a fixed bottom sheet on mobile (≤480px). Use it for any popup content. Menus, confirmations, option lists, or custom overlays.

The parent component controls the `open` state. The action sheet fires `civ-close` when it wants to close and the parent decides whether to close.

## Behavior

### Desktop (>480px)

Renders as an absolute-positioned dropdown below its parent. No backdrop.

### Mobile (≤480px)

Renders as a fixed bottom sheet:
- Full-width, anchored to bottom of viewport
- Rounded top corners with upward shadow
- Semi-transparent backdrop overlay
- Scrollable content with configurable max-height
- Tap backdrop or press Escape to close

### Keyboard

- **Escape:** fires `civ-close`
- **Tab:** cycles through focusable elements (when `trap-focus` is set)

## Usage

```html
<!-- Basic popup menu -->
<div style="position: relative;">
  <civ-button label="Options" @click="${() => this.menuOpen = !this.menuOpen}"></civ-button>
  <civ-action-sheet
    ?open="${this.menuOpen}"
    @civ-close="${() => this.menuOpen = false}"
  >
    <ul>
      <li>Edit</li>
      <li>Duplicate</li>
      <li>Delete</li>
    </ul>
  </civ-action-sheet>
</div>

<!-- Confirmation dialog with focus trap -->
<civ-action-sheet
  ?open="${this.confirmOpen}"
  trap-focus
  @civ-close="${() => this.confirmOpen = false}"
>
  <div class="civ-p-4">
    <p>Are you sure?</p>
    <div class="civ-button-row">
      <civ-button emphasis="secondary" label="Cancel" @click="${() => this.confirmOpen = false}"></civ-button>
      <civ-button label="Confirm" @click="${this.onConfirm}"></civ-button>
    </div>
  </div>
</civ-action-sheet>
```

## Design principle

All popups, dropdowns, and dialogs in CivUI must use bottom sheets on mobile. The action sheet component enforces this automatically. Components that currently implement their own popup CSS (date picker, combobox) follow the same pattern.
