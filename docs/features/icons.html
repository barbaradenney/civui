Zero-Dependency Icons

44 pure-CSS icons drawn with ::before and ::after pseudo-elements. No icon fonts, no SVG sprites, no external dependencies.

No Network Requests Inherits Color Scales with Font Size Works Everywhere

The Approach

CivUI icons are drawn with pure CSS — each icon is a class (civ-icon--name) that paints a shape using ::before and ::after pseudo-elements with borders, clip-path, and transforms. No font files, no SVG sprites, no CDN. Each icon is a <civ-icon> custom element that inherits currentColor from its parent, scales with font-size, and has a platform mapping to SF Symbols on iOS and Material Symbols on Android.

Benefits

Inherit Color

Icons use currentColor, so they match surrounding text automatically. Change the parent color and the icon follows.

Scale with Font Size

Icons are sized in em units. They scale proportionally with the text around them, no extra sizing needed.

Zero Network Requests

No font files, no SVGs, no CDN. Icons are part of the component bundle. Nothing extra to download at runtime.

Works in Any Context

Renders in email clients, print stylesheets, high-contrast mode, and screen readers with proper aria-label support.

Usage

<!-- Basic usage -->
<civ-icon name="check"></civ-icon>

<!-- With size variant -->
<civ-icon name="search" size="lg"></civ-icon>

<!-- Inherits parent color -->
<span style="color: #00a91c;">
  <civ-icon name="check"></civ-icon> Approved
</span>

<!-- Inside a button -->
<button>
  <civ-icon name="plus"></civ-icon> Add item
</button>

<!-- Accessible label for standalone icons -->
<civ-icon name="close" aria-label="Close dialog"></civ-icon>

Custom Icons

Register custom icons using the registerIcon() API to extend the built-in library with your own Unicode compositions.

import { registerIcon } from '@civui/core';

// Register a single custom icon
registerIcon('agency-seal', {
  characters: ['\u2B21', '\u2605'],  // hexagon + star
  label: 'Agency seal',
});

// Use it like any built-in icon
// <civ-icon name="agency-seal"></civ-icon>
