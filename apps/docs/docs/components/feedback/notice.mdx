`<civ-notice>`

Icon-prefixed emphasis text for highlighting a specific passage inside
longer content — legal, safety, or financial consequences a reader
could miss in flowing prose. Based on GOV.UK's
[warning text](https://design-system.service.gov.uk/components/warning-text/)
pattern, generalized with semantic intents (`info` / `warning` /
`error` / `success` / `neutral`).

There is no background and no dismiss button — just a large semantic
icon plus bold paragraphs. This is the lightweight inline variant of
the alert / callout family, and it sits in `@civui/feedback` alongside
`civ-alert` and `civ-badge`.

## When to use

| You need | Use |
|---|---|
| Emphasis on a passage the user is already reading ("You can be fined…") | **`civ-notice`** |
| A notification handed to the user with dismiss + live-region announce | [`civ-alert`](./alert) |
| A panel surface with background + rich body content + inline links | [`civ-callout`](./callout) |
| A panel with heading + body as a section opener | [`civ-section-intro`](../form/section-intro) |

The component is display-only and not form-associated.

## Content model

`header` and `body` are string props — the component owns the HTML so
the rendered markup stays semantically correct. For body content that
needs inline links or other rich markup, reach for
[`civ-callout`](./callout) instead.

```html
<civ-notice
  intent="warning"
  header="You must register"
  body="You can be fined up to $5,000 if you do not register."
></civ-notice>
```

Either prop is optional; the icon renders even with no text. In
practice, set at least `body`.

## Intents and default icons

Each intent picks a default icon from the CivUI icon library. The
`iconStyle` prop chooses between two visual treatments — `filled`
(default, heavier presence) and `outline` (lighter, softer cue).
Override the icon name entirely via the `icon` prop when a placement
needs a different glyph.

| Intent | Filled default (default) | Outline default | Color |
|---|---|---|---|
| `info` (default) | `info-fill` | `info` | info |
| `warning` | `warning-fill` | `warning` | warning-dark |
| `error` | `error-fill` | `error` | error |
| `success` | `check-circle-fill` | `check-circle` | success-dark |
| `neutral` | `info-fill` | `info` | base-dark (gray) |

Color is never the sole signal — the icon shape and the bold weight
on the header carry the emphasis too, so colorblind users get the
full meaning.

```html
<!-- Default: filled icon -->
<civ-notice intent="warning" body="You can be fined up to $5,000."></civ-notice>

<!-- Opt into the lighter outline glyph -->
<civ-notice intent="warning" icon-style="outline" body="Optional reminder."></civ-notice>
```

## Spacing

| Value | When to use | Approximate sizes |
|---|---|---|
| `default` | Standalone block or inside a card / page section; matches GOV.UK warning-text presence | ~2.5rem icon + lg body text |
| `sm` | Inline within body prose where the affordance shouldn't dominate the reading flow | ~1.5rem icon + base body text |

## Accessibility

- The icon carries `aria-hidden="true"`; the visual treatment isn't
  re-announced by screen readers.
- The optional `header` renders as `<p role="heading" aria-level="N">`
  so the heading is surfaced via the screen-reader rotor without
  forcing a specific `<h*>` level into the page outline.
- For severity that should be spoken before the header, set
  `sr-prefix` to a string like `"Warning:"`. It renders in a
  visually-hidden span before the content. The prop is empty by
  default — teams opt in per placement so the announcement matches
  the editorial tone of the surrounding copy.
- Body and header text inherit the default color (base-darkest), per
  the CivUI rule that visual hierarchy comes from font size and
  weight, not color muting.

## Related

- [`civ-alert`](./alert) — dismissible banner with ARIA live region.
  Use when the message is a notification. Accepts `civ-notice` as a
  composed child for the icon-prefixed body emphasis.
- [`civ-callout`](./callout) — panel with rail + background.
  Use when the surface needs heading + rich body content. Also
  accepts `civ-notice` as a composed child.
- [`civ-section-intro`](../form/section-intro) — section opener built
  on `civ-callout` with a heading and a `role="region"` landmark.
- [`civ-badge`](./badge) — small status indicator. Pairs well with
  notice inside an alert or callout body.
