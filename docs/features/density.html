Density Scale

Responsive spacing and typography that adapts to context — from data-heavy dashboards to accessibility-focused citizen forms.

Three Density Modes

CivUI ships three density scales that adjust spacing and font sizes across all components. Each mode overrides the design token CSS variables, so every component automatically responds when a parent element sets data-civ-scale.

Property Dense 0.75x Default 1x Spacious 1.5x
Base font size 14px (0.875rem) 16–18px fluid 18–22px fluid
Line height 1.4 1.6 1.75
Spacing factor 0.75x (e.g., 12px for spacing-4) 1x (e.g., 20px for spacing-4) 1.5x (e.g., 30px for spacing-4)
Best for Data-heavy dashboards, admin panels, tables Standard forms and pages Accessibility-focused, large touch targets
Font scaling Fixed sizes (no fluid) Fixed sizes Fluid clamp() values

Fluid Typography

In spacious mode, font sizes use CSS clamp() to scale fluidly between a minimum and maximum value based on viewport width. This ensures text is comfortable to read on any screen size without abrupt breakpoints.

320px viewport
18px
Scales smoothly
1200px viewport
22px

The formula follows the pattern:

font-size: clamp(min, preferred, max);

/* Example: spacious base font size */
font-size: clamp(1.125rem, calc(1.0341rem + 0.4545vw), 1.375rem);
/*         18px min    fluid middle               22px max */

The preferred value uses calc() with a vw unit so the font size interpolates linearly between the min (at 320px viewport) and max (at 1200px viewport). Larger type scales like headings grow more aggressively, creating a natural typographic hierarchy that adapts to screen size.

How to Use

Set the data-civ-scale attribute on any parent element. All CivUI components inside that element inherit the density scale via CSS custom property overrides.

Wrap a section

<!-- Dense: compact dashboard layout -->
<div data-civ-scale="dense">
  <civ-text-input label="Search" name="q"></civ-text-input>
  <civ-select label="Filter" name="filter" .options="..."></civ-select>
</div>

Wrap an entire page

<!-- Spacious: accessibility-first citizen form -->
<body data-civ-scale="spacious">
  <civ-form>
    <civ-text-input label="Full name" name="name" required
      required-message="Enter your full name"></civ-text-input>
    <civ-memorable-date label="Date of birth" name="dob"
      hint="For example: January 15 1990"></civ-memorable-date>
    <button type="submit">Submit</button>
  </civ-form>
</body>

Mix densities on one page

<!-- Dense sidebar + spacious main content -->
<aside data-civ-scale="dense">
  <!-- Compact navigation and filters -->
</aside>
<main data-civ-scale="spacious">
  <!-- Generous form with large touch targets -->
</main>

Toggle dynamically with JavaScript

// Let the user pick their preferred density
function setDensity(scale) {
  if (scale === 'default') {
    document.body.removeAttribute('data-civ-scale');
  } else {
    document.body.setAttribute('data-civ-scale', scale);
  }
}

// Example: respond to a user preference control
densitySelect.addEventListener('change', (e) => {
  setDensity(e.target.value);
});

How It Works

The density system is powered entirely by CSS custom properties. The scales.tokens.json file in @civui/tokens defines three scale variants: dense, spacious, and fluid. The token build pipeline generates CSS variable overrides scoped to the [data-civ-scale] attribute selector.

Because CivUI uses Light DOM, there are no Shadow DOM boundaries blocking CSS inheritance. Setting data-civ-scale on a parent element cascades the overridden spacing and typography variables to every descendant component.

Default density requires no attribute — the base :root token values apply. Dense and spacious modes override only the values that change, keeping the CSS payload small.
