<!-- contract-link:end -->

`<civ-confirmation-panel>`

Post-submission success surface for government forms. Pairs naturally with [`<civ-form>`](./form) and [`<civ-summary>`](./summary) as the "after submit" sibling to the "before submit" review page.

Aligned with the [USWDS confirmation pattern](https://designsystem.digital.gov/templates/form-templates/confirmation/) — a restrained left-border success treatment on the header rather than a full-bleed banner. Heading sits in normal body color; the reference number lives in a bordered summary-box callout below.

## When to use it

- After a form submission that creates a case, claim, application, or appointment.
- After any one-shot action where the user needs assurance the system received their input ("Your message has been sent", "Your subscription is confirmed").
- After eligibility checks where the answer IS the receipt.

## Reference numbers are optional

GOV.UK's pattern shows reference numbers as a hard requirement; USWDS treats them as optional, and so does this component. Common cases that skip the reference:

- **Informational submissions** (subscribe to updates, request a brochure) — no case file is opened.
- **Anonymous tip lines / feedback forms** — no tracking identifier is returned by design.
- **Real-time eligibility checks** — the answer IS the receipt.
- **Async backends** — the reference doesn't exist until a backend job finishes. Use the `pending-reference` slot to explain when and how the user will receive it.

## Slots

| Marker attribute | Purpose |
|------------------|---------|
| _(none — default)_ | Body content shown under the heading. Typically 1–2 short paragraphs confirming what happened and what was sent. Any direct child without a `data-*` slot marker lands here. |
| `data-pending-reference` | Rendered in place of the reference callout when `reference` is omitted. Use for "We'll email your reference number within 24 hours" copy. |
| `data-next-steps` | "What happens next" block. Author the heading and paragraphs / list freely. |
| `data-actions` | Buttons / links cluster. Wrapped automatically in `civ-button-row` so primary + secondary stack on mobile per the [government-patterns rule](https://github.com/barbaradenney/civui/blob/main/.claude/rules/government-patterns.md). |

CivUI uses `data-*` attribute markers on slotted children (rather than the standard `slot="…"` attribute) for the same reason the Light-DOM mixin needs to capture children before Lit renders — `data-*` attributes survive HTML parsing without triggering the browser's native slot-fallback rendering before the component upgrades.

## Accessibility

- Renders a `role="status"` region labelled by the heading.
- Heading is rendered at `aria-level="1"` by default (the panel typically replaces the form page entirely). Override with `heading-level` when embedding under another heading.
- On mount, focus moves to the heading and the heading text is announced politely via the shared CivUI live region. Suppress with `no-autofocus` when the surrounding page handles focus / SR announcements.
- The success icon is `aria-hidden`; the heading carries the accessible name.

## Print

`@media print` hides the action row (Print / Return navigation isn't meaningful on paper) and applies `break-inside: avoid` to the success header and reference callout so they aren't split across pages.

## Usage

```html
<!-- Full confirmation with reference number -->
<civ-confirmation-panel
  heading="Application complete"
  reference="HDJ2-123F"
>
  <p>We have sent a confirmation email to sarah@example.gov.</p>

  <div data-next-steps>
    <h2>What happens next</h2>
    <p>Your application will be reviewed within 5 working days.</p>
  </div>

  <div data-actions>
    <civ-button label="Print confirmation" variant="secondary"></civ-button>
    <civ-link href="/dashboard">Return to dashboard</civ-link>
  </div>
</civ-confirmation-panel>

<!-- Informational submission — no reference -->
<civ-confirmation-panel heading="Subscription confirmed">
  <p>You're signed up to receive benefit updates at sarah@example.gov.</p>
  <div data-actions>
    <civ-link href="/">Return to home</civ-link>
  </div>
</civ-confirmation-panel>

<!-- Async backend — reference comes later -->
<civ-confirmation-panel heading="Claim received">
  <p>We're processing your claim.</p>
  <p data-pending-reference>
    <strong>We'll email your reference number within 24 hours.</strong>
  </p>
</civ-confirmation-panel>
```
