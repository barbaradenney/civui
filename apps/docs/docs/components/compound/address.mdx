# Address

<!-- contract-link:start -->

> **Contract reference:** [`civ-address`](/contract/civ-address) — auto-generated structured contract for cross-platform implementation.

<!-- contract-link:end -->

`<civ-address>`

Structured address input with country selection, street, city, state, and ZIP/postal code fields. Adapts automatically based on the selected country — US addresses show a state dropdown (including military codes AA/AE/AP), other countries show a text input for state/province, and countries without states or postal codes hide those fields entirely.

| `heading-level` | `1`–`6` | — | Promote the legend to a heading via `role="heading"` + `aria-level=N`. |

| `zip-error` | `string` | `''` | Error for ZIP code field |

## Variants

### Default

The standard full address form with country, street, city, state, and ZIP/postal code.

### General Delivery (`variant="general-delivery"`)

For USPS General Delivery addresses. Renders the same fields as the default variant but adds a hint explaining how to use General Delivery (enter the post office city/state/ZIP, use "General Delivery" as the street address).

### Contact (`variant="contact"`)

For situations where someone cannot receive mail. Renders a simplified form with just city/state and a contact method textarea (phone, email, shelter name, or someone who can relay messages).

## Value Structure

The component stores its value as a JSON string:

```typescript
interface AddressValue {
  country: string;   // ISO 3166-1 alpha-2 code (e.g., 'US', 'GB', 'JP')
  street1: string;
  street2: string;
  street3: string;
  city: string;
  state: string;     // 2-letter code for US, free text for others
  zip: string;
}
```

## Country-Adaptive Fields

The form adapts based on the selected country:

| Country | State field | Postal code | Labels |
|---------|-----------|-------------|--------|
| **United States** | Select dropdown (50 states + territories + AA/AE/AP) | Shown | "State", "ZIP code" |
| **UK, Canada, etc.** | Text input | Shown | "State/Province/Region", "Postal code" |
| **Hong Kong, Singapore, etc.** | Hidden (no states) | Hidden (no postal codes) | — |

Military addresses use the standard form — select AA, AE, or AP from the State dropdown and enter APO, FPO, or DPO as the city.

## Sub-field Autocomplete

Fields automatically set appropriate `autocomplete` values:
- Country: `country`
- Street: `address-line1`, `address-line2`, `address-line3`
- City: `address-level2`
- State: `address-level1`
- ZIP: `postal-code`

## Events

| Event | Detail | Description |
|-------|--------|-------------|
| `civ-input` | `{ value: AddressValue }` | Fires on every field change |
| `civ-change` | `{ value: AddressValue }` | Fires on committed field change |

## Usage

```html
<!-- Standard address -->
<civ-address
  legend="Mailing address"
  name="mailing"
  required
></civ-address>

<!-- With extra street line -->
<civ-address
  legend="Complete address"
  name="fullAddr"
  show-street3
  required
></civ-address>

<!-- Minimal (no street2) -->
<civ-address
  legend="Home address"
  name="home"
  show-street2="false"
  required
></civ-address>
```
