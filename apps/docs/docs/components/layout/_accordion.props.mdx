| Attribute | Type | Default | Description |
|-----------|------|---------|-------------|
| `single` | `boolean` | `false` | When true, opening one item collapses any other open siblings. Default behavior (omitted) allows multiple items open at once. The single-open invariant is enforced on first paint AND when this prop transitions from false to true at runtime. |
| `disabled` | `boolean` | `false` | Disables every direct-child item. Each item checks the nearest ancestor accordion's `disabled` when deciding whether to accept a toggle; setting `disabled` here cascades to all items without mutating their own `disabled` prop. Individual items can also be disabled independently. |
| `variant` | `'primary'` \| `'secondary'` \| `'tertiary'` | `'tertiary'` | Visual variant. `tertiary` (default) is a bordered group with transparent triggers — list-like, quiet. `secondary` renders each item as a filled neutral-gray chip (base-lightest bg, default text color) matching `civ-disclosure` / `civ-read-more`; when open, the content drops the bg and uses a disclosure-style left-edge gray rail with indent. `primary` renders as a filled primary-lightest (blue) button with the larger padding and bolder type of `civ-btn--secondary`; when open, trigger + content extend the colored bg as a single card. Items in primary / secondary stack with a small gap; no outer border or inter-item dividers. |
