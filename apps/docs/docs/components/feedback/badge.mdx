# Badge

`<civ-badge>`

A compact status or count indicator. Use for state ("Approved", "Pending"), severity, or numeric counts. Always carries `role="status"` so assistive tech announces the value. For categorization (topics, taxonomies), use [`civ-tag`](../layout/tag) instead.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | `''` | Badge text (or accessible name when `dot` is set) |
| `count` | `number \| null` | `null` | Numeric value; renders as a count badge |
| `max` | `number` | `99` | Overflow threshold; values above render as `{max}+` |
| `dot` | `boolean` | `false` | Render as a small colored dot only |
| `variant` | `'info' \| 'warning' \| 'error' \| 'success' \| 'neutral'` | `'neutral'` | Semantic color |
| `badge-style` | `'primary' \| 'secondary'` | `'secondary'` | Emphasis: primary (filled dark bg, light text) or secondary (light tint, dark text) |
| `spacing` | `'default' \| 'sm'` | `'default'` | Compact variant with smaller padding |
| `overlay` | `boolean` | `false` | Position absolutely in the top-end corner of a relatively-positioned parent (notification mode) |

## Modes

Precedence: **dot > count > label**.

### Status text

```html
<civ-badge label="Approved" variant="success"></civ-badge>
<civ-badge label="Pending" variant="warning"></civ-badge>
<civ-badge label="Denied" variant="error"></civ-badge>
```

### Count

```html
<civ-badge count="5" variant="info"></civ-badge>
<civ-badge count="150" max="99" variant="error"></civ-badge>
<!-- renders as "99+" -->
```

### Dot

```html
<civ-badge dot label="Unread messages" variant="error"></civ-badge>
<!-- label becomes aria-label, no visible text -->
```

## Accessibility

- Always renders `role="status"` so screen readers announce the badge value.
- In dot mode, `label` becomes the `aria-label` so the marker has an accessible name.
- Restricted to semantic colors so visual meaning matches the announced value.

### Notification overlay

Wrap an icon (or any container) with `civ-badge-anchor` to anchor the badge:

```html
<button class="civ-badge-anchor">
  <civ-icon name="mail" size="lg"></civ-icon>
  <civ-badge overlay count="3" variant="error"></civ-badge>
</button>
```

The `civ-badge-anchor` utility class sets `position: relative; display: inline-block`. Any other relatively-positioned parent works too.

## Tag vs Badge

| Use case | Component |
|----------|-----------|
| Status pill ("Approved", "Pending", "Error") | `civ-badge` |
| Count / notification indicator (`5`, `99+`) | `civ-badge` (with `count`) |
| Notification dot | `civ-badge` (with `dot`) |
| Categorization ("Healthcare", "Disability") | [`civ-tag`](../layout/tag) |
| Filter chip / metadata | [`civ-tag`](../layout/tag) |

`civ-tag`'s color set excludes red/green/yellow/teal — those semantic colors only appear via `civ-badge`.
