`<civ-filter-chip>` · `<civ-filter-chip-group>`

An interactive, button-like control for filter selection. Click to toggle the chip's `selected` state. Use horizontal rows of chips to represent active or available filters.

For removable user-entered tokens (recipient lists, applied-filter readouts), use [`civ-input-chip`](./input-chip) instead — its always-present × is the canonical removable-chip affordance. For static categorization labels use [`civ-tag`](../layout/tag); for status indicators use [`civ-badge`](../feedback/badge); for primary CTAs use [`civ-button`](./button).

When you have more than two or three chips, wrap them in `<civ-filter-chip-group>` to get keyboard navigation and (optionally) single-select coordination. See [the group section](#group) below.

## Accessibility

- The chip renders a non-interactive `<span role="presentation">` wrapper containing one real `<button>` element. The wrapper exists so the chip's chrome (background, border, focus ring) sits at the same DOM level as siblings in the chip family.
- In default `variant="toggle"` mode the action button uses `aria-pressed`. In `variant="radio"` (set by a single-select group) it uses `role="radio"` + `aria-checked`. Re-clicking an already-selected radio is a no-op (matches platform radio semantics).
- Touch targets meet WCAG 2.5.8 (24×24 minimum) in both default and `--sm` sizes.

## Patterns

### Faceted search results

Wire `civ-change` to your filter store; let the chip own its `selected` state and observe the event:

```html
<civ-filter-chip label="Healthcare" value="healthcare"></civ-filter-chip>
<civ-filter-chip label="Education" value="education"></civ-filter-chip>
<civ-filter-chip label="Housing" value="housing"></civ-filter-chip>

<script>
  document.addEventListener('civ-change', (e) => {
    const { value, selected } = e.detail;
    console.log(value, selected ? 'applied' : 'removed');
  });
</script>
```

### Applied-filters bar

For "currently-applied filters with × to dismiss" — use [`civ-input-chip`](./input-chip) instead. Filter-chip is toggle-only; the input-chip's always-present remove handle is the canonical removable-chip affordance:

```html
<civ-input-chip label="Healthcare" value="healthcare"></civ-input-chip>
<civ-input-chip label="VA Form 21-526EZ" value="form-21-526ez"></civ-input-chip>
```

## Anti-patterns

- **Don't use a filter chip as a category label.** If it doesn't filter anything, use `civ-tag` (a static `<span>`).
- **Don't use a filter chip for status.** Use `civ-badge`, which carries `role="status"` for assistive tech.
- **Don't use a filter chip as a primary action.** Use `civ-button` for "Submit", "Save", etc.
- **Don't reach for `removable` on a filter chip.** The prop was removed in the chip-family responsibility split — toggleable selection lives here, removable user-entered tokens live on [`civ-input-chip`](./input-chip).

## Group

`<civ-filter-chip-group>` coordinates a row of chip children. Adds **roving tabindex** so the row is one Tab stop with arrow-key navigation, enforces single-select if `mode="single"`, and emits an aggregated `civ-change` carrying the full selected value(s).

### Group properties

| Property | Type | Description |
|----------|------|-------------|
| `value` | `string \| string[]` | Currently selected value(s). Single mode returns a string (or `''` if none); multi mode returns an array. **Settable**. Assigning updates each child chip's `selected` flag to match. |

### Keyboard

| Key | Action |
|-----|--------|
| `Tab` | Move into / out of the group (one stop, lands on the first selected chip or first chip) |
| `→` / `↓` | Focus next chip |
| `←` / `↑` | Focus previous chip (wraps at the start) |
| `Home` | Focus first chip |
| `End` | Focus last chip |
| `Enter` / `Space` | Toggle the focused chip (handled by the chip itself) |

Disabled chips are skipped during arrow-key navigation.

### Group patterns

#### Faceted search with multi-select

```html
<civ-filter-chip-group mode="multi" label="Filter results">
  <civ-filter-chip label="Healthcare" value="health" count="24"></civ-filter-chip>
  <civ-filter-chip label="Education" value="education" count="12"></civ-filter-chip>
  <civ-filter-chip label="Housing" value="housing" count="8"></civ-filter-chip>
</civ-filter-chip-group>

<script>
  document.querySelector('civ-filter-chip-group').addEventListener('civ-change', (e) => {
    console.log('selected:', e.detail.value); // ['health', 'housing']
  });
</script>
```

#### View mode picker (single-select)

```html
<civ-filter-chip-group mode="single" label="Sort">
  <civ-filter-chip label="Newest" value="newest" selected></civ-filter-chip>
  <civ-filter-chip label="Oldest" value="oldest"></civ-filter-chip>
  <civ-filter-chip label="A → Z" value="alpha"></civ-filter-chip>
</civ-filter-chip-group>
```

#### Appointment time picker

Use `mode="single"` with compact `spacing="sm"` chips to show available appointment slots for each date. Mark unavailable times as `disabled`. Each date gets its own group so the user picks one time per day.

```html
<h3>Monday, May 5</h3>
<civ-filter-chip-group mode="single" label="Available times for May 5">
  <civ-filter-chip label="9:00 AM" value="0900" spacing="sm"></civ-filter-chip>
  <civ-filter-chip label="9:30 AM" value="0930" spacing="sm"></civ-filter-chip>
  <civ-filter-chip label="10:00 AM" value="1000" spacing="sm" disabled></civ-filter-chip>
  <civ-filter-chip label="10:30 AM" value="1030" spacing="sm"></civ-filter-chip>
  <civ-filter-chip label="11:00 AM" value="1100" spacing="sm"></civ-filter-chip>
  <civ-filter-chip label="1:00 PM" value="1300" spacing="sm"></civ-filter-chip>
  <civ-filter-chip label="2:00 PM" value="1400" spacing="sm"></civ-filter-chip>
</civ-filter-chip-group>

<script>
  document.querySelector('civ-filter-chip-group').addEventListener('civ-change', (e) => {
    console.log('selected time:', e.detail.value); // e.g. '1030'
  });
</script>
```

Use `emphasis="primary"` for stronger visual emphasis on the selected time slot.
