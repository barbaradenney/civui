# Validation & Masking

CivUI provides built-in validators for common field types and input masks for formatted fields like phone numbers, SSNs, and currency.

## Validators

15 built-in validators via `validate` from `@civui/core`. Each returns `{ valid: boolean, error?: string }`.

| Validator | Description |
|-----------|-------------|
| `required` | Value is not empty |
| `email` | Valid email address |
| `phone` | US phone number |
| `phoneIntl` | International phone number |
| `ssn` | Social Security number |
| `ein` | Employer Identification Number |
| `zip` / `zip4` | 5-digit ZIP / ZIP+4 |
| `usState` | US state abbreviation |
| `isoDate` | ISO 8601 date (YYYY-MM-DD) |
| `url` | Valid URL |
| `currency` | Currency amount |
| `range` | Numeric range |
| `length` | String length range |
| `alphanumeric` | Letters and numbers only |

### Declarative validation

Set the `validate` attribute on form components to auto-validate on submit:

<StoryEmbed id="foundations-validation--declarative-validation" />

```html
<civ-text-input label="Email" name="email" validate="email" required></civ-text-input>
<civ-text-input label="SSN" name="ssn" validate="ssn" required></civ-text-input>
<civ-text-input label="ZIP code" name="zip" validate="zip" required></civ-text-input>
```

### Programmatic validation

```javascript
import { validate } from '@civui/core';

const result = validate.email('user@example.com');
// { valid: true }

const bad = validate.ssn('000-12-3456');
// { valid: false, error: 'Enter a valid Social Security number' }
```

### Form-level validation

`<civ-form>` coordinates validation across all its child fields:

1. **Validate on submit, not on blur** — government form users often tab through fields to understand the form first.
2. **Error summary at top** — renders automatically with anchor links to each invalid field.
3. **Custom required messages** — use `required-message` for field-specific instructions.

<StoryEmbed id="foundations-validation--form-validation" />

```html
<civ-form @civ-submit="${handleSubmit}" @civ-invalid="${handleErrors}">
  <civ-text-input label="Full name" name="name" required
    required-message="Enter your full name"></civ-text-input>
  <civ-text-input label="Email" name="email" type="email" required
    validate="email"></civ-text-input>
  <button type="submit">Submit</button>
</civ-form>
```

### Server-side validation

After server validation, set errors on individual fields:

```javascript
async function handleSubmit(e) {
  const response = await fetch('/api/submit', {
    method: 'POST',
    body: JSON.stringify(e.detail.formData)
  });

  if (!response.ok) {
    const { errors } = await response.json();
    for (const err of errors) {
      const field = document.querySelector(`[name="${CSS.escape(err.field)}"]`);
      if (field) field.error = err.message;
    }
  }
}

// Clear errors before re-submit
formEl.clearErrors();
```

## Input masks

Input masking for formatted fields. CivUI uses blur-mode by default — the mask is applied on blur and raw input is shown on focus.

### Presets

| Preset | Pattern | Example | PII |
|--------|---------|---------|-----|
| `ssn` | `###-##-####` | 123-45-6789 | Yes |
| `phone-us` | `(###) ###-####` | (555) 123-4567 | No |
| `zip` | `#####` | 20500 | No |
| `zip4` | `#####-####` | 20500-1234 | No |
| `ein` | `##-#######` | 12-3456789 | Yes |
| `currency` | (variable) | $1,234.56 | No |

<StoryEmbed id="foundations-input-masks--all-presets" />

```html
<civ-text-input label="Phone number" name="phone" mask="phone-us"></civ-text-input>
<civ-text-input label="Social Security number" name="ssn" mask="ssn"></civ-text-input>
<civ-text-input label="ZIP code" name="zip" mask="zip"></civ-text-input>
```

### Custom patterns

| Character | Meaning |
|-----------|---------|
| `#` | Digit (0-9) |
| `A` | Letter (a-z, A-Z) |
| `*` | Any character |
| Other | Literal separator |

<StoryEmbed id="foundations-input-masks--custom-patterns" />

```html
<civ-text-input label="Case number" name="case" mask-pattern="AA-####"></civ-text-input>
<civ-text-input label="License plate" name="plate" mask-pattern="AAA-####"></civ-text-input>
```

### Mask modes

- **`blur`** (default): Mask applied on blur. Most accessible.
- **`live`**: Formats as user types. Use sparingly — has cursor-jumping tradeoffs.

<StoryEmbed id="foundations-input-masks--mask-modes" />

### PII protection

Presets flagged as PII (`ssn`, `ein`) automatically:
- Set `autocomplete="off"` on the input
- Add `data-civ-pii` attribute for persistence exclusion
- Enable blur-mode masking to avoid exposing sensitive data while typing
