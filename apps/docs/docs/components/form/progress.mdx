# Progress

CivUI provides three progress indicator components for multi-step forms:

- **`civ-progress-steps`** ŌĆö segmented progress bar (one rectangle per step)
- **`civ-progress-header`** ŌĆö compact "Step X of Y: Title" text header
- **`civ-progress-percent`** ŌĆö percentage-based progress bar

All three are used by `civ-form-step` via the `progress` prop, or standalone.

Each step is a small rectangle ŌĆö filled for completed, highlighted for current, and empty for upcoming. Compact at any screen size.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `steps` | `string` | `'[]'` | JSON array of step labels or step objects (`{label}`) |
| `current` | `number` | `0` | Current step index (0-based) |
| `clickable` | `boolean` | `false` | When true, completed segments become clickable |
| `error-steps` | `string` | `'[]'` | JSON array of step indices with validation errors |

### Events

| Event | Detail | Description |
|-------|--------|-------------|
| `civ-step-click` | `{ step: number }` | Fired when a completed segment is clicked (requires `clickable`) |

### Usage

```html
<civ-progress-steps
  steps='["Personal Info", "Address", "Review", "Submit"]'
  current="1"
  clickable
></civ-progress-steps>
```

## Using with `civ-form-step`

The `progress` prop on `civ-form-step` selects which indicator to display:

```html
<!-- Compact header (default) -->
<civ-form-step progress="minimal">...</civ-form-step>

<!-- Segmented bar -->
<civ-form-step progress="steps">...</civ-form-step>

<!-- Percentage bar -->
<civ-form-step progress="bar">...</civ-form-step>
```

The back link is rendered consistently above all three indicators and is handled by `civ-form-step`.
