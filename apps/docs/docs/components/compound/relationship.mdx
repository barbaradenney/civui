`<civ-relationship>`

Compound component for capturing a person and their relationship to the applicant. Includes conditional follow-up fields based on the relationship category — spousal types show marriage date, child types show date of birth, etc.

Use the `preset` prop for pre-defined option lists, or pass custom `options` to override.

## Presets

Each preset provides a curated list of relationship types for a specific context:

| Preset | Options | Use case |
|--------|---------|----------|
| `general` | Spouse, Child, Parent, Sibling, Other | Generic forms |
| `dependent` | Spouse, Biological child, Adopted child, Stepchild, Parent | Dependency claims |
| `survivor` | Spouse, Child, Parent, Executor, Funeral director, Other | Burial and survivor benefits |
| `benefits-survivor` | Spouse, Former spouse, Child, Grandchild, Parent | Survivor benefits with extended family |
| `immigration` | Spouse, Biological child, Adopted child, Stepchild, Parent, Sibling | Family-based petitions |
| `tax` | Spouse, Biological/Adopted/Step/Foster child, Sibling, Parent | Tax dependents |

## Conditional fields

Fields appear automatically based on the selected relationship category:

| Category | Relationship types | Fields shown |
|----------|-------------------|--------------|
| Spousal | Spouse, Former spouse, Domestic partner | Marriage date, divorce date (if `show-divorce-date`) |
| Child | Child, Biological/Adopted/Step/Foster child, Grandchild | Date of birth, adoption date (if `show-adoption-date`) |
| Other | Other | Free-text description |
| None | Parent, Sibling, Executor, Funeral director | No conditional fields |

When the user switches relationship types and the category changes, stale conditional field values are automatically cleared.

## Usage

### Basic

```html
<civ-relationship
  legend="About the dependent"
  name="dependent"
  preset="general"
  required
></civ-relationship>
```

### Survivor with deceased fields

```html
<civ-relationship
  legend="About the person who died"
  name="deceased"
  preset="survivor"
  show-deceased
  required
></civ-relationship>
```

### Immigration with adoption date

```html
<civ-relationship
  legend="About the person you are petitioning for"
  name="beneficiary"
  preset="immigration"
  show-adoption-date
  required
></civ-relationship>
```

### Without name fields

Use `hide-name` when the name is captured in a separate component:

```html
<civ-name legend="Spouse's name" name="spouseName" required></civ-name>
<civ-relationship
  legend="Relationship details"
  name="spouse"
  preset="general"
  hide-name
></civ-relationship>
```

**Heads up: `show-name="false"` does NOT work.** HTML boolean attributes are truthy whenever present, so `show-name="false"` still reads as `true`. Use `hide-name` (no value) to disable from HTML, or bind the property directly in lit-html: `.showName=${false}`.
