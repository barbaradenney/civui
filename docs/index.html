Design Philosophy

Use the Platform

CivUI uses native <select>, <input>, <textarea>, <fieldset>, and <button> elements — not JavaScript re-implementations. Native elements get keyboard navigation, form participation, and screen reader support for free. Custom code is only added for components that don't exist natively.

System Fonts, No Downloads

System font stack that resolves to the platform's native typeface — SF Pro on Apple, Segoe UI on Windows, Roboto on Android. Zero network requests, no flash of unstyled text, no layout shift.

CSS Over JavaScript

Visual states like error borders, disabled opacity, and selection highlights are handled with CSS attribute selectors ([aria-invalid], :disabled, :has(:checked)) — not JavaScript class toggling. Even icons are pure CSS — 44 icons drawn with borders, clip-path, and pseudo-elements. Zero SVGs, zero icon fonts.

Border Radius Signals Interactivity

Only clickable elements get rounded corners — buttons, inputs, tiles. Static containers like alerts and error summaries use sharp edges. This visual convention helps users identify interactive elements at a glance.

Underlines Signal Navigation

Text is only underlined when it navigates somewhere. Buttons styled as links (href prop) get underlines; action buttons do not. This preserves the semantic distinction between navigation and action.

Every Byte Is Earned

44 icons drawn entirely with CSS (no SVGs, no icon fonts), no CSS-in-JS (static Tailwind), no hidden input hacks (ElementInternals), no polyfills (ES2022 target). Lit (~16K gzipped) is the only runtime dependency. The full forms package is 137K minified.

Built for Government Forms

Declarative Validation

Add validate="email" to any text input and it validates on blur — no JavaScript needed. Ships with 15 validators for government patterns: SSN, EIN, phone (US and international), ZIP, US state, email, URL, currency, date, and more. Error messages are localizable via the i18n system.

Accessible Input Masking

Masks format on blur by default — users type freely, then see formatted output like 123-45-6789 when they tab away. No cursor jumping, no unexpected characters for screen readers. Live formatting available as an opt-in for teams that need it (mask-mode="live").

Form Persistence

Add persist="enrollment" to a form and user progress auto-saves to session storage. Long government forms are safe from accidental navigation, browser crashes, and session timeouts. PII fields are automatically excluded from storage.

Cross-Platform Parity

Every component ships on web (Lit), iOS (SwiftUI), and Android (Jetpack Compose) with matching APIs, accessibility, and design tokens. Icons map to SF Symbols and Material Symbols automatically. An automated parity report (pnpm parity) catches drift.

Read-Only Review Mode

Add readonly to any form component and it renders as plain text — perfect for review-before-submit pages. The value is still included in form submission, unlike disabled which excludes it.

Localization Ready

Every user-facing string — error messages, hints, labels, screen reader announcements — flows through a centralized i18n system. Override any string with setLocaleStrings() on web, CivLocale.setStrings() on native. Supports all three platforms.

Packages

Web Components

@civui/tokens

Design tokens in W3C DTCG format. Outputs CSS custom properties, Tailwind preset, JS constants, SwiftUI, and Compose values. Includes dark mode and density scales.

@civui/core

Base classes (CivBaseElement, CivFormElement), accessibility utilities, screen reader announcements, civ-icon component, and shared helpers.

@civui/forms

Form components: text-input, textarea, select, checkbox, radio-group, combobox, date-picker, memorable-date, file-upload, toggle, segmented-control, and more.

@civui/ui

UI components: Button, Link, Link Card, Tag, Card, Page Header, Divider. Primary/secondary/tertiary/danger variants.

@civui/feedback

Feedback components: Alert (info, warning, error, success), Progress Bar.

@civui/navigation

Navigation components: Skip Link, Task List, Task Group, Task. Sequential task-based form navigation.

@civui/content

JSON-based content management for labels, hints, errors, and placeholders across components.

Tooling

@civui/schema

Platform-neutral component schemas — the single source of truth for generating cross-platform implementations.

@civui/codegen

Cross-platform code generator — produces Lit, SwiftUI, and Compose components from shared schemas.

@civui/cli

Developer CLI for generating components, building, testing, validating, and releasing.

@civui/mcp-server

80-tool MCP server: form parsing, generation, validation, testing, 20 VA form definitions, React output, workflow support. Full docs →

@civui/test-utils

Shared test utilities for component testing with Vitest and jsdom.

Native

CivUI for iOS

SwiftUI components for iOS 16+ and macOS 13+. Accessibility-first government design system with native look and feel.

CivUI for Android

Jetpack Compose components for Android 8+. Material 3 based government design system with shared design tokens.

Form Example

<civ-form>
  <civ-text-input label="Full name" name="name" required
    required-message="Enter your full name"></civ-text-input>
  <civ-text-input label="Email address" name="email" type="email" required
    hint="For example: name@agency.gov"></civ-text-input>
  <civ-memorable-date label="Date of birth" name="dob"
    hint="For example: January 15 1990"></civ-memorable-date>
  <civ-select label="State" name="state" .options="${JSON.stringify([
    { value: '', label: 'Select a state' },
    { value: 'CA', label: 'California' },
    { value: 'NY', label: 'New York' },
  ])}"></civ-select>
  <civ-textarea label="Message" name="message"
    hint="Maximum 500 characters" maxlength="500"></civ-textarea>
  <civ-checkbox label="I agree to the terms" name="terms" required></civ-checkbox>
  <button type="submit">Submit</button>
</civ-form>

CLI

# Generate a new component
civui generate component my-widget

# Cross-platform codegen from schema
civui codegen --schema checkbox --targets lit,swiftui,compose

# Build packages
civui build all

# Run tests
civui test --unit

# Validate everything
civui validate

# Convert legacy forms to CivUI
civui convert legacy-form.pdf
