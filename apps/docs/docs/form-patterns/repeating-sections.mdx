# Repeating Sections

Collect variable-length lists of items — dependents, service periods, previous addresses, employment history. CivUI supports three modes depending on item complexity.

## Inline mode (simple items)

Best for 1-3 fields per item. Fields are visible on the page with add/remove buttons.

<StoryEmbed id="forms-form-repeater--default" />

```html
<civ-repeater
  legend="Other phone numbers"
  name="phones"
  item-label="phone number"
  min="0"
  max="5"
>
  <civ-phone name="phone" required></civ-phone>
  <civ-text-input label="Label" name="phoneLabel" hint="For example: Work, Partner"></civ-text-input>
</civ-repeater>
```

## Detail mode (complex items)

Best for 4+ fields. Shows summary cards for existing items. Clicking "Add" or "Edit" opens a focused detail view.

<StoryEmbed id="forms-form-repeater--multiple-fields" />

```html
<civ-repeater
  legend="Dependents"
  name="dependents"
  item-label="dependent"
  mode="form-steps"
  min="0"
  max="20"
>
  <civ-text-input label="First name" name="firstName" required></civ-text-input>
  <civ-text-input label="Last name" name="lastName" required></civ-text-input>
  <civ-memorable-date legend="Date of birth" required name="dob"></civ-memorable-date>
  <civ-ssn name="ssn"></civ-ssn>
</civ-repeater>
```

## Form-steps mode (multi-step items)

For items complex enough to warrant their own multi-step flow. Each item is added through a step-by-step form.

## Min and max constraints

Set `min` and `max` to control the allowed number of items:

<StoryEmbed id="forms-form-repeater--min-max" />

```html
<!-- At least 1 service period, up to 10 -->
<civ-repeater legend="Service periods" name="service" item-label="service period" min="1" max="10">
  ...
</civ-repeater>
```

## Which mode to use

| Item complexity | Fields per item | Mode | Example |
|----------------|----------------|------|---------|
| Simple | 1-3 | `inline` | Phone numbers, email addresses |
| Complex | 4+ or multi-step | `form-steps` | Dependents, employment history, service periods |

## Components used

- [`civ-repeater`](/components/form/repeater) — repeating section container
- Compound components work inside repeaters: `civ-name`, `civ-address`, `civ-service-history`, `civ-relationship`
