# Button Group

<!-- contract-link:start -->

> **Contract reference:** [`civ-button-group`](pathname:///civui/storybook/?path=/docs/contract-civ-button-group--docs) — auto-generated structured contract for cross-platform implementation.

`<civ-button-group>`

Joins adjacent action buttons into a connected toolbar. Removes inner border-radius so buttons sit flush together. Renders with `role="toolbar"` for proper screen reader semantics.

Use a button group when actions are related and equally weighted — like text formatting controls or view switchers. If one action is primary and the others are secondary, place them independently instead.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `orientation` | `'horizontal' \| 'vertical'` | `'horizontal'` | Layout direction of the button group |
| `label` | `string` | `''` | Accessible label for the toolbar (`aria-label`) |

## Accessibility

Button group renders a `<div role="toolbar">` with `aria-orientation` set to match the `orientation` prop. Always provide a `label` so screen readers announce the toolbar's purpose.

```html
<civ-button-group label="Text formatting">
  <civ-action-button label="Bold"></civ-action-button>
  <civ-action-button label="Italic"></civ-action-button>
  <civ-action-button label="Underline"></civ-action-button>
</civ-button-group>
```

## When to use

- **Toolbar actions** — formatting, alignment, view toggles
- **Toggle groups** — use with `pressed` on action buttons for mutually exclusive or multi-select options
- **Pagination controls** — previous/next/page number buttons

## When NOT to use

- **Primary + secondary actions** — use separate buttons with distinct variants instead
- **Navigation** — use links or tabs, not a button group
- **Form submission** — submit and cancel buttons should stand alone

## Vertical orientation

Stack buttons vertically when horizontal space is limited or the group represents a vertical menu.

```html
<civ-button-group orientation="vertical" label="Options">
  <civ-action-button label="Option 1"></civ-action-button>
  <civ-action-button label="Option 2"></civ-action-button>
  <civ-action-button label="Option 3"></civ-action-button>
</civ-button-group>
```
