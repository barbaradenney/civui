# File List

`<civ-file-list>`

Read-only display of uploaded files. Use on review pages, confirmation pages, and summary sections where the user can see what was uploaded but cannot modify the list.

For interactive file upload (add/remove/cancel/retry), use `<civ-file-upload>` instead.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `files` | `FileListItem[]` | `[]` | Array of files to display. Set via JavaScript property (not an HTML attribute). |
| `label` | `string` | `''` | Accessible label for the list (`aria-label` on the `<ul>`). |

## FileListItem Interface

Each item in the `files` array has the following shape:

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `name` | `string` | Yes | File name displayed in the list. |
| `size` | `number` | Yes | File size in bytes. Automatically formatted as B, KB, or MB. |
| `url` | `string` | No | Download or preview URL. When set, the file name renders as a link (`target="_blank"`). |
| `type` | `string` | No | MIME type for icon heuristic (e.g., `'application/pdf'`). |

## Usage

```html
<civ-file-list label="Uploaded documents"></civ-file-list>
```

```js
const fileList = document.querySelector('civ-file-list');
fileList.files = [
  { name: 'DD214.pdf', size: 2400000, url: '/files/dd214.pdf' },
  { name: 'medical-records.pdf', size: 8100000 },
  { name: 'tax-return.pdf', size: 1200000, url: '/files/tax-return.pdf' },
];
```

## Accessibility

- Renders as a `<ul role="list">` so screen readers announce the file count.
- The `label` prop sets `aria-label` on the list for context (e.g., "Uploaded supporting documents").
- File links open in a new tab with `rel="noopener noreferrer"` for security.
- `javascript:` URLs are blocked to prevent XSS.
