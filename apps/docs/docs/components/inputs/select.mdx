# Select

`<civ-select>`

## Shared Props

Props like `label`, `hint`, `error`, `required`, and `disabled` go on the `<civ-form-field>` wrapper. The component itself receives `name`, `value`, and component-specific props.

## Component-Specific Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `options` | `Array<{ value: string, label: string, disabled?: boolean, group?: string }>` | `[]` | Array of options to render |
| `empty-label` | `string` | `'- Select -'` | Label for the empty/default option |
| `preset` | `string` | ‚Äî | Pre-populated option list. See [Presets](#presets) below. |
| `preset-variant` | `string` | ‚Äî | Variant of the preset (e.g., `"territories"`, `"all"`, `"binary"`). |

## Option Groups

Options with a `group` property are rendered inside `<optgroup>` elements. Ungrouped options appear before any groups.

## Events

## Presets

The `preset` prop pre-populates the select with a standard option list. Use `preset-variant` to choose a variant of that list.

| Preset | Variants | Description |
|--------|----------|-------------|
| `us-state` | (default), `territories` | US states + DC. `territories` adds PR, GU, VI, AS, MP. |
| `service-branch` | (default), `all` | Active-duty branches. `all` adds reserve, guard, and historical. |
| `discharge-type` | ‚Äî | Military discharge types |
| `suffix` | ‚Äî | Name suffixes (Jr., Sr., II, III, etc.) |
| `relationship-type` | (default), `dependent` | Relationship to veteran |
| `marital-status` | ‚Äî | Marital status options |
| `ethnicity` | ‚Äî | OMB ethnicity categories |
| `gender` | (default), `binary` | Gender identity. `binary` limits to Male/Female. |
| `language` | ‚Äî | Preferred language (Census Bureau frequency) |
| `housing-status` | ‚Äî | Housing status (HUD/Census categories) |
| `education-level` | ‚Äî | Educational attainment (Census ACS / ISCED) |
| `employment-status` | ‚Äî | Employment status (BLS CPS categories) |
| `income-source` | ‚Äî | Income source (SSA/HUD benefits patterns) |
| `veteran-status` | ‚Äî | Veteran preference status (OPM 5 USC ¬ß 2108) |
| `disability-type` | ‚Äî | Disability categories (ADA / DOL CC-305) |
| `citizenship-status` | ‚Äî | Citizenship/work authorization (USCIS I-9) |
| `pay-frequency` | ‚Äî | Pay frequency (DOL FLSA / IRS Pub 15) |
| `contact-preference` | ‚Äî | Preferred contact method (USDS/18F patterns) |

### Customizing Presets

Presets are a starting point ‚Äî you can modify them or provide your own options entirely.

**Override with your own options** (ignore presets):

```html
<civ-form-field label="Status">
  <civ-select name="status">
    <option value="active">Active</option>
    <option value="inactive">Inactive</option>
  </civ-select>
</civ-form-field>
```

**Start from a preset and modify in JavaScript:**

```js
import { resolvePresetOptions } from '@civui/core';

// Get the base preset options
const options = resolvePresetOptions('education-level');

// Add a custom option
options.push({ value: 'trade-certificate', label: 'Trade certificate' });

// Remove an option
const filtered = options.filter(o => o.value !== 'doctoral');

// Apply to the select
document.querySelector('civ-select').options = filtered;
```

The `resolvePresetOptions(preset, variant?)` function is exported from `@civui/core` and returns a plain array of `{ value, label }` objects that you can modify freely.

> **Tip:** Presets also work on `<civ-radio-group>` and `<civ-checkbox-group>` ‚Äî use the same `preset` and `preset-variant` attributes. This is useful when you want to present a shorter preset list as radio buttons or checkboxes instead of a dropdown.

### Data Sources

All preset data is sourced from federal standards and regulations. Each preset includes a source citation in the source code. Key references:

- **OMB Directive 15** ‚Äî Race and ethnicity categories
- **Census Bureau ACS** ‚Äî Marital status, education, language
- **BLS Current Population Survey** ‚Äî Employment status
- **OPM 5 USC ¬ß 2108** ‚Äî Veteran preference categories
- **ADA / DOL CC-305** ‚Äî Disability categories
- **USCIS I-9** ‚Äî Citizenship/work authorization
- **10 USC ¬ß 101** ‚Äî Military service branches

## Usage

```html
<!-- Basic select with slotted options -->
<civ-form-field label="Branch of service" required>
  <civ-select name="branch" required>
    <option value="army">Army</option>
    <option value="navy">Navy</option>
    <option value="airforce">Air Force</option>
    <option value="marines">Marine Corps</option>
    <option value="coastguard">Coast Guard</option>
    <option value="spaceforce">Space Force</option>
  </civ-select>
</civ-form-field>

<!-- With grouped options -->
<civ-form-field label="Document type" hint="Choose a document type">
  <civ-select name="docType">
    <optgroup label="Military">
      <option value="dd214">DD214</option>
      <option value="dd215">DD215</option>
    </optgroup>
    <optgroup label="Identity">
      <option value="passport">Passport</option>
      <option value="license">Driver's license</option>
    </optgroup>
  </civ-select>
</civ-form-field>

<!-- Using a preset -->
<civ-form-field label="State" required>
  <civ-select name="state" preset="us-state" required></civ-select>
</civ-form-field>
```
