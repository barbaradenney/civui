# Input Group

`<civ-input-group>`

Joins a form input and action button(s) flush together into a single visual control. The container removes inner border-radius so children render seamlessly connected.

Works with any CivUI input (`civ-text-input`, `civ-select`, `civ-combobox`) and action buttons on either side.

## Props

Input group is a layout-only container with no props. It applies CSS to remove inner border-radius between its children.

## Usage

### Search field

The most common pattern. A text input with an adjacent search button.

```html
<civ-input-group>
  <civ-text-input label="Search" name="search"></civ-text-input>
  <civ-action-button label="Search" emphasis="primary"></civ-action-button>
</civ-input-group>
```

### Currency prefix

Use a tertiary button on the left as a visual prefix.

```html
<civ-input-group>
  <civ-action-button label="$" emphasis="tertiary"></civ-action-button>
  <civ-text-input label="Amount" name="amount"></civ-text-input>
</civ-input-group>
```

### Quantity stepper

Buttons on both sides for increment/decrement controls.

```html
<civ-input-group>
  <civ-action-button label="-" emphasis="tertiary"></civ-action-button>
  <civ-text-input label="Quantity" name="qty" value="1" width="xs"></civ-text-input>
  <civ-action-button label="+" emphasis="tertiary"></civ-action-button>
</civ-input-group>
```

### Compact (dense surfaces)

`civ-input-group` is a pass-through layout container with no chrome of its own — no `spacing` prop. For a compact group, set `spacing="sm"` on **each child** (both the input and the button). The group's per-child border-radius math handles either density correctly.

```html
<civ-input-group>
  <civ-text-input label="Search" name="search" spacing="sm"></civ-text-input>
  <civ-action-button label="Search" emphasis="primary" spacing="sm"></civ-action-button>
</civ-input-group>
```

## When to use

- **Search bars:** input paired with a search or filter button
- **Prefix/suffix indicators:** currency symbols, units, or lookup actions
- **Stepper controls:** increment/decrement buttons flanking a numeric input

## When NOT to use

- **Standalone inputs:** if the input doesn't need an attached button, don't wrap it
- **Multiple inputs:** input group is for one input with button(s), not for grouping multiple inputs together
- **Decorative icons:** use the `icon-start` or `icon-end` props on the input itself instead
