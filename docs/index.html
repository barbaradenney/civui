CivUI

Accessibility-first web components for modern government applications. Built with Lit, Light DOM, and ElementInternals.

Light DOM WCAG 2.1 AA ElementInternals Web + iOS + Android Schema-Driven Codegen Dark Mode
Web Components Storybook GitHub

Quick Start

npm install @civui/forms

# Use in HTML
<script type="module">
  import '@civui/forms';
</script>

<civ-text-input
  label="Email address"
  name="email"
  type="email"
  required
></civ-text-input>

Why CivUI?

Accessibility First

Light DOM means external labels, ARIA IDREFs, and focus management work without Shadow DOM workarounds. Every component is WCAG 2.1 AA compliant with Section 508 standards.

Learn more →

Cross-Platform Native

Web components (Lit) for browsers plus native SwiftUI (iOS 16+) and Jetpack Compose (Android 8+) companions — all generated from shared component schemas.

Learn more →

Density Scale

Three density modes — dense, default, and spacious — with fluid responsive typography via clamp(). Adapts to any screen size and user preference.

Zero-Dependency Icons

Icons built from layered Unicode characters — no icon fonts, SVG sprites, or CDN needed. Inherit color, scale with font-size, zero network requests.

MCP Server

AI-powered form modernization — 83 tools, 10 resources, and 8 guided prompts for converting legacy government forms into accessible CivUI components.

Learn more →

Design Tokens

W3C DTCG format tokens as the single source of truth. Auto-generated CSS custom properties, Tailwind preset, JS constants, SwiftUI, and Compose values.

Native Form Participation

ElementInternals enables custom elements to participate in native HTML forms. No hidden inputs or JavaScript hacks — works with FormData and validation APIs.

Packages

Web Components

@civui/tokens

Design tokens in W3C DTCG format. Outputs CSS custom properties, Tailwind preset, JS constants, SwiftUI, and Compose values. Includes dark mode and density scales.

@civui/core

Base classes (CivBaseElement, CivFormElement), accessibility utilities, screen reader announcements, civ-icon component, and shared helpers.

@civui/forms

Form components: text-input, textarea, select, checkbox, radio-group, combobox, date-picker, memorable-date, file-upload, toggle, segmented-control, and more.

@civui/ui

General-purpose UI components: Button, Link, Badge, and more.

@civui/feedback

Feedback components: Alert, Banner, Modal, and more.

@civui/navigation

Navigation components: Skip Link, Breadcrumb, Tabs, and more.

@civui/content

JSON-based content management for labels, hints, errors, and placeholders across components.

Tooling

@civui/schema

Platform-neutral component schemas — the single source of truth for generating cross-platform implementations.

@civui/codegen

Cross-platform code generator — produces Lit, SwiftUI, and Compose components from shared schemas.

@civui/cli

Developer CLI for generating components, building, testing, validating, and releasing.

@civui/mcp-server

MCP server for converting legacy PDF/HTML forms into accessible CivUI web components.

@civui/test-utils

Shared test utilities for component testing with Vitest and jsdom.

Native

CivUI for iOS

SwiftUI components for iOS 16+ and macOS 13+. Accessibility-first government design system with native look and feel.

CivUI for Android

Jetpack Compose components for Android 8+. Material 3 based government design system with shared design tokens.

Form Example

<civ-form>
  <civ-text-input label="Full name" name="name" required
    required-message="Enter your full name"></civ-text-input>
  <civ-text-input label="Email address" name="email" type="email" required
    hint="For example: name@agency.gov"></civ-text-input>
  <civ-memorable-date label="Date of birth" name="dob"
    hint="For example: January 15 1990"></civ-memorable-date>
  <civ-select label="State" name="state" .options="${JSON.stringify([
    { value: '', label: 'Select a state' },
    { value: 'CA', label: 'California' },
    { value: 'NY', label: 'New York' },
  ])}"></civ-select>
  <civ-textarea label="Message" name="message"
    hint="Maximum 500 characters" maxlength="500"></civ-textarea>
  <civ-checkbox label="I agree to the terms" name="terms" required></civ-checkbox>
  <button type="submit">Submit</button>
</civ-form>

CLI

# Generate a new component
civui generate component my-widget

# Cross-platform codegen from schema
civui codegen --schema checkbox --targets lit,swiftui,compose

# Build packages
civui build all

# Run tests
civui test --unit

# Validate everything
civui validate

# Convert legacy forms to CivUI
civui convert legacy-form.pdf
