`<civ-time-picker>`

Self-contained time input. Always stores its value in 24-hour ISO format
(`HH:MM`) regardless of display format or input mode — the wire format
never changes.

For known past dates use [`<civ-memorable-date>`](./date.mdx); for future
dates use [`<civ-date-picker>`](./date.mdx).

## Two input modes

The component picks between two interaction patterns via the `mode` prop:

### `mode="combo"` (default, recommended) — USWDS pattern

A single typeable combobox with pre-built time slots driven by
`minute-step`. Users can type to filter (`"230"` → "2:30 AM"/"2:30 PM"),
or pick from the dropdown. Optional `min` / `max` bounds restrict the
slot list to business hours. Best for **scheduling** — appointments,
hearing times, callback windows.

Matches the [USWDS time picker](https://designsystem.digital.gov/components/time-picker/) pattern.
Composes the existing [`<civ-combobox>`](./combobox.mdx).

### `mode="select"` — three dropdowns

Three selects (hour, minute, AM/PM) inside a fieldset. Predictable,
every-device-safe, no typing required. Use when slot-list filtering
would be confusing — capturing an arbitrary incident time, or
`minute-step="1"` precision that would create a 1,440-slot combobox.

## When to use which mode

| Scenario | Mode | Why |
| --- | --- | --- |
| Schedule an appointment, hearing, callback | **combo** | Pre-built slots match the mental model ("pick a time slot"); typing skips to a slot fast |
| Pick from business-hours offering | **combo** | `min` / `max` keep the slot list short |
| Record an arbitrary incident time | **select** | No fixed slots — user types/picks any minute |
| Need minute-level precision (`minute-step="1"`) | **select** | A 1,440-slot combobox is unwieldy |
| Form will be filled on devices with poor IME support | **select** | No typing needed |

## Value format

The `value` is always a 24-hour ISO time string (`HH:MM`), regardless of
the `format` prop. A server-rendered `value="14:30"` displays as "2:30 PM"
in 12-hour mode and "14:30" in 24-hour mode — the wire format never
changes.

When the user hasn't fully picked a time, `value` is `""`. In 12-hour
select mode the AM/PM select must be set for the value to assemble; in
24-hour mode only hour + minute are required. In combo mode, the value is
either an exact match for a slot or empty.

## Usage

```html
<!-- Schedule an appointment (default combo, 15-min slots, business hours) -->
<civ-time-picker
  label="Appointment time"
  name="appt_time"
  min="09:00"
  max="17:00"
  minute-step="30"
  required
></civ-time-picker>

<!-- 24-hour format (military / international scheduling) -->
<civ-time-picker
  label="Hearing time"
  name="hearing"
  format="24"
  min="08:00"
  max="18:00"
></civ-time-picker>

<!-- Pre-populated from server data -->
<civ-time-picker
  label="Departure time"
  name="departure"
  value="14:30"
></civ-time-picker>

<!-- Free-form precision: three dropdowns + minute-step=1 -->
<civ-time-picker
  mode="select"
  legend="Exact time of incident"
  name="incident_time"
  minute-step="1"
></civ-time-picker>
```

## Accessibility

### Combo mode
- Single `<input role="combobox">` with an associated listbox. Arrow keys
  navigate, Enter selects, Escape closes. Typing filters the slot list by
  substring match.
- The slot list is bounded — typing an unmatched value clears the
  selection rather than committing arbitrary text. This means users can't
  type "14:37" and have it accepted unless that slot exists; pick
  `minute-step` accordingly, or fall back to select mode for free-form
  precision.

### Select mode
- Real `<fieldset>` with a `<legend>`. If no `legend` is set, a
  visually-hidden "Time" fallback ensures the fieldset stays Section 508
  compliant.
- Each sub-field is a real `<civ-select>` with its own labelled
  `<select>` element. Keyboard navigation between hour/minute/period uses
  native tab order.
- `aria-describedby` chains the legend → hint → error so screen readers
  announce them in order on focus.

## Localization

All user-facing strings come from the locale registry:

- `timePickerDefaultLegend` (default `"Time"`, used when no `legend` /
  `label` is set)
- `timePickerPlaceholder` (default `"e.g. 9:00 AM"`, combo-mode input
  placeholder)
- `timePickerHourLabel` (default `"Hour"`, select mode)
- `timePickerMinuteLabel` (default `"Minute"`, select mode)
- `timePickerPeriodLabel` (default `"AM/PM"`, select mode)
- `timePickerAm` / `timePickerPm` (defaults `"AM"` / `"PM"`)

Override via `setLocale()` or per-component with the `placeholder`,
`hour-label`, `minute-label`, `period-label` props.
