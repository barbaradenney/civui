`<civ-link>`

An accessible link component that always renders an `<a>` element. For a navigation affordance that should look like a button, use `<civ-button href="…">`. That renders an `<a>` with button chrome plus an underline so the link identity stays visible. All link variants are underlined for accessibility.

## Variants

### Primary

Bold underlined link with a trailing chevron-right caret. Most emphasis. Use for the main next-step link in a section.

```html
<civ-link href="/benefits" variant="primary" label="View all benefits"></civ-link>
```

### Secondary (default)

Plain underlined link. Use for inline navigation that doesn't need extra emphasis.

```html
<civ-link href="/privacy" label="Privacy policy"></civ-link>
```

### Back

Back-navigation link with a leading chevron-left icon.

```html
<civ-link href="/previous-step" variant="back" label="Go back"></civ-link>
```

## Danger modifier

Add `danger` to any variant for destructive action styling.

```html
<civ-link href="/delete" variant="primary" danger label="Remove application"></civ-link>
```

## Disabled state

```html
<civ-link href="/locked" variant="primary" disabled label="Locked section"></civ-link>
```

## New Tab

Use `<civ-link new-tab>` to open links in a new tab. This automatically sets `target="_blank"`, `rel="noopener noreferrer"`, appends an external-link icon, and adds screen-reader-only "(opens in new tab)" text. Works for both external and internal links.

```html
<!-- External site -->
<civ-link
  href="https://www.ssa.gov"
  label="Social Security Administration"
  new-tab
></civ-link>

<!-- Internal link in new tab (e.g., keep form state) -->
<civ-link
  href="/forms/va-10-10ez.pdf"
  label="View form (PDF)"
  new-tab
></civ-link>
```
