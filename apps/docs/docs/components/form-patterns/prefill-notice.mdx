# Prefill Notice

`<civ-prefill-notice>`

An informational banner for chapter review pages explaining that data was prefilled from a user profile. Used in the prefill chapter flow pattern to set expectations before the user reviews their data.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `heading` | `string` | i18n default | Custom heading text |
| `body` | `string` | i18n default | Custom body text |
| `profile-href` | `string` | `''` | URL for the "update profile" link (omit to hide link) |
| `link-text` | `string` | i18n default | Custom link text |

## Usage

```html
<!-- Basic usage -->
<civ-prefill-notice profile-href="/profile"></civ-prefill-notice>

<!-- With custom text -->
<civ-prefill-notice
  heading="We've filled in some information"
  body="This data comes from your VA.gov profile. If it's wrong, update your profile."
  profile-href="/profile/settings"
  link-text="Go to your profile"
></civ-prefill-notice>
```

## When to use

- On chapter review pages in multi-step forms where data has been prefilled from a user profile, API, or saved state.
- Always pair with `<civ-summary>` to show the prefilled data with edit links.
- See the [Prefill](./prefill) page for the full chapter flow pattern.
