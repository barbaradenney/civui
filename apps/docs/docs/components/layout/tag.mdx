`<civ-tag>`

A small label rendered as a colored pill for **categorization**. Topics, taxonomies, filter chips, metadata. For status indicators ("Approved", "Pending") or counts, use [`civ-badge`](../feedback/badge) instead.

## Styles

### Primary (high emphasis)

Bold/dark background with light text. Use for prominent category labels.

```html
<civ-tag label="Healthcare" color="blue" tag-style="primary"></civ-tag>
<civ-tag label="Disability" color="purple" tag-style="primary"></civ-tag>
```

### Secondary (low emphasis, default)

Light background with dark text. Use for filter chips and metadata.

```html
<civ-tag label="Healthcare" color="blue"></civ-tag>
<civ-tag label="Education" color="orange"></civ-tag>
```

## Color variants

| Variant | Typical use |
|---------|-------------|
| `blue` | Primary brand category |
| `orange` | Secondary category, accent |
| `purple` | Specialized program category |
| `gray` | Archived, inactive, neutral metadata |

Semantic colors (red/green/yellow/teal) are intentionally excluded. They're reserved for [`civ-badge`](../feedback/badge), which carries `role="status"` for assistive tech.

## Dark mode

Tag colors are token-based and automatically adjust in dark mode via `prefers-color-scheme`. Primary style tags use darker backgrounds with lighter text; secondary style tags use lighter backgrounds with darker text.

## Density

Tag size is controlled by the density system. Wrap in a parent with `data-civ-scale="dense"` or `data-civ-scale="spacious"` to adjust sizing.

```html
<div data-civ-scale="dense">
  <civ-tag label="Healthcare" color="blue"></civ-tag>
</div>
```

## Usage

```html
<civ-tag label="Healthcare" color="blue"></civ-tag>
<civ-tag label="Education" color="orange"></civ-tag>
<civ-tag label="Disability" color="purple" tag-style="primary"></civ-tag>
<civ-tag label="Archived" color="gray"></civ-tag>
```
