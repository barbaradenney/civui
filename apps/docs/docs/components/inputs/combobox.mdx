# Combobox

`<civ-combobox>` `<civ-country>`

Accessible combobox (autocomplete) with text input and filterable dropdown listbox. Implements the ARIA combobox pattern with full keyboard navigation (Arrow keys, Enter, Escape, Home, End). Includes a [country preset](#country-preset) for government forms.

Use combobox when the option list is long (more than 15 items) or when users benefit from type-ahead filtering. For shorter lists, use `civ-select`.

### Chevron Toggle

A decorative chevron on the trailing edge opens the full unfiltered list — same affordance as a native `<select>`. Click clears any in-progress filter and shows every option. Keyboard users still use `ArrowDown` / `ArrowUp` to open (the chevron is `aria-hidden` and `tabindex="-1"`).

<StoryEmbed id="forms-inputs-combobox--chevron-toggle" minHeight={350} />

### Async: remote loadOptions

Set `loadOptions` to a `(query: string) => Promise<ComboboxOption[]>` and the combobox switches into remote mode. Calls are debounced (300ms by default), stale responses are discarded automatically, and the dropdown surfaces dedicated loading / error / below-min-query states.

## Shared Props

Set `label`, `hint`, `error`, `required`, and `disabled` directly on the component along with `name`, `value`, and component-specific props. The component renders its own label / hint / error chrome.

## Component-Specific Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `options` | `Array<{ value: string, label: string, group?: string }>` | `[]` | Available options. Ignored when `loadOptions` is set. |
| `placeholder` | `string` | `''` | Placeholder text for the input |
| `no-results-text` | `string` | `'No results found'` | Message shown when filter matches nothing |
| `loadOptions` | `(query: string) => Promise<ComboboxOption[]>` | `null` | Async loader. When set, switches the combobox into remote mode. |
| `load-debounce` | `number` | `300` | Debounce delay (ms) before `loadOptions` is called after the user types. |
| `min-query-length` | `number` | `0` | Minimum query length before `loadOptions` runs. Below the threshold the dropdown shows a "type at least N characters" prompt. |
| `loading-text` | `string` | `'Loading…'` | Override the in-flight message. |
| `loading-error-text` | `string` | _(error.message or `'Could not load results. Try again.'`)_ | Override the error message shown when `loadOptions` rejects. |

## Typeahead Filtering

As the user types, options are filtered by substring match (case-insensitive). Matching text is highlighted in the dropdown with a `<mark>` element.

## Keyboard Navigation

| Key | Action |
|-----|--------|
| Arrow Down | Open dropdown / move to next option |
| Arrow Up | Open dropdown / move to previous option |
| Home | Move to first option |
| End | Move to last option |
| Enter | Select highlighted option |
| Escape | Close dropdown, restore previous selection |
| Tab | Close dropdown |

## Events

| Event | Detail | Description |
|-------|--------|-------------|
| `civ-input` | `{ value: string }` | Fires when filter text changes |
| `civ-change` | `{ value: string }` | Fires when an option is selected |

## Country Preset

`<civ-country>`

Pre-configured combobox for country selection with type-ahead filtering. Lists the United States first by default. Supports filtering by inclusion or exclusion lists.

Default label: "Country"

### Component-Specific Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `us-first` | `boolean` | `true` | List the United States first in the dropdown |
| `include` | `string` | — | Comma-separated list of country codes to include (e.g., `"US,CA,MX"`) |
| `exclude` | `string` | — | Comma-separated list of country codes to exclude |

<StoryEmbed id="forms-inputs-combobox--country" minHeight={350} />

<StoryEmbed id="forms-inputs-combobox--country-of-birth" minHeight={350} />

```html
<!-- Country select with US listed first -->
<civ-country label="Country" name="country" required></civ-country>

<!-- North America only -->
<civ-country label="Country" name="country" include="US,CA,MX" required></civ-country>

<!-- Country of birth with custom label -->
<civ-country label="Country of birth" name="birthCountry" required></civ-country>
```

## Usage

```html
<!-- Basic combobox -->
<civ-combobox label="State or territory" name="state"
    placeholder="Start typing a state name"
    required></civ-combobox>

<script>
  document.querySelector('civ-combobox').options = [
    { value: 'AL', label: 'Alabama' },
    { value: 'AK', label: 'Alaska' },
    // ... all 50 states + territories
  ];
</script>

<!-- With grouped options -->
<civ-combobox label="Medical facility" name="facility"
    placeholder="Search by name or city"></civ-combobox>

<script>
  document.querySelector('[name="facility"]').options = [
    { value: 'va-dc', label: 'Washington DC VAMC', group: 'District of Columbia' },
    { value: 'va-baltimore', label: 'Baltimore VAMC', group: 'Maryland' },
    { value: 'va-richmond', label: 'Richmond VAMC', group: 'Virginia' },
  ];
</script>
```
