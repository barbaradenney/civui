# Badge

`<civ-badge>`

A compact status indicator. Use for state ("Approved", "Pending"), severity, or as a notification dot. Renders `role="status"` so screen readers announce the value.

For **numeric counts** ("5", "99+", or notification overlays like "3 unread"), use [`civ-count`](./count) instead. For categorization (topics, taxonomies), use [`civ-tag`](../layout/tag).

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | `''` | Badge text (or accessible name when `dot` is set) |
| `dot` | `boolean` | `false` | Render as a small colored dot only |
| `variant` | `'info' \| 'warning' \| 'error' \| 'success' \| 'neutral'` | `'neutral'` | Semantic color |
| `badge-style` | `'primary' \| 'secondary'` | `'secondary'` | Emphasis: primary (filled dark bg, light text) or secondary (light tint, dark text) |
| `spacing` | `'default' \| 'sm'` | `'default'` | Compact variant with smaller padding |
| `overlay` | `boolean` | `false` | Position absolutely in the top-end corner of a relatively-positioned parent (notification mode) |
| `with-icon` | `boolean` | `false` | Auto-render the variant's semantic icon at the start (success → check-circle, warning → warning, error → error, info → info; neutral has no auto-icon) |
| `icon-start` | `string` | `''` | Explicit leading icon name. Overrides `with-icon`'s variant default. |
| `icon-end` | `string` | `''` | Explicit trailing icon name. |

## Modes

### Status text

```html
<civ-badge label="Approved" variant="success"></civ-badge>
<civ-badge label="Pending" variant="warning"></civ-badge>
<civ-badge label="Denied" variant="error"></civ-badge>
```

### Dot

```html
<civ-badge dot label="Unread messages" variant="error"></civ-badge>
<!-- label becomes aria-label, no visible text -->
```

## Accessibility

- Text badges and dots-with-label render `role="status"` so screen readers announce the badge value.
- In **dot mode**, `label` becomes the `aria-label`. **If `label` is omitted, the dot is marked `aria-hidden="true"`** (no empty live region) — always set `label` when the dot conveys meaning that isn't already present in a parent's accessible name.
- Restricted to semantic colors so visual meaning matches the announced value.

### Notification overlay

Wrap a target (icon, button) with `civ-badge-anchor` to anchor the badge:

```html
<button class="civ-badge-anchor">
  <civ-icon name="mail" size="lg"></civ-icon>
  <civ-badge overlay dot label="New mail" variant="error"></civ-badge>
</button>
```

The `civ-badge-anchor` utility class sets `position: relative; display: inline-block`. Any other relatively-positioned parent works too. For numeric overlays ("3 unread"), use [`civ-count` with `overlay`](./count#overlay) instead.

## Component selection

| Use case | Component |
|----------|-----------|
| Status pill ("Approved", "Pending", "Error") | `civ-badge` |
| Notification dot | `civ-badge` (with `dot`) |
| Numeric count (`5`, `99+`) | [`civ-count`](./count) |
| Notification count over an icon | [`civ-count`](./count) (with `overlay`) |
| Categorization ("Healthcare", "Disability") | [`civ-tag`](../layout/tag) |
| Interactive filter chip | [`civ-filter-chip`](../actions/filter-chip) |
