# Filter Chip

`<civ-filter-chip>` · `<civ-filter-chip-group>`

An interactive, button-like control for filter selection. Click to toggle the chip's `selected` state; in `removable` mode, click the trailing `×` to dismiss without toggling. Use horizontal rows of chips to represent active or available filters.

For static categorization labels use [`civ-tag`](../layout/tag); for status indicators use [`civ-badge`](../feedback/badge); for primary CTAs use [`civ-button`](./button).

When you have more than two or three chips, wrap them in `<civ-filter-chip-group>` to get keyboard navigation and (optionally) single-select coordination — see [the group section](#group) below.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | `''` | Chip text content |
| `value` | `string` | `''` | Filter identifier; passed in event detail |
| `selected` | `boolean` | `false` | Active state (reflected attribute) |
| `removable` | `boolean` | `false` | Renders trailing `×` dismiss button |
| `disabled` | `boolean` | `false` | Disabled state (reflected attribute) |
| `chip-style` | `'primary' \| 'secondary'` | `'secondary'` | Selected-state emphasis: primary (filled) or secondary (light tint) |
| `spacing` | `'default' \| 'sm'` | `'default'` | Compact variant with smaller padding |
| `icon-start` | `string` | `''` | Leading icon name (replaced by check on selection) |
| `icon-end` | `string` | `''` | Trailing icon (kept on selection — useful for chevrons or open/close hints) |
| `count` | `number \| null` | `null` | Match count rendered after the label via [`civ-count`](../feedback/count). Slightly de-emphasised so the label stays primary. |
| `chip-role` | `'toggle' \| 'radio'` | `'toggle'` | ARIA mode. `civ-filter-chip-group` sets `radio` automatically in single-select mode. |

## Events

| Event | Detail | Fires when |
|-------|--------|------------|
| `civ-change` | `{ value, selected }` | The chip body is clicked, toggling selection |
| `civ-remove` | `{ value }` | The `×` is clicked or activated (removable mode only) |
| `civ-analytics` | `{ action }` | On any click — `change` (body toggle) or `remove` |

## Accessibility

- The chip renders a non-interactive `<span role="presentation">` wrapper containing one or two real `<button>` elements (toggle + optional remove). This avoids the "interactive content nested inside a `<button>`" anti-pattern.
- In default `chip-role="toggle"` mode the action button uses `aria-pressed`. In `chip-role="radio"` (set by a single-select group) it uses `role="radio"` + `aria-checked`. Re-clicking an already-selected radio is a no-op (matches platform radio semantics).
- The remove `×` is its own real `<button>` with `aria-label="Remove {label} filter"` — Enter and Space activate it natively. Clicking it does not toggle the chip.
- Touch targets meet WCAG 2.5.8 (24×24 minimum) in both default and `--sm` sizes.
- `disabled` propagates to both inner buttons.

## Patterns

### Faceted search results

Wire `civ-change` to your filter store; let the chip own its `selected` state and observe the event:

```html
<civ-filter-chip label="Healthcare" value="healthcare"></civ-filter-chip>
<civ-filter-chip label="Education" value="education"></civ-filter-chip>
<civ-filter-chip label="Housing" value="housing"></civ-filter-chip>

<script>
  document.addEventListener('civ-change', (e) => {
    const { value, selected } = e.detail;
    console.log(value, selected ? 'applied' : 'removed');
  });
</script>
```

### Applied-filters bar

Use `removable` to show currently-active filters that the user can dismiss:

```html
<civ-filter-chip label="Healthcare" value="healthcare" selected removable></civ-filter-chip>
<civ-filter-chip label="VA Form 21-526EZ" value="form-21-526ez" selected removable></civ-filter-chip>
```

## Anti-patterns

- **Don't use a filter chip as a category label.** If it doesn't filter anything, use `civ-tag` (a static `<span>`).
- **Don't use a filter chip for status.** Use `civ-badge`, which carries `role="status"` for assistive tech.
- **Don't use a filter chip as a primary action.** Use `civ-button` for "Submit", "Save", etc.

## Group

`<civ-filter-chip-group>` coordinates a row of chip children. Adds **roving tabindex** so the row is one Tab stop with arrow-key navigation, enforces single-select if `mode="single"`, and emits an aggregated `civ-change` carrying the full selected value(s).

### Group props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `mode` | `'single' \| 'multi'` | `'multi'` | `single` enforces radio-like behavior (one chip selected); `multi` lets each chip toggle independently. The wrapper role changes accordingly: `radiogroup` for single, `toolbar` for multi. |
| `label` | `string` | `''` | Accessible label, applied as `aria-label` on the wrapper |

### Group events

| Event | Detail | Fires when |
|-------|--------|------------|
| `civ-change` | `{ value: string \| string[], aggregated: true }` | Any child chip toggles. `value` is a single string in single mode, an array in multi mode. The per-chip `civ-change` is stopped at the group, so listeners attached to the group only receive the aggregate. |

### Group properties

| Property | Type | Description |
|----------|------|-------------|
| `value` | `string \| string[]` | Currently selected value(s). Single mode returns a string (or `''` if none); multi mode returns an array. **Settable** — assigning updates each child chip's `selected` flag to match. |

### Keyboard

| Key | Action |
|-----|--------|
| `Tab` | Move into / out of the group (one stop, lands on the first selected chip or first chip) |
| `→` / `↓` | Focus next chip |
| `←` / `↑` | Focus previous chip (wraps at the start) |
| `Home` | Focus first chip |
| `End` | Focus last chip |
| `Enter` / `Space` | Toggle the focused chip (handled by the chip itself) |

Disabled chips are skipped during arrow-key navigation.

### Group patterns

#### Faceted search with multi-select

```html
<civ-filter-chip-group mode="multi" label="Filter results">
  <civ-filter-chip label="Healthcare" value="health" count="24"></civ-filter-chip>
  <civ-filter-chip label="Education" value="education" count="12"></civ-filter-chip>
  <civ-filter-chip label="Housing" value="housing" count="8"></civ-filter-chip>
</civ-filter-chip-group>

<script>
  document.querySelector('civ-filter-chip-group').addEventListener('civ-change', (e) => {
    console.log('selected:', e.detail.value); // ['health', 'housing']
  });
</script>
```

#### View mode picker (single-select)

```html
<civ-filter-chip-group mode="single" label="Sort">
  <civ-filter-chip label="Newest" value="newest" selected></civ-filter-chip>
  <civ-filter-chip label="Oldest" value="oldest"></civ-filter-chip>
  <civ-filter-chip label="A → Z" value="alpha"></civ-filter-chip>
</civ-filter-chip-group>
```
