`<civ-image>`

General-purpose accessible image. Distinct from [`civ-image-preview`](./image-preview), which is upload chrome with a filename + filesize caption next to the image. Use `civ-image` for content imagery â€” marketing visuals, illustrations, profile photos, avatar thumbnails â€” anything that isn't a file-upload preview.

The defaults bake in what gov sites typically miss:

- **`loading="lazy"`** â€” defers off-screen images.
- **`decoding="async"`** â€” never blocks render.
- **CSS `aspect-ratio` reserved** from `ratio` or `width`/`height` so the box has shape before the bytes arrive â€” no Cumulative Layout Shift.
- **`object-fit: cover`** by default so the image fills its box without stretching.
- **Required `alt`** â€” the component dev-warns if you omit the attribute.

### Decorative images

For purely decorative imagery, pass `alt=""` (empty string). This is the W3C-documented opt-out â€” screen readers skip the image entirely. **Never** omit the `alt` attribute; the component will dev-warn and screen readers fall back to reading the filename, which on .gov uploads is often opaque ("DSC_4827_final_v3.jpg").

### Thumbnails

The `thumbnail` variant is a fixed-pixel-size image on Primer's stepped scale (32 / 48 / 64 / 96 / 128 px). Use for inline avatars or row thumbnails. Defaults to `64` if `size` is omitted.

## Accessibility

- **Alt is required.** WCAG 1.1.1 / Section 508 Â§1194.22(a). The component dev-warns if `alt` is undefined. Pass `alt=""` (empty string) for decorative images â€” that's the documented opt-out, never omit.
- **Don't auto-generate alt from filenames.** Many image components fall back to the `src` basename; we deliberately don't, because government uploads frequently have opaque names that are worse than nothing.
- **Aspect-ratio reservation prevents CLS.** Set `ratio="16:9"` (or pass intrinsic `width` and `height` in px) so the box has shape before the image bytes arrive. The component dev-warns if neither is set on the content variant.
- **For informative images, alt should describe meaning, not appearance.** "A veteran fills out their application" â€” not "A man holding a clipboard."
- **For complex images** (charts, diagrams), put the long description in adjacent prose or use `aria-describedby` to point at it. Don't stuff a paragraph into `alt`.
- **For image-links** (the image is the only content of an `<a>`), alt must describe the destination, not the picture.
