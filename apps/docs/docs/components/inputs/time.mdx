`<civ-time-picker>`

Self-contained time input. Always stores its value in 24-hour ISO format
(`HH:MM`) regardless of display format or input mode — the wire format
never changes.

For known past dates use [`<civ-memorable-date>`](./date.mdx); for future
dates use [`<civ-date-picker>`](./date.mdx).

## Three input modes

The component picks between three interaction patterns via the `mode` prop:

### `mode="combo"` (default, recommended) — USWDS pattern

A single typeable combobox with pre-built time slots driven by
`minute-step`. Users can type to filter (`"230"` → "2:30 AM"/"2:30 PM"),
or pick from the dropdown. Optional `min` / `max` bounds restrict the
slot list to business hours. Best for **scheduling** — appointments,
hearing times, callback windows.

Matches the [USWDS time picker](https://designsystem.digital.gov/components/time-picker/) pattern.
Composes the existing [`<civ-combobox>`](./combobox.mdx). When the user
types a time that isn't on the slot grid (e.g. "9:27" with 15-min slots),
the dropdown shows the nearest slot ("9:30 AM") instead of "No results."

### `mode="select"` — selects + AM/PM segmented control

Hour + minute selects plus an AM/PM segmented control. Predictable
picking for discrete time choices; no typing required.

### `mode="text"` — masked text input + AM/PM segmented control

A single text input (`HH:MM` with auto-formatting mask) plus the same
AM/PM segmented control. Best for **arbitrary precision** — incident
reports, medical event times, "exactly when did this happen" prompts
where the user knows the time and a slot grid would slow them down.
`minute-step` is ignored in this mode (any minute 00–59 is valid).

## When to use which mode

| Scenario | Mode | Why |
| --- | --- | --- |
| Schedule an appointment, hearing, callback | **combo** | Pre-built slots match the mental model ("pick a time slot"); typing skips to a slot fast |
| Pick from a business-hours offering | **combo** | `min` / `max` keep the slot list short |
| User knows the exact time (incident, fall, medical event) | **text** | Free-form `HH:MM` input; no slot grid to fight |
| Need any-minute precision (every minute of the day valid) | **text** | A 1,440-slot combobox is unwieldy; selects would scroll forever |
| Discrete picks where typing isn't faster than tapping | **select** | Predictable dropdowns, every-device-safe |
| User-facing self-service in trauma-sensitive flows | **select** | No on-focus dropdown surprise, no typing pressure |

## Value format

The `value` is always a 24-hour ISO time string (`HH:MM`), regardless of
the `format` prop. A server-rendered `value="14:30"` displays as "2:30 PM"
in 12-hour mode and "14:30" in 24-hour mode — the wire format never
changes.

When the user hasn't fully picked a time, `value` is `""`. In 12-hour
select mode the AM/PM select must be set for the value to assemble; in
24-hour mode only hour + minute are required. In combo mode, the value is
either an exact match for a slot or empty.

## Usage

```html
<!-- Schedule an appointment (default combo, 15-min slots, business hours) -->
<civ-time-picker
  label="Appointment time"
  name="appt_time"
  min="09:00"
  max="17:00"
  minute-step="30"
  required
></civ-time-picker>

<!-- 24-hour format (military / international scheduling) -->
<civ-time-picker
  label="Hearing time"
  name="hearing"
  format="24"
  min="08:00"
  max="18:00"
></civ-time-picker>

<!-- Pre-populated from server data -->
<civ-time-picker
  label="Departure time"
  name="departure"
  value="14:30"
></civ-time-picker>

<!-- Free-form precision via text input + AM/PM -->
<civ-time-picker
  mode="text"
  label="Exact time of fall"
  name="fall_time"
  hint="Type the time as accurately as you remember (e.g. 2:34)"
  required
></civ-time-picker>

<!-- Appointment scheduling with some slots already booked -->
<civ-time-picker
  label="Appointment time"
  name="appt_time"
  min="09:00"
  max="17:00"
  minute-step="30"
  disabled-slots='["09:00","09:30","13:00"]'
  required
></civ-time-picker>

<!-- Forward-looking scheduling — earliest time is the device clock -->
<civ-time-picker
  label="Callback time today"
  name="callback_time"
  min="now"
  minute-step="30"
></civ-time-picker>

<!-- Text mode in 24-hour format — no AM/PM control -->
<civ-time-picker
  mode="text"
  label="Event timestamp"
  name="event_time"
  format="24"
></civ-time-picker>
```

## Accessibility

### Combo mode
- Single `<input role="combobox">` with an associated listbox. Arrow keys
  navigate, Enter and Tab both commit the highlighted slot, Escape
  restores the previous selection.
- **Digit-aware filter.** Typing matches both the visible label and the
  digit-only form of the label, so `"230"` finds `"2:30 AM"` and
  `"2:30 PM"`, and `"9 AM"` finds `"9:00 AM"`. The full label substring
  also matches (`"5:30"` → 5:30 AM/PM), as does the AM/PM modifier
  combined with digits (`"9p"` → 9:00 PM, 9:15 PM, …).
- **Out-of-grid / out-of-bounds `value`.** If the consumer sets
  `value="14:37"` with `minute-step="15"`, OR `value="08:00"` with
  `min="09:00"`, the value doesn't match a generated slot. The
  component injects a synthetic option so the input still renders the
  formatted label — the user sees what's submitted. Picking a real
  slot replaces it.
- **Free-typed unmatched text.** Typing a string the filter can't
  match leaves the form value empty. On blur, the input snaps back
  to the selected option's label, or clears entirely if there's no
  selection.

### Combo mode props
- Use `label` for combo mode (the combobox renders a `<label for>`).
  `legend` is accepted as a fallback so consumers sharing markup
  between modes don't have to swap props — but `label` is the
  canonical surface in combo mode.

### Select mode
- Real `<fieldset>` with a `<legend>`. If no `legend` is set, a
  visually-hidden "Time" fallback ensures the fieldset stays Section 508
  compliant.
- Hour and minute are real `<civ-select>` elements with their own
  labelled `<select>` controls. AM/PM is a `<civ-segmented-control>` —
  single-tap binary choice on every viewport, full-width on mobile.
- `aria-describedby` chains the legend → hint → error so screen readers
  announce them in order on focus.

### Text mode
- Same fieldset semantics as select mode. When only `label` is set
  (not `legend`), the fieldset legend is visually-hidden so the text
  input's own label is the canonical surface (single control, one
  visible label).
- `<civ-text-input>` with `mask-pattern="##:##"` and
  `inputmode="numeric"`. Mobile shows the numeric keypad; the colon
  is inserted automatically on blur. Raw value stored as 4 digits
  (no colon).
- Keyboard: type digits, tab to the AM/PM control, arrow keys
  toggle the segments.
- 12-hour format requires hour 1–12 + an AM/PM selection to
  assemble. Inputs outside that range (hour 0, hour > 12) leave the
  form value empty until corrected.

## Localization

All user-facing strings come from the locale registry:

- `timePickerDefaultLegend` (default `"Time"`, used when no `legend` /
  `label` is set)
- `timePickerPlaceholder` (default `"e.g. 9:00 AM"`, combo-mode input
  placeholder)
- `timePickerHourLabel` (default `"Hour"`, select mode)
- `timePickerMinuteLabel` (default `"Minute"`, select mode)
- `timePickerPeriodLabel` (default `"AM/PM"`, select + text modes)
- `timePickerTimeLabel` (default `"Time"`, text mode — sr-only label
  on the masked text input when the consumer hasn't set one)
- `timePickerAm` / `timePickerPm` (defaults `"AM"` / `"PM"`)

Override via `setLocale()` or per-component with the `placeholder`,
`hour-label`, `minute-label`, `period-label` props.
