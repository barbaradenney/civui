# Callout

<!-- contract-link:start -->

> **Contract reference:** [`civ-callout`](pathname:///civui/storybook/?path=/docs/contract-civ-callout--docs). Auto-generated structured contract for cross-platform implementation.

<!-- contract-link:end -->

`<civ-callout>`

A presentational primitive that frames slotted content with a left accent border and consistent padding. Five color variants map to the semantic palette.

Use for important notices, tips, or contextual information. The component is purely visual — it carries no ARIA role of its own and does not announce changes. For dismissible alerts with `role="status"` / `role="alert"`, use [`civ-alert`](../feedback/alert). For section openers with a heading and `role="region"` landmark, use [`civ-section-intro`](../form/section-intro) (which composes `civ-callout` internally).

## When to use it

- A short notice, tip, or reminder beside body content.
- A "before you start" block listing what the user will need.
- Inline emphasis around an arbitrary chunk of markup (paragraphs, lists, links).

For status updates that need to be announced to assistive tech, use `civ-alert` instead.

## Weight

| `callout-style` | Rail width | When to use |
|---|---|---|
| `primary` _(default)_ | 5px | Standalone callouts, "before you start" blocks, anything that should command attention. |
| `secondary` | 3px | Subtler emphasis when a callout sits near a primary callout or body content that shouldn't be visually overpowered. |

Padding and per-variant rail color stay identical across both
weights — swapping `callout-style` doesn't reflow surrounding layout.

```html
<civ-callout variant="info">
  <p>5px rail — the established treatment.</p>
</civ-callout>

<civ-callout variant="info" callout-style="secondary">
  <p>3px rail — quieter affordance for nested or stacked placements.</p>
</civ-callout>
```

## Composing other CivUI components inside

Callout uses a detached render root, so anything you author as a child
flows through as a normal light-DOM descendant. That means custom
elements (`civ-notice`, `civ-badge`, `civ-link`, `civ-button`, etc.)
compose freely — no slot wiring, no special props.

| Pair callout with | When to reach for it |
|---|---|
| [`civ-notice`](./notice) | Highlight a specific consequence inside the callout's body — legal warning, financial impact, safety detail |
| [`civ-badge`](./badge) | Add a status label to summarize what state the surrounding copy describes ("In review", "Verified", "Pending") |
| Both | Status badge at top → context paragraph → must-not-miss notice below |

For the same composition surface with banner chrome + dismiss button +
ARIA live region, use [`civ-alert`](./alert) instead — it
accepts the same children via the same Light-DOM mixin pattern.

## Accessibility

- The host element imposes **no role** of its own. When you need a landmark, set `role` (`region` / `complementary`) and `aria-labelledby` **directly on the `<civ-callout>` element** — never on an outer wrapper. Placing the role on the callout itself keeps the labelled bounding box and the visual chrome (accent border + padding) aligned. Wrapping the callout in `<div role="region"><civ-callout>...</civ-callout></div>` would push the labelled region outside the callout, so the heading would appear *before* the visual region rather than inside it.
- Content lives in Light DOM, so links, lists, and paragraphs are announced normally.
- Color is never the sole signal: pair the variant with descriptive text inside (e.g. *"Action needed: ..."* for `error`).

## Usage

```html
<!-- Plain notice -->
<civ-callout>
  <p>You must complete this section before continuing.</p>
</civ-callout>

<!-- Semantic variant -->
<civ-callout variant="warning">
  <p>Your session will expire in 15 minutes. Save your progress.</p>
</civ-callout>

<!-- Rich content -->
<civ-callout variant="info">
  <p>To finish this section, have ready:</p>
  <ul>
    <li>Your Social Security number</li>
    <li>Your most recent W-2</li>
  </ul>
</civ-callout>

<!-- As a labelled landmark -->
<civ-callout variant="warning" role="region" aria-labelledby="notice-heading">
  <p id="notice-heading" class="civ-font-bold">Important notice</p>
  <p>This region is announced as a landmark to assistive tech.</p>
</civ-callout>
```
