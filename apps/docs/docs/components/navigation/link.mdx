# Link

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | `''` | Link text (preferred over child text) |
| `href` | `string` | `''` | Link destination (required) |
| `variant` | `'primary' \| 'secondary' \| 'tertiary' \| 'back'` | `'tertiary'` | Visual variant |
| `danger` | `boolean` | `false` | Destructive action styling (red text) |
| `disabled` | `boolean` | `false` | Disabled state (sets `aria-disabled` and removes href) |
| `icon-start` | `string` | `''` | Icon name before link text (overrides back variant default chevron-left) |
| `icon-end` | `string` | `''` | Icon name after link text (overrides secondary variant default chevron-right) |

## Events

| Event | Detail | Description |
|-------|--------|-------------|
| `civ-analytics` | — | Analytics tracking event fired on click |

## Variants

### Primary

Button-styled link with filled blue background and underlined text.

```html
<civ-link href="/apply" variant="primary" label="Start application"></civ-link>
```

### Secondary

Underlined text link with a trailing chevron-right icon.

```html
<civ-link href="/benefits" variant="secondary" label="View all benefits"></civ-link>
```

### Tertiary

Plain underlined link (default variant).

```html
<civ-link href="/privacy" label="Privacy policy"></civ-link>
```

### Back

Back-navigation link with a leading chevron-left icon.

```html
<civ-link href="/previous-step" variant="back" label="Go back"></civ-link>
```

## Danger modifier

Add `danger` to any variant for destructive action styling.

```html
<civ-link href="/delete" variant="secondary" danger label="Remove application"></civ-link>
```

## Disabled state

```html
<civ-link href="/locked" variant="secondary" disabled label="Locked section"></civ-link>
```

## Action Link

`<civ-action-link>`

A unified link for device actions — phone calls and email compose. Set the `type` prop to switch between phone and email behavior. Renders the correct protocol (`tel:` / `mailto:`), icon (phone / mail), and formatted display text automatically.

<StoryEmbed id="actions-action-link--all-variants" />

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `type` | `'phone' \| 'email'` | `'phone'` | Action type — determines protocol and icon |
| `number` | `string` | `''` | Phone number (used when type="phone") |
| `address` | `string` | `''` | Email address (used when type="email") |
| `subject` | `string` | `''` | Pre-filled email subject (used when type="email") |
| `label` | `string` | `''` | Override display text |
| `disabled` | `boolean` | `false` | Disabled state |

```html
<!-- Phone -->
<civ-action-link type="phone" number="8005551234"></civ-action-link>
<civ-action-link type="phone" number="8005551234" label="Call us"></civ-action-link>

<!-- Email -->
<civ-action-link type="email" address="help@va.gov"></civ-action-link>
<civ-action-link type="email" address="help@va.gov" subject="Question"></civ-action-link>
```

## Specialized Links

For external links, use `<civ-link>` with the `external` prop. For file downloads, use `<civ-action-link type="download">`.

### External Link

Use `<civ-link external>` to open links in a new tab. This automatically sets `target="_blank"`, `rel="noopener noreferrer"`, appends an external-link icon, and adds screen-reader-only "(opens in new tab)" text.

```html
<civ-link
  href="https://www.ssa.gov"
  label="Social Security Administration"
  external
></civ-link>
```

### Download Link

Use `<civ-action-link type="download">` for file downloads. This sets the `download` attribute, shows a download icon, and optionally displays the file size.

```html
<civ-action-link
  type="download"
  href="/forms/va-10-10ez.pdf"
  label="Application for Health Benefits (VA 10-10EZ)"
  filename="va-10-10ez.pdf"
  file-size="1.2 MB"
></civ-action-link>
```
