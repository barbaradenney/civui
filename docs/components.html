Layout Components

Structural components for organizing page content.

<civ-page-header>

A structured page heading with optional eyebrow, tag, and subheading. Uses slot attributes to assign children to each area.

Slots

data-tag data-eyebrow data-heading data-subheading

Slot AttributeDescription
data-tagTag element displayed above the eyebrow (e.g., a <civ-tag> for status)
data-eyebrowSmall text above the heading (e.g., category or section name)
data-headingMain heading (typically an <h1>). Supports inline content like tags next to the title. Default slot for unattributed children.
data-subheadingText below the heading (e.g., form number, subtitle)

Example: Full header

<civ-page-header>
  <civ-tag data-tag label="Active" variant="green" tag-style="primary"></civ-tag>
  <span data-eyebrow>Benefits</span>
  <h1 data-heading>
    Apply for disability compensation
    <civ-tag label="In progress" variant="teal"></civ-tag>
  </h1>
  <span data-subheading>VA Form 21-526EZ</span>
</civ-page-header>

Example: Minimal header

<civ-page-header>
  <h1 data-heading>Dashboard</h1>
</civ-page-header>

Example: Eyebrow and heading only

<civ-page-header>
  <span data-eyebrow>Claims</span>
  <h1 data-heading>Check your claim status</h1>
</civ-page-header>
Note: Children without a data-* attribute are placed in the heading slot by default.

<civ-card>

A structured container with optional header and footer sections. Uses slot attributes for content placement.

Slots

data-card-header (default body) data-card-footer

Props

PropTypeDefaultDescription
spacing'default' | 'sm''default'Padding size inside the card

Example: Basic card

<civ-card>
  <p>Card content goes here.</p>
</civ-card>

Example: Card with header and footer

<civ-card>

    <h3 class="civ-heading-md">Disability compensation</h3>

Example: Compact card

  <p>Compact content with smaller padding.</p>
</civ-card>
Note: Children without a data-card-header or data-card-footer attribute are placed in the body section.

Navigation Components

Components for links, navigation cards, and task list patterns.

<civ-link>

An accessible link component. Always renders an <a> element. For buttons, use <civ-button> instead.

Props

PropTypeDefaultDescription
labelstring''Link text (preferred over child text)
hrefstring''Link destination (required)
variant'primary' | 'secondary' | 'tertiary' | 'back''tertiary'Visual variant
dangerbooleanfalseDestructive action styling (red color)
disabledbooleanfalseDisabled state (sets aria-disabled, removes href)

Variants

VariantAppearance
primaryButton-styled link (filled blue, underlined)
secondaryUnderlined text with trailing caret icon
tertiaryPlain underlined link
backBack navigation link with leading left chevron

Events

  • civ-analytics — Analytics tracking event on click

Example: All variants

<!-- Primary (button-styled) -->
<civ-link href="/apply" label="Start your application" variant="primary"></civ-link>

<!-- Secondary (with trailing caret) -->
<civ-link href="/details" label="View details" variant="secondary"></civ-link>

<!-- Tertiary (plain underlined) -->
<civ-link href="/help" label="Get help" variant="tertiary"></civ-link>

<!-- Back navigation -->
<civ-link href="/previous" label="Back to previous page" variant="back"></civ-link>

<!-- Danger modifier -->
<civ-link href="/delete" label="Delete account" variant="tertiary" danger></civ-link>
Note: All variants are underlined for accessibility. The danger modifier can be combined with any variant.

<civ-link-card>

A clickable card that navigates to a destination. The entire card is the click target. Renders as an <a> element.

Props

PropTypeDefaultDescription
hrefstring''Navigation destination (required)
headingstring''Card heading text
descriptionstring''Descriptive text below the heading
variant'primary' | 'secondary' | 'tertiary' | 'critical''primary'Visual variant

Variants

VariantAppearance
primaryBlue filled card with white text
secondaryWhite card with blue border and blue heading
tertiaryWhite card with gray border and blue heading
criticalYellow/gold background with black text for urgent actions

Events

  • civ-analytics — Analytics tracking event on click

Example

<!-- Primary (filled blue) -->
<civ-link-card
  href="/benefits/disability"
  heading="Disability compensation"
  description="File a claim for a service-connected disability."
></civ-link-card>

<!-- Secondary (blue border) -->
<civ-link-card
  href="/benefits/education"
  heading="Education benefits"
  description="Apply for GI Bill and other education benefits."
  variant="secondary"
></civ-link-card>

<!-- Critical (gold/yellow, urgent) -->
<civ-link-card
  href="/action-required"
  heading="Action required"
  description="Your application needs additional documentation."
  variant="critical"
></civ-link-card>

<civ-list> / <civ-list-item>

Generic list primitives. Use for task list hubs (the GDS Task List pattern), side navigation, link collections, and any vertical row layout where some rows are clickable and some are static.

<civ-list> Props

PropTypeDefaultDescription
dividersbooleanfalseRender a 1px divider between adjacent items

<civ-list-item> Props

PropTypeDefaultDescription
hrefstring''Navigation target. When set, the entire row becomes a clickable anchor (whole-row click target, hover state, focus ring). Omit for static rows.

Slots

SlotDescription
(default)Primary content (label, description, anything)
data-list-item-endTrailing content (status tag, switch, secondary action). Place the attribute on a child element.

Task list pattern

The "task list" is a usage pattern, not a dedicated component. Compose <civ-list> + <civ-list-item> + <civ-badge>. civ-badge carries role="status" and supports with-icon for the variant's semantic icon.

StatusBadge labelvariantbadge-style
not-startedNot startedinfosecondary
in-progressIn progressinfoprimary
completeCompletesuccessprimary
cannot-startCannot start yetneutralsecondary
errorHas errorserrorsecondary
reviewNeeds reviewwarningprimary

Example

<h3 class="civ-heading-md">Fill out your application</h3>
<civ-list dividers>
  <civ-list-item href="/name">
    <span class="civ-block civ-font-bold">Your name</span>
    <civ-badge data-list-item-end label="Complete" variant="success" badge-style="primary" with-icon></civ-badge>
  </civ-list-item>
  <civ-list-item href="/contact">
    <span class="civ-block civ-font-bold">Contact information</span>
    <span class="civ-block civ-text-sm civ-text-muted">Phone and email</span>
    <civ-badge data-list-item-end label="In progress" variant="info" badge-style="primary" with-icon></civ-badge>
  </civ-list-item>
  <civ-list-item>  <!-- no href = locked, same visual rhythm -->
    <span class="civ-block civ-font-bold">Review and submit</span>
    <civ-badge data-list-item-end label="Cannot start yet" variant="neutral" badge-style="secondary" with-icon></civ-badge>
  </civ-list-item>
</civ-list>
Note: Locked rows omit href and render as a plain <li> with no anchor — same visual rhythm, honest semantics.

Action Components

Interactive elements for triggering actions and displaying status.

<civ-button>

An accessible button component. Always renders a <button> element. For links, use <civ-link> instead.

Props

PropTypeDefaultDescription
labelstring''Button text (preferred over child text)
variant'primary' | 'secondary' | 'tertiary''primary'Visual variant
dangerbooleanfalseDestructive action styling
disabledbooleanfalseDisabled state
type'button' | 'submit' | 'reset''button'Button type attribute

Variants

VariantAppearance
primaryFilled blue button
secondaryOutlined border button
tertiaryGray filled button (same height as form inputs)

Events

  • civ-analytics — Analytics tracking event on click

Example

<!-- Primary -->
<civ-button label="Submit application" type="submit"></civ-button>

<!-- Secondary -->
<civ-button label="Save draft" variant="secondary"></civ-button>

<!-- Tertiary -->
<civ-button label="Cancel" variant="tertiary"></civ-button>

<!-- Danger -->
<civ-button label="Delete claim" variant="secondary" danger></civ-button>

<!-- Disabled -->
<civ-button label="Submit" disabled></civ-button>

<!-- Form reset -->
<civ-button label="Clear form" variant="tertiary" type="reset"></civ-button>
Note: Buttons do not get underlined text. Only links (<civ-link>) use underlines, preserving the distinction between navigation and action.

<civ-tag>

A small status label rendered as a colored pill. Used for status indicators, categories, or metadata.

Props

PropTypeDefaultDescription
labelstring''Tag text content
variant'blue' | 'teal' | 'red' | 'green' | 'yellow' | 'orange' | 'purple' | 'gray''gray'Color variant
tag-style'primary' | 'secondary''secondary'Emphasis level: primary (dark bg, light text) or secondary (light bg, dark text)

Color Variants

blue teal red green yellow orange purple gray

Example

<!-- Secondary style (light bg, default) -->
<civ-tag label="Not started" variant="blue"></civ-tag>
<civ-tag label="In progress" variant="teal"></civ-tag>
<civ-tag label="Pending review" variant="yellow"></civ-tag>

<!-- Primary style (dark bg) -->
<civ-tag label="Approved" variant="green" tag-style="primary"></civ-tag>
<civ-tag label="Denied" variant="red" tag-style="primary"></civ-tag>
<civ-tag label="Complete" variant="green" tag-style="primary"></civ-tag>
Note: Tag size scales with the density system. Wrap in a parent with data-civ-scale="dense" or data-civ-scale="spacious" to adjust.

Specialized Link Components

<civ-link new-tab> Opens in a new tab with security attributes and "opens in new tab" screen reader text. Set the new-tab prop on <civ-link>. Works for external and internal links.
<civ-action-link type="download"> File download link with download icon and optional file size display. Props: label, href, filename, file-size.
<civ-action-link type="phone"> Clickable phone (tel:) link with icon. Props: number, label.
<civ-action-link type="email"> Clickable email (mailto:) link with icon. Props: address, label, subject.

<civ-progress-steps>

A step indicator for multi-step forms. Responsive: horizontal on desktop, auto-switches to vertical on narrow screens.

Props

PropTypeDefaultDescription
stepsstring (JSON)'[]'JSON array of step labels or objects: ["Step 1","Step 2"] or [{"label":"Step 1","description":"Details"}]
currentnumber0Current step index (0-based)
orientation'horizontal' | 'vertical''horizontal'Layout direction
clickablebooleanfalseWhen true, completed steps become clickable buttons
show-counterbooleanfalseShow "Step X of Y" text below the steps
error-stepsstring (JSON)'[]'JSON array of step indices with validation errors
show-backbooleanfalseShow a "Go back" link before the step counter (visible when current > 0)
back-labelstring'Go back'Label for the back link

Events

  • civ-step-click — When a completed step is clicked (requires clickable). Detail: { step: number }
  • civ-step-back — When the "Go back" link is clicked (requires show-back). Detail: { from, to }

Example: Simple steps

<civ-progress-steps
  steps='["Personal info","Employment","Review","Submit"]'
  current="1"
  show-counter
></civ-progress-steps>

Example: Rich steps with descriptions

<civ-progress-steps
  steps='[
    {"label":"Personal info","description":"Name and contact details"},
    {"label":"Employment","description":"Current employer"},
    {"label":"Review","description":"Check your answers"}
  ]'
  current="1"
  clickable
></civ-progress-steps>

Example: Vertical with errors

<civ-progress-steps
  steps='["Step 1","Step 2","Step 3","Step 4"]'
  current="2"
  orientation="vertical"
  error-steps="[0]"
></civ-progress-steps>

<civ-form-step>

Multi-step form wizard. Shows one step at a time, validates required fields before advancing, and composes <civ-progress-steps> for visual progress tracking.

Props

PropTypeDefaultDescription
persiststring''Storage key for civ-form session persistence
continue-labelstring'Continue'Label for the continue button
complete-labelstring'Save and continue'Label for the final step's button
validatebooleantrueEnable built-in required field validation before advancing
nav-disabledbooleanfalseDisable navigation buttons (e.g., during async validation)

Events

  • civ-step-complete — All steps completed and validated. Detail: { total }
  • civ-step-continue — Advancing to next step. Detail: { from, to }
  • civ-step-back — Going back to previous step. Detail: { from, to }
  • civ-step-change — Step changed (any direction). Detail: { current, total, label }

How it works

  • Each child element with data-step-label defines a step
  • Renders <civ-progress-steps> automatically with step labels, counter, and "Go back" link
  • Validates required fields before advancing (uses component reportValidity() for SSN, email, etc.)
  • Wraps content in <civ-form persist="..."> when persist is set
  • Manages focus — moves to first focusable field on step change
  • Screen reader announcements via announce() on step change

Example

<civ-form-step persist="form-21-526ez-personal" complete-label="Save and continue">
  <div data-step-label="Your name">
    <civ-name legend="Your name" name="fullName" required></civ-name>
  </div>
  <div data-step-label="Date of birth">
    <civ-memorable-date label="Date of birth" name="dob" required
      hint="For example: January 15 1990"></civ-memorable-date>
  </div>
  <div data-step-label="Social Security number">
    <civ-text-input label="Social Security number" name="ssn" required
      type="tel" mask="ssn"></civ-text-input>
  </div>
</civ-form-step>

Component composition

<civ-form-step> composes <civ-progress-steps> internally, passing the step labels, current index, show-counter, and show-back. <civ-progress-steps> is a standalone component that can also be used independently outside of forms.

Form Components

Form-participating components for building accessible government forms. All form components extend CivFormElement and share a standard API: label, name, value, hint, error, required, disabled. Group components use legend instead of label.

For interactive examples and full configuration options, see the Storybook.

Input Components

<civ-text-input> Standard text input. Supports types: text, email, number, password, search, tel, url. Props: type, width, placeholder, maxlength, minlength, pattern, autocomplete, inputmode, mask, validate.
<civ-textarea> Multi-line text input. Shows character count when maxlength is set. Props: rows, maxlength, placeholder.
<civ-select> Dropdown select. Populate via options property or slotted <option> elements. Props: options, emptyLabel.
<civ-combobox> Searchable dropdown with type-ahead filtering. Props: options, placeholder, noResultsText. Events: civ-change includes { value, label }.

Preset Input Components

Pre-configured wrappers around civ-text-input and civ-combobox with built-in masking, validation, and labeling for common government form fields.

<civ-ssn> Social Security number with masking, validation, and PII protection. Props: mode ('full' | 'last4').
<civ-ein> Employer Identification Number with masking, validation, and PII protection.
<civ-phone> Phone number with US domestic masking or E.164 international format. Props: international.
<civ-email> Email address with built-in validation and autocomplete.
<civ-zip> US ZIP code (5-digit or ZIP+4). Props: extended.
<civ-currency> Dollar amount with currency masking and validation.
<civ-routing-number> Bank routing number with 9-digit masking and mod-10 checksum validation.
<civ-va-file-number> VA file number (8-9 digits) with labeling to reduce SSN confusion.
<civ-country> Country combobox with full ISO 3166-1 list. Props: us-first, include, exclude.

Choice Components

<civ-checkbox> Single checkbox. Props: checked, indeterminate, description, tile. Event detail: { checked, value }.
<civ-checkbox-group> Multiple checkboxes grouped under a legend. Props: legend, tile, orientation. Event detail: { values: string[] }.
<civ-radio-group> Mutually exclusive choice group. Contains <civ-radio> children. Props: legend, tile, orientation. Arrow keys navigate.
<civ-radio> Individual radio option (child of <civ-radio-group> only). Props: label, value, checked, description, tile.
<civ-toggle> On/off switch with immediate-effect semantics. Uses role="switch". Props: checked, description. Event detail: { checked, value }.
<civ-segmented-control> Button-style radio group for UI mode switching. Contains <civ-segment> children. Props: legend.
<civ-segment> Individual segment button (child of <civ-segmented-control> only). Props: label, value, selected.
<civ-yes-no> Binary yes/no radio group for eligibility questions. Props: legend, yesLabel, noLabel. Event detail: { value } ("yes" or "no").

Date Components

<civ-date-picker> Calendar dialog with text input. For scheduling and appointment dates. Props: min, max, placeholder, locale, weekStartsOn. Form value: YYYY-MM-DD.
<civ-memorable-date> Three-field date entry (Month/Day/Year). For known past dates like birthdays. Props: legend, monthLabel, dayLabel, yearLabel, locale.

File & Upload

<civ-file-upload> Drag-and-drop file upload with validation. Props: accept, multiple, maxSize, maxFiles. Event detail: { files: File[] }.

Layout & Structure

<civ-form> Form validation coordinator. Renders error summary with anchor links on submit. Props: action, method. Events: civ-submit, civ-invalid.
<civ-fieldset> Structural grouping wrapper with legend, hint, and error support. Not form-participating. Props: legend, hint, error, required, disabled.
<civ-conditional> Conditionally shows children based on another field's value. Props: when, equals, not-equals, includes, has-value, matches. Not form-participating.
<civ-form-step> Multi-step form wizard. One step at a time with validation, back navigation, progress tracking via civ-progress-steps, and form persistence.

Composite Components

<civ-name> Full name input with first, middle, last, and suffix fields. Common government form pattern.
<civ-address> Complete address input with street, city, state, and ZIP fields.
<civ-direct-deposit> Bank account entry with routing number, account number, and account type fields.
<civ-signature> Electronic signature component for form certification.
<civ-race-ethnicity> OMB-standard race and ethnicity collection with ethnicity radio group and race multi-select checkboxes.
<civ-marriage-history> Single marriage entry with spouse name, marriage date/location, status, and conditional end date. Use inside civ-repeater for multiple marriages.
<civ-service-history> Single military service period with branch, dates, discharge type, and optional service number. Use inside civ-repeater for multiple periods.
<civ-relationship> Person and relationship to applicant with presets for dependent, survivor, immigration, and tax scenarios.
<civ-summary> Review/summary display for form data before submission.
<civ-repeater> Repeatable field group for dynamic lists (e.g., multiple addresses, employment history).
<civ-read-only-field> Displays a label/value pair in read-only mode for review pages.
