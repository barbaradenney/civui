# Repeating Sections

Collect variable-length lists of items — dependents, service periods, previous addresses, employment history. `civ-repeater` has two modes; pick based on how much information you collect per item.

## Inline mode (1–3 fields per item)

Default. Every row's fields are visible on the page at the same time with add / remove buttons in line.

<StoryEmbed id="forms-form-repeater--default" />

```html
<civ-repeater legend="Dependents" name="dependents" item-label="dependent">
  <civ-text-input label="Full name" name="fullName"></civ-text-input>
</civ-repeater>
```

A more substantial inline example with multiple fields per row:

<StoryEmbed id="forms-form-repeater--multiple-fields" />

```html
<civ-repeater legend="Dependents" name="dependents" item-label="dependent">
  <civ-text-input label="First name" name="firstName"></civ-text-input>
  <civ-text-input label="Last name" name="lastName"></civ-text-input>
</civ-repeater>
```

## Form-steps mode (4+ fields, or multi-step entry)

Set `mode="form-steps"`. Existing rows render as compact summary cards; clicking **Add** or **Edit** opens a step-by-step entry flow, one section at a time. Use this when each item has enough fields that an inline row would overwhelm the page.

<StoryEmbed id="forms-form-repeater--form-steps-mode" />

```html
<civ-repeater
  legend="Dependents"
  name="dependents"
  item-label="dependent"
  mode="form-steps"
>
  <div data-step-label="Name">
    <civ-text-input label="First name" name="firstName" required></civ-text-input>
    <civ-text-input label="Last name" name="lastName" required></civ-text-input>
  </div>
  <div data-step-label="Contact">
    <civ-text-input label="Email" name="email" type="email"></civ-text-input>
    <civ-text-input label="Phone" name="phone" type="tel"></civ-text-input>
  </div>
</civ-repeater>
```

Form-steps mode also composes with the compound components — here, a dependents list that captures a name in step 1 and a relationship in step 2:

<StoryEmbed id="forms-form-repeater--form-steps-with-relationship" />

## Min and max constraints

Set `min` and `max` to control the allowed number of items:

<StoryEmbed id="forms-form-repeater--min-max" />

```html
<!-- At least 1 service period, up to 10 -->
<civ-repeater legend="Service periods" name="service" item-label="service period" min="1" max="10">
  ...
</civ-repeater>
```

## Which mode to use

| Fields per item | Mode | Example |
|----------------|------|---------|
| 1–3 simple fields | `inline` (default) | Phone numbers, email addresses, other phone numbers |
| 4+ fields, or multi-step entry | `form-steps` | Dependents, employment history, service periods |

## Components used

- [`civ-repeater`](/components/form/repeater) — repeating section container
- Compound components work inside repeaters: `civ-name`, `civ-address`, `civ-service-history`, `civ-relationship`
