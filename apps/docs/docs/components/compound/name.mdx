`<civ-name>`

Compound name input with first, middle (optional), last, and suffix fields. Follows the VA.gov name pattern with plain language labels. Field labels read "First name or given name" and "Last name or family name" so the form works for any naming convention. The `required` indicator is NOT placed on the legend since middle name and suffix are always optional; instead `required` is set on the first and last name sub-fields individually.

## Value Structure

The component stores its value as a JSON string:

```typescript
interface NameValue {
  first: string;
  middle: string;
  last: string;
  suffix: string;  // 'Jr.', 'Sr.', 'II', 'III', 'IV'
}
```

## Form Data

The component submits as separate fields:
- `{name}.first`
- `{name}.middle`
- `{name}.last`
- `{name}.suffix`

## Required Logic

When `required` is set:
- First name and last name fields are marked required
- Middle name and suffix are always optional
- The legend does NOT show the required indicator (since not all fields are required)

## Sub-field Autocomplete

Fields automatically set appropriate `autocomplete` values:
- First name: `given-name`
- Middle name: `additional-name`
- Last name: `family-name`
- Suffix: `honorific-suffix`

## Programmatic API

```js
const name = document.querySelector('civ-name');
name.nameValue; // { first: 'Jane', middle: 'A', last: 'Doe', suffix: '' }
```

## Usage

```html
<!-- Full name with all fields -->
<civ-name
  legend="Your name"
  name="veteranName"
  required
></civ-name>

<!-- Without middle name and suffix -->
<civ-name
  legend="Spouse name"
  name="spouseName"
  show-middle="false"
  show-suffix="false"
  required
></civ-name>

<!-- With per-field errors (server validation) -->
<civ-name
  legend="Legal name"
  name="legalName"
  first-error="First name must contain only letters"
  required
></civ-name>
```
