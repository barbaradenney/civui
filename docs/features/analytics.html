Analytics Events

PII-safe, composable analytics events built into every component. Wire to Google Analytics, Mixpanel, or any platform with a single listener.

How It Works

CivUI's analytics system uses standard DOM events. There is no vendor lock-in, no SDK to install, and no configuration file. Every component emits a civ-analytics custom event on user interactions, and you wire that event to whatever analytics platform your agency uses.

  1. Components fire civ-analytics custom DOM events on user interactions — form changes, button clicks, form submissions, and more.
  2. Events bubble up through the DOM — listen at the document level, on a specific form, or on an individual component. Standard event propagation, no magic.
  3. Wire the listener to your analytics platform — Google Analytics, Mixpanel, Amplitude, a custom backend, or anything else that accepts JavaScript calls.

The simplest possible setup:

document.addEventListener('civ-analytics', (e) => {
  const { componentName, action, fieldName } = e.detail;

  // Google Analytics
  gtag('event', action, {
    event_category: componentName,
    event_label: fieldName,
  });
});

Event Payload

Every civ-analytics event carries a detail object with the AnalyticsEventDetail shape: 

{
  componentName: string;    // "civ-text-input", "civ-checkbox"
  action: string;           // "change", "click", "submit"
  fieldName?: string;       // from the name prop
  label?: string;           // from the label/legend prop
  timestamp: string;        // ISO 8601
  details?: Record<string, string | number | boolean>;
}
PII Safety: Values are never included. The event payload contains only metadata — component name, field name, action type. A checkbox fires { action: "change", checked: true } but never the checkbox's value. A text input fires { action: "change" } but never what the user typed. This means analytics events are safe to send to third-party platforms without risking personally identifiable information.

What Each Component Tracks

Every CivUI component fires analytics events automatically. The table below shows what each component tracks and when.

Component Action Extra Details When
text-input change — Value committed (blur/enter)
textarea change — Value committed
select change — Option selected
combobox select — Option selected from dropdown
checkbox change { checked } Toggled
checkbox-group change — Any child toggled
radio-group change — Selection changed
toggle change { checked } Toggled
date-picker change — Date selected
memorable-date change — Date committed
segmented-control change — Segment selected
file-upload upload { fileCount } Files added
file-upload remove { fileCount } File removed
form submit — Valid submission
form invalid { errorCount } Validation failed
button click — Clicked
alert dismiss — Dismissed

Opt-Out

Any component can opt out of analytics events with the disable-analytics attribute. When present, the component will not fire any civ-analytics events. 

<!-- No analytics events fired for this field -->
<civ-text-input label="SSN" name="ssn" disable-analytics></civ-text-input>

This is useful for sensitive fields where even metadata could be a privacy concern. For example, a Social Security number field should not emit analytics events because the field name alone (ssn) reveals that a user interacted with that field on the form. Use disable-analytics on any field where tracking interaction metadata is inappropriate.

Platform Integration Examples

The civ-analytics event works with any analytics platform that accepts JavaScript calls. Here are three common integrations.

Google Analytics (gtag)

document.addEventListener('civ-analytics', (e) => {
  gtag('event', e.detail.action, {
    event_category: e.detail.componentName,
    event_label: e.detail.fieldName,
  });
});

Mixpanel

document.addEventListener('civ-analytics', (e) => {
  mixpanel.track(e.detail.action, {
    component: e.detail.componentName,
    field: e.detail.fieldName,
  });
});

Custom Backend

document.addEventListener('civ-analytics', (e) => {
  fetch('/api/analytics', {
    method: 'POST',
    body: JSON.stringify(e.detail),
  });
});

Form Funnel Tracking

A common use case is tracking form completion funnels — how many users start a form, how many hit validation errors, and how many successfully submit. The civ-analytics events make this straightforward. 

document.addEventListener('civ-analytics', (e) => {
  const { action, componentName, details } = e.detail;

  if (action === 'submit') {
    gtag('event', 'form_complete', { form: componentName });
  }
  if (action === 'invalid') {
    gtag('event', 'form_error', {
      form: componentName,
      error_count: details?.errorCount,
    });
  }
});

Combined with field-level change events, you can build a complete picture of how users move through a form — which fields they interact with, where they encounter errors, and where they drop off — all without capturing any PII.
