# Text Input

`<civ-text-input>` `<civ-zip>`

Accessible text input with label, hint, error, and width variants. Supports input masking, declarative validation, prefix/suffix text, and inline icons. Uses ElementInternals for native form participation. The remaining single-input ZIP preset is documented [below](#specialized-presets); contact fields live on [**Contact**](./contact.mdx), sensitive identifiers on [**PII**](./pii.mdx), and numeric inputs (dollar amounts, routing numbers, generic numbers) on [**Number**](./number.mdx).

### Spacing — default vs compact

Set `spacing="sm"` to render the bare `<input>` with no surrounding label / hint / error chrome. The host's `aria-label` is propagated to the inner control so assistive tech still gets an accessible name; mask and validation behavior are preserved. Used by data-grid cell editors and other dense surfaces where the row or card around the input already names the field.

### Character Counter

When `maxlength` is set, the input renders a "characters remaining" counter below the field. Visual count updates immediately; screen-reader announcements debounce on a 1s interval to avoid spamming. The counter is suppressed when a mask is active (the mask drives the length constraint).

### Clearable

Set `clearable` to render an inline "×" button on the trailing edge whenever the value is non-empty. Clicking it clears the value, refocuses the input, and fires `civ-input` + `civ-change` with `{ value: "" }`. The clear button takes precedence over any `trailing-icon` while the field has content.

<StoryEmbed id="forms-inputs-text-input--clearable" />

### Password Reveal

Set `reveal-password` on a `type="password"` input to render a **Show password** / **Hide password** toggle chip in the helper row below the input. Clicking it flips the rendered input type between `password` and `text`; the host's `type` prop stays `password` so the field's semantic identity doesn't change. Suppressed when `type` is anything other than `password`.

```html
<civ-text-input label="Password" name="password" type="password" reveal-password></civ-text-input>
```

<StoryEmbed id="forms-inputs-text-input--password-reveal" />

The reveal toggle lives **below** the input rather than inset for two reasons. (1) The only affordance that stays inset is the clear button — a destructive control that operates on the value and stays tightly bound to the input's visual box. (2) USWDS, GOV.UK, and Material all place "Show password" below the field; an inset Show/Hide text label on mobile overlaps the typed value and competes with the clear button for the same tap region.

Reveal coexists cleanly with `clearable`: the × clear button stays inset on the trailing edge, the Show password / Hide password toggle sits in the helper row below. They never collide.

### Below-Action Slot

For affordances CivUI doesn't ship as props — copy-to-clipboard, paste, scan, generate — pass a child element with the `data-below-action` attribute. The child renders in a helper row directly under the input (not inset inside the input chrome). Drop in `<civ-confirm-button>` for the common "Copy → Copied ✓" pattern, or pass any styled `<button class="civ-text-btn civ-text-btn--chip">` if you need full control. The slot is **web-only** (Light DOM design system; iOS/Android/Drupal do not project this) and is intentionally an escape hatch — if a use case becomes common, we'll promote it to a first-class prop.

```html
<civ-text-input label="API key" name="apiKey" value="sk-...">
  <civ-confirm-button
    data-below-action
    label="Copy"
    success-label="Copied"
    @civ-confirm="${onCopy}"
  ></civ-confirm-button>
</civ-text-input>
```

<StoryEmbed id="forms-inputs-text-input--below-action-slot" />

#### Why below the input, not inset?

The inset region inside the input is reserved for **one** component-rendered control: the close / × clear button. It has universally-understood semantics ("clear this value") and stays tightly coupled to the field's visual box.

Anything else (Copy, Today, Now, Show / Hide password, generate, units toggle) sits in a helper row below the input. This separation:

- Eliminates touch-target conflicts with the input's bounding box on mobile.
- Removes the "does this button modify the value or submit it?" ambiguity.
- Mirrors USWDS, GOV.UK, and Material — all place "Show password" below the field, not inset.

#### Trailing inset region order

The trailing inset region inside the input holds at most one component-rendered button at a time. **Component-rendered buttons follow this precedence** (highest wins):

1. **Clear button** — when `clearable` is set and the value is non-empty
2. **Trailing icon** — passive icon from the `trailing-icon` prop
3. **Suffix** — text label (mutually exclusive with the trailing icon; `suffix` always renders when set)

The Show / Hide password toggle is **not** inset — it lives in the helper row below the input, so `clearable` and `reveal-password` can both be set without colliding.

## Custom Mask Patterns

For formats not covered by the specialized components, use `mask-pattern` with slot syntax:

| Slot | Matches |
|------|---------|
| `#` | Digit (0-9) |
| `A` | Letter (a-z, A-Z) |
| `*` | Any printable character |

All other characters are inserted as literals.

### Mask Modes

- **`blur`** (default): User types freely. Mask applied on blur for display. Accessible and predictable.
- **`live`**: Formats as user types. Has accessibility tradeoffs (cursor jumping).

## Specialized Presets

These wrappers extend `<civ-text-input>` with pre-configured masking, validation, default labels, hints, and HTML attributes. They share the same events (`civ-input`, `civ-change` with `{ value: string }` detail). The presets below are general-purpose text-input wrappers; identifier presets that need PII protection are on the [**PII**](./pii.mdx) page, contact presets on [**Contact**](./contact.mdx), and numeric presets (dollar amounts, routing numbers) on [**Number**](./number.mdx).

### ZIP Code (`<civ-zip>`)

Pre-configured input for US ZIP codes with automatic masking and validation. Supports standard 5-digit and extended ZIP+4 formats. Used as a leaf field in `<civ-address>`.

Default label: "ZIP code"

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `extended` | `boolean` | `false` | Enable ZIP+4 format (e.g., 12345-6789) |

<StoryEmbed id="forms-inputs-text-input--zip-code" />

<StoryEmbed id="forms-inputs-text-input--zip-plus-4" />

```html
<!-- Standard 5-digit ZIP code -->
<civ-zip label="ZIP code" name="zip" required></civ-zip>

<!-- ZIP+4 format -->
<civ-zip label="ZIP code" name="zip" extended required></civ-zip>
```

## Usage

```html
<!-- Basic text input -->
<civ-text-input label="Full name" name="fullName" required></civ-text-input>

<!-- Custom mask pattern -->
<civ-text-input label="Case number" hint="For example: ABC-1234" name="caseNumber" mask-pattern="AAA-####"></civ-text-input>

<!-- With prefix -->
<civ-text-input label="Social handle" name="handle" prefix="@" width="md"></civ-text-input>

<!-- With suffix -->
<civ-text-input label="Weight" name="weight" suffix="lbs" inputmode="decimal" width="sm"></civ-text-input>

<!-- With character counter -->
<civ-text-input label="Short bio" name="bio" maxlength="60"></civ-text-input>
```
