`<civ-conditional>`

A declarative show/hide wrapper that reveals its content only when a named field has a specified value. Listens for `civ-input` and `civ-change` events scoped to the nearest form ancestor for performance.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `when` | `string` | `''` | The `name` of the field to watch |
| `equals` | `string` | `''` | Show when the field value matches exactly |
| `not-equals` | `string` | `''` | Show when the field value does NOT match |
| `includes` | `string` | `''` | Show when the field value is in this comma-separated list |
| `has-value` | `boolean` | `false` | Show when the field has any non-empty value |
| `matches` | `string` | `''` | Show when the field value matches this regex pattern |

## Behavior

- Content starts hidden (`display: none`, `aria-hidden="true"`).
- The component listens for `civ-input` and `civ-change` events scoped to the nearest form ancestor (see "Event Scoping" below).
- When the condition becomes true, the wrapper switches to `display: block` and `aria-hidden="false"`. When it becomes false, it flips back.
- Hiding via `display: none` (rather than DOM removal) preserves child state — typed values, focus history, and uploaded files survive a hide/show cycle — and keeps screen readers from announcing the content while it's hidden.
- `civ-form` recognizes the hidden wrapper and excludes fields inside it from both `validate()` and form-data collection (`getFormData()` / `toFormData()`), so required fields in an inactive branch don't block submit or send stale values.

## Supported field types

| Field type | Event detail | How to match |
|-----------|-------------|--------------|
| Text input, select, yes-no | `{ value: string }` | `equals`, `not-equals`, `includes`, `has-value`, `matches` |
| Checkbox, toggle | `{ checked: boolean, value: string }` | `equals="true"` (show when checked), `equals="false"` (show when unchecked) |
| Checkbox group | `{ values: string[] }` | `equals="optionValue"` (show when that option is selected) |
| Radio group | `{ value: string }` | Same as text input — matches the selected radio's value |

## Event Scoping

For performance, event listeners are scoped to the nearest ancestor:
1. `civ-form`
2. Native `<form>`
3. `civ-fieldset`
4. Native `<fieldset>`
5. `document` (fallback)

## Usage

```html
<!-- Show follow-up question when "Yes" is selected -->
<civ-yes-no
  legend="Do you have a service-connected disability?"
  name="hasDisability"
  required
></civ-yes-no>

<civ-conditional when="hasDisability" equals="yes">
  <civ-text-input
    label="Describe your disability"
    name="disabilityDescription"
    required
  ></civ-text-input>
</civ-conditional>

<!-- Show section when a specific option is NOT selected -->
<civ-select
  label="Country of residence"
  name="country"
></civ-select>

<civ-conditional when="country" not-equals="US">
  <civ-text-input
    label="International phone number"
    name="intlPhone"
    type="tel"
  ></civ-text-input>
</civ-conditional>

<!-- Chain multiple conditionals -->
<civ-radio-group legend="Employment status" name="employment">
  <civ-radio label="Employed" value="employed"></civ-radio>
  <civ-radio label="Self-employed" value="self-employed"></civ-radio>
  <civ-radio label="Unemployed" value="unemployed"></civ-radio>
  <civ-radio label="Retired" value="retired"></civ-radio>
</civ-radio-group>

<civ-conditional when="employment" equals="employed">
  <civ-text-input label="Employer name" name="employer" required></civ-text-input>
</civ-conditional>

<civ-conditional when="employment" equals="self-employed">
  <civ-text-input label="Business name" name="businessName" required></civ-text-input>
</civ-conditional>

<!-- Show for multiple values (comma-separated list) -->
<civ-conditional when="employment" includes="employed, self-employed">
  <civ-text-input label="Annual income" name="income" required></civ-text-input>
</civ-conditional>

<!-- Show when any value is entered -->
<civ-conditional when="email" has-value>
  <civ-checkbox label="Send me email updates" name="emailUpdates"></civ-checkbox>
</civ-conditional>

<!-- Show when value matches a regex pattern -->
<civ-conditional when="zip" matches="^\\d{5}$">
  <p class="civ-text-caption">We found your local VA office.</p>
</civ-conditional>
```
