# Task List Hub

For large applications with multiple chapters (personal info, service history, documents, etc.), the task list hub is the user's home base. It shows all sections with their completion status and lets users work through them in any order.

Based on the [GDS Task List pattern](https://design-system.service.gov.uk/components/task-list/) adapted for government benefit applications. Composed from `<civ-list>` + `<civ-list-item>` + `<civ-badge>` — there's no dedicated task-list component.

```html
<h1>Apply for disability compensation</h1>

  </civ-list-item>

```

Setting `href` on a `<civ-list-item>` makes the whole row a clickable anchor. Omit `href` for locked rows — the row still renders with the same layout but is no longer a link, which is honest both visually and to assistive tech.

## Status badges

`<civ-badge>` carries `role="status"` and the `with-icon` attribute auto-renders the variant's semantic icon.

| Status | Badge label | variant | badge-style | Meaning |
|--------|-------------|---------|-------------|---------|
| `not-started` | Not started | `info` | `secondary` | User hasn't begun this section |
| `in-progress` | In progress | `info` | `primary` | Started but incomplete |
| `complete` | Complete | `success` | `primary` | All steps done and reviewed |
| `cannot-start` | Cannot start yet | `neutral` | `secondary` | Depends on other sections being complete (no `href`) |
| `error` | Has errors | `error` | `secondary` | Validation errors need attention |
| `review` | Needs review | `warning` | `primary` | Prefilled data needs user confirmation |

## Sequential unlocking

Lock later sections until prerequisites are complete by **omitting the `href`** on `<civ-list-item>`. Without an `href`, the row renders as a plain `<li>` with no anchor — visually identical, but no longer a click target. Pair with the `cannot-start` badge so users see why the row is locked.

```html
<civ-list-item>
  <span class="civ-block civ-font-bold">Supporting documents</span>
  <span class="civ-block civ-text-sm civ-text-muted">Complete service history first</span>
  <civ-badge data-list-item-end label="Cannot start yet" variant="neutral" badge-style="secondary" with-icon></civ-badge>
</civ-list-item>
```

## Updating status from JavaScript

When a chapter completes, find the matching `<civ-list-item>` by `data-chapter-id` and update its inner `<civ-badge>`:

```js
function setItemStatus(item, status, href) {
  const badge = item.querySelector('civ-badge');
  const map = {
    'not-started':  { label: 'Not started',      variant: 'info',    style: 'secondary' },
    'in-progress':  { label: 'In progress',      variant: 'info',    style: 'primary' },
    'complete':     { label: 'Complete',         variant: 'success', style: 'primary' },
    'cannot-start': { label: 'Cannot start yet', variant: 'neutral', style: 'secondary' },
    'error':        { label: 'Has errors',       variant: 'error',   style: 'secondary' },
    'review':       { label: 'Needs review',     variant: 'warning', style: 'primary' },
  };
  const def = map[status];
  if (badge && def) {
    badge.setAttribute('label', def.label);
    badge.setAttribute('variant', def.variant);
    badge.setAttribute('badge-style', def.style);
  }
  if (href) {
    item.setAttribute('href', href);
  } else {
    item.removeAttribute('href');
  }
}
```

## Chapter flow

Each row links to a chapter. Inside the chapter, the user works through steps using `civ-form-step`. When complete, they return to the hub with the row marked as `complete`.

```
Hub → Chapter (multi-step) → Hub (updated status)
```

## Accessibility

- Each clickable row is a real `<a>` filling the full `<li>` — works with keyboard, screen readers, and right-click "open in new tab".
- Static (locked) rows are plain `<li>` with no anchor — assistive tech doesn't announce them as links.
- Section headings are real `<h3>` elements that screen readers can navigate to as landmarks.
- `<civ-badge>` carries `role="status"` so status changes are announced.
- Progress bar announces completion percentage.

## Components used

- [`civ-list`](/docs/components/layout/list) — generic list container with optional `dividers`
- [`civ-list-item`](/docs/components/layout/list) — list row; set `href` to make the whole row clickable, omit for locked rows
- `civ-badge` — semantic status indicator with `role="status"` and `with-icon` for the variant's icon
- `civ-progress-bar` — overall completion
