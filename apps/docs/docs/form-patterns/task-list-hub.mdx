# Task List Hub

For large applications with multiple chapters (personal info, service history, documents, etc.), the task list hub is the user's home base. It shows all sections with their completion status and lets users work through them in any order.

Based on the [GDS Task List pattern](https://design-system.service.gov.uk/components/task-list/) adapted for government benefit applications. Composed from `<civ-list>` + `<civ-list-item>` + `<civ-badge>`. There's no dedicated task-list component.

Choosing between the hub, a process list, a timeline, and progress indicators? See [Choosing a Step Display](./choosing-step-displays).

## Basic structure

```html
<h1>Apply for disability compensation</h1>
<p>Complete each section. You can do them in any order.</p>

<civ-progress-percent
  value="40"
  label="Overall progress"
  status="2 of 5 sections complete"
></civ-progress-percent>

<h3 class="civ-heading-md">Prepare</h3>
<civ-list dividers>
  <civ-list-item href="#/eligibility" heading="Check your eligibility">
    <civ-badge data-list-item-end label="Complete" intent="success" emphasis="primary" with-icon></civ-badge>
  </civ-list-item>
</civ-list>

<h3 class="civ-heading-md">Fill out your application</h3>
<civ-list dividers>
  <civ-list-item href="#/personal" heading="Personal information">
    <civ-badge data-list-item-end label="Complete" intent="success" emphasis="primary" with-icon></civ-badge>
  </civ-list-item>
  <civ-list-item href="#/contact" heading="Contact information" description="Phone number needed">
    <civ-badge data-list-item-end label="In progress" intent="info" emphasis="primary" with-icon></civ-badge>
  </civ-list-item>
  <civ-list-item href="#/service" heading="Service history">
    <civ-badge data-list-item-end label="Not started" intent="info" emphasis="secondary" with-icon></civ-badge>
  </civ-list-item>
  <civ-list-item heading="Supporting documents">
    <civ-badge data-list-item-end label="Cannot start yet" intent="neutral" emphasis="secondary" with-icon></civ-badge>
  </civ-list-item>
</civ-list>

<h3 class="civ-heading-md">Review and submit</h3>
<civ-list dividers>
  <civ-list-item heading="Review your application" description="Complete all sections first">
    <civ-badge data-list-item-end label="Cannot start yet" intent="neutral" emphasis="secondary" with-icon></civ-badge>
  </civ-list-item>
</civ-list>
```

Use the `heading` (and optional `description`) props for the row text rather than hand-rolled `<span>`s — on a row with `href`, `civ-list-item` renders the heading as an **underlined, primary-colored link** so the clickable text reads as a link. Omit `href` for locked rows: the heading then renders as plain bold text (no underline, no link color), which is honest both visually and to assistive tech. On mobile (≤480px) a row whose trailing content is a status badge stacks the badge **above** the heading so neither gets squeezed against the viewport edge.

## Status badges

`<civ-badge>` carries `role="status"` and the `with-icon` attribute auto-renders the variant's semantic icon.

| Status | Badge label | variant | badge-style | Meaning |
|--------|-------------|---------|-------------|---------|
| `not-started` | Not started | `info` | `secondary` | User hasn't begun this section |
| `in-progress` | In progress | `info` | `primary` | Started but incomplete |
| `complete` | Complete | `success` | `primary` | All steps done and reviewed |
| `cannot-start` | Cannot start yet | `neutral` | `secondary` | Depends on other sections being complete (no `href`) |
| `error` | Has errors | `error` | `secondary` | Validation errors need attention |
| `review` | Needs review | `warning` | `primary` | Prefilled data needs user confirmation |

## Sequential unlocking

Lock later sections until prerequisites are complete by **omitting the `href`** on `<civ-list-item>`. Without an `href`, the row renders without an anchor. Visually identical, but no longer a click target. Pair with the `cannot-start` badge so users see why the row is locked.

```html
<civ-list-item>
  <span class="civ-block civ-font-bold">Supporting documents</span>
  <span class="civ-block civ-text-sm civ-text-muted">Complete service history first</span>
  <civ-badge data-list-item-end label="Cannot start yet" intent="neutral" emphasis="secondary" with-icon></civ-badge>
</civ-list-item>
```

## Updating status from JavaScript

When a chapter completes, find the matching `<civ-list-item>` by `data-chapter-id` and update its inner `<civ-badge>`:

```js
function setItemStatus(item, status, href) {
  const badge = item.querySelector('civ-badge');
  const map = {
    'not-started':  { label: 'Not started',      variant: 'info',    style: 'secondary' },
    'in-progress':  { label: 'In progress',      variant: 'info',    style: 'primary' },
    'complete':     { label: 'Complete',         variant: 'success', style: 'primary' },
    'cannot-start': { label: 'Cannot start yet', variant: 'neutral', style: 'secondary' },
    'error':        { label: 'Has errors',       variant: 'error',   style: 'secondary' },
    'review':       { label: 'Needs review',     variant: 'warning', style: 'primary' },
  };
  const def = map[status];
  if (badge && def) {
    badge.setAttribute('label', def.label);
    badge.setAttribute('variant', def.variant);
    badge.setAttribute('badge-style', def.style);
  }
  if (href) {
    item.setAttribute('href', href);
  } else {
    item.removeAttribute('href');
  }
}
```

## Chapter flow

Each row links to a chapter. Inside the chapter, the user works through steps using `civ-form-step`. When complete, they return to the hub with the row marked as `complete`.

```
Hub → Chapter (multi-step) → Hub (updated status)
```

## Accessibility

- Each clickable row is a real `<a>` filling the row. Works with keyboard, screen readers, and right-click "open in new tab".
- Static (locked) rows render without an anchor. Assistive tech doesn't announce them as links. The `<civ-list-item>` host carries `role="listitem"` so the parent `<ul role="list">` accessibility tree sees a proper list item either way.
- Section headings are real `<h3>` elements that screen readers can navigate to as landmarks.
- `<civ-badge>` carries `role="status"` so status changes are announced.
- Progress bar announces completion percentage.

## Components used

- [`civ-list`](/components/layout/list). Generic list container with optional `dividers`
- [`civ-list-item`](/components/layout/list). List row; set `href` to make the whole row clickable, omit for locked rows
- `civ-badge`: semantic status indicator with `role="status"` and `with-icon` for the variant's icon
- `civ-progress-percent`: overall completion
