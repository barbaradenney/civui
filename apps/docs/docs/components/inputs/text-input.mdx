# Text Input

`<civ-text-input>`

Accessible text input with label, hint, error, and width variants. Supports input masking, declarative validation, prefix/suffix text, and inline icons. Uses ElementInternals for native form participation.

:::tip Specialized presets are documented below
For SSN, phone, email, ZIP, EIN, currency, routing number, and VA file number, see the [Specialized Presets](#specialized-presets) section below. These dedicated components (`<civ-ssn>`, `<civ-phone>`, etc.) pre-configure masking, validation, labels, hints, and HTML attributes automatically.
:::

### Character Counter

When `maxlength` is set, the input renders a "characters remaining" counter below the field. Visual count updates immediately; screen-reader announcements debounce on a 1s interval to avoid spamming. The counter is suppressed when a mask is active (the mask drives the length constraint).

## Shared Props

Props like `label`, `hint`, `error`, `required`, and `disabled` go on the `<civ-form-field>` wrapper. The component itself receives `name`, `value`, and component-specific props.

## Component-Specific Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `type` | `'text' \| 'email' \| 'number' \| 'password' \| 'search' \| 'tel' \| 'url'` | `'text'` | Input type attribute |
| `width` | `'default' \| '2xs' \| 'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl' \| '2xl'` | `'default'` | Input width variant |
| `placeholder` | `string` | `''` | Placeholder text |
| `pattern` | `string` | `''` | HTML pattern attribute for native validation |
| `maxlength` | `number` | — | Maximum character length |
| `minlength` | `number` | — | Minimum character length |
| `autocomplete` | `string` | `''` | Autocomplete hint (e.g., `'email'`, `'given-name'`) |
| `inputmode` | `string` | `''` | Virtual keyboard hint (e.g., `'numeric'`, `'tel'`) |
| `mask` | `'ssn' \| 'phone-us' \| 'zip' \| 'zip4' \| 'ein' \| 'currency' \| ''` | `''` | Mask preset name |
| `mask-pattern` | `string` | `''` | Custom mask pattern (`#` = digit, `A` = letter, `*` = any) |
| `mask-mode` | `'blur' \| 'live'` | `'blur'` | Mask formatting mode |
| `validate` | `'email' \| 'phone' \| 'phoneIntl' \| 'ssn' \| 'ein' \| 'zip' \| 'zip4' \| 'usState' \| 'url' \| 'currency' \| 'alphanumeric' \| ''` | `''` | Declarative validation preset |
| `prefix` | `string` | `''` | Text rendered inside the input on the leading edge (e.g., `'$'`, `'@'`) |
| `suffix` | `string` | `''` | Text rendered inside the input on the trailing edge (e.g., `'lbs'`, `'USD'`) |
| `clearable` | `boolean` | `false` | Show a clear button when the field has a value |
| `leading-icon` | `string` | `''` | civ-icon name rendered as a decorative overlay on the leading edge. Ignored when `prefix` is set. |
| `leading-icon-label` | `string` | `''` | Accessible label for the leading icon. Empty = decorative (`aria-hidden`). |
| `trailing-icon` | `string` | `''` | civ-icon name rendered on the trailing edge. Ignored when `suffix` is set or the clear button is showing. |
| `trailing-icon-label` | `string` | `''` | Accessible label for the trailing icon. Empty = decorative. |

## Custom Mask Patterns

For formats not covered by the specialized components, use `mask-pattern` with slot syntax:

| Slot | Matches |
|------|---------|
| `#` | Digit (0-9) |
| `A` | Letter (a-z, A-Z) |
| `*` | Any printable character |

All other characters are inserted as literals.

### Mask Modes

- **`blur`** (default): User types freely. Mask applied on blur for display. Accessible and predictable.
- **`live`**: Formats as user types. Has accessibility tradeoffs (cursor jumping).

## Events

| Event | Detail | Description |
|-------|--------|-------------|
| `civ-input` | `{ value: string }` | Fires on every value change |
| `civ-change` | `{ value: string }` | Fires on committed value change |

## Specialized Presets

These components extend `<civ-text-input>` with pre-configured masking, validation, labels, hints, and HTML attributes. All share the same events (`civ-input`, `civ-change` with `{ value: string }` detail).

### SSN (`<civ-ssn>`)

Pre-configured input for Social Security numbers with automatic masking, validation, and PII-aware blur-mode masking. Supports full 9-digit or last-4-only modes.

Default label: "Social Security number"

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `mode` | `'full' \| 'last4'` | `'full'` | Whether to collect full 9-digit SSN or last 4 digits only |

<StoryEmbed id="forms-inputs-text-input--ssn" />

<StoryEmbed id="forms-inputs-text-input--ssn-last-4" />

```html
<civ-form-field label="Social Security number" required>
  <civ-ssn name="ssn" required></civ-ssn>
</civ-form-field>

<!-- Last 4 digits only -->
<civ-form-field label="Last 4 of Social Security number" required>
  <civ-ssn name="ssn-last4" mode="last4" required></civ-ssn>
</civ-form-field>
```

## Usage

```html
<!-- Basic text input -->
<civ-form-field label="Full name" required>
  <civ-text-input name="fullName" required></civ-text-input>
</civ-form-field>

<!-- Custom mask pattern -->
<civ-form-field label="Case number" hint="For example: ABC-1234">
  <civ-text-input name="caseNumber" mask-pattern="AAA-####"></civ-text-input>
</civ-form-field>

<!-- With prefix -->
<civ-form-field label="Social handle">
  <civ-text-input name="handle" prefix="@" width="md"></civ-text-input>
</civ-form-field>

<!-- With suffix -->
<civ-form-field label="Weight">
  <civ-text-input name="weight" suffix="lbs" inputmode="decimal" width="sm"></civ-text-input>
</civ-form-field>

<!-- With character counter -->
<civ-form-field label="Short bio">
  <civ-text-input name="bio" maxlength="60"></civ-text-input>
</civ-form-field>
```
