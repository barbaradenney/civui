# Progress

CivUI provides three progress indicator components for multi-step forms:

- **`civ-progress`** — segmented progress bar (one rectangle per step)
- **`civ-progress-header`** — compact "Step X of Y: Title" text header
- **`civ-progress-bar`** — percentage-based progress bar

All three are used by `civ-form-step` via the `progress` prop, or standalone.

## Segmented Progress (`civ-progress`)

Each step is a small rectangle — filled for completed, highlighted for current, and empty for upcoming. Compact at any screen size.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `steps` | `string` | `'[]'` | JSON array of step labels or step objects (`{label}`) |
| `current` | `number` | `0` | Current step index (0-based) |
| `clickable` | `boolean` | `false` | When true, completed segments become clickable |
| `show-counter` | `boolean` | `false` | Show "Step X of Y" text below the segments |
| `error-steps` | `string` | `'[]'` | JSON array of step indices with validation errors |

### Events

| Event | Detail | Description |
|-------|--------|-------------|
| `civ-step-click` | `{ step: number }` | Fired when a completed segment is clicked (requires `clickable`) |

### Usage

```html
<civ-progress
  steps='["Personal Info", "Address", "Review", "Submit"]'
  current="1"
  clickable
  show-counter
></civ-progress>
```

## Progress Header (`civ-progress-header`)

<StoryEmbed id="forms-form-progress--progress-header" />

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `current` | `number` | `0` | Current step index (0-based, clamped) |
| `total` | `number` | `0` | Total number of steps |
| `step-title` | `string` | `''` | Title of the current step |
| `header-size` | `'sm' \| 'md' \| 'lg' \| 'xl'` | `'lg'` | Heading size |
| `header-spacing` | `'default' \| 'compact'` | `'default'` | Spacing variant |
| `heading-level` | `number` | `2` | Semantic heading level (1–6) |

### Usage

```html
<civ-progress-header
  current="1"
  total="4"
  step-title="Contact Information"
></civ-progress-header>
```

## Progress Bar (`civ-progress-bar`)

<StoryEmbed id="forms-form-progress--progress-bar" />

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `number` | `0` | Current progress percentage (0–100, clamped) |
| `label` | `string` | `'Progress'` | Accessible label for the bar |
| `status` | `string` | `''` | Optional status text (e.g., "3 of 8 sections complete") |
| `show-percent` | `boolean` | `true` | Show percentage text |

### Usage

```html
<civ-progress-bar
  value="40"
  label="Form completion"
  status="3 of 8 sections complete"
></civ-progress-bar>
```

## Using with `civ-form-step`

The `progress` prop on `civ-form-step` selects which indicator to display:

```html
<!-- Compact header (default) -->
<civ-form-step progress="minimal">...</civ-form-step>

<!-- Segmented bar -->
<civ-form-step progress="steps">...</civ-form-step>

<!-- Percentage bar -->
<civ-form-step progress="bar">...</civ-form-step>
```

The back link is rendered consistently above all three indicators and is handled by `civ-form-step`.
