`<civ-partnership-history>`

Compound component for a single marriage entry — spouse name, marriage date and location, status, and conditional end date. Designed to be used inside a `civ-repeater` for multiple marriages.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `legend` | `string` | `'About this marriage'` | Fieldset legend |
| `heading-level` | `1`–`6` | — | Promote the legend to a heading via `role="heading"` + `aria-level=N`. See [Heading hierarchy](/components/form/form-group#heading-hierarchy). |
| `size` | `'sm' \| 'md' \| 'lg' \| 'xl'` | `'sm'` | Visual size. `sm` = body size; `md`/`lg`/`xl` step up for section/page heading use. |
| `name` | `string` | `''` | Form field name prefix |
| `value` | `string` | `''` | JSON-serialized MarriageValue |
| `required` | `boolean` | `false` | Shows required indicator on legend |
| `disabled` | `boolean` | `false` | Disables all fields |
| `spouse-error` | `string` | `''` | Error for spouse name fields |
| `marriage-date-error` | `string` | `''` | Error for marriage date |
| `city-error` | `string` | `''` | Error for marriage city |
| `state-error` | `string` | `''` | Error for marriage state |
| `status-error` | `string` | `''` | Error for marriage status |
| `end-date-error` | `string` | `''` | Error for end date |

## Conditional fields

The "Date marriage ended" field only appears when the status is **not** "Currently married." Selecting "Currently married" clears any previously entered end date.

## Usage

```html
<civ-repeater legend="Marriage history" name="marriages" item-label="marriage" mode="form-steps" min="0" max="10">
  <civ-partnership-history name="entry"></civ-partnership-history>
</civ-repeater>
```
