# Conditional Reveal

Show or hide form fields based on the user's answers. This keeps forms short by only revealing fields that are relevant to the user's situation.

## Basic example

A single trigger reveals different follow-up fields depending on which value the user selects:

<StoryEmbed id="forms-form-conditional--default" />

```html
<civ-select
  label="What is your employment status?"
  name="employment"
  options='[
    {"label":"Select one","value":""},
    {"label":"Employed","value":"employed"},
    {"label":"Self-employed","value":"self-employed"},
    {"label":"Unemployed","value":"unemployed"}
  ]'
></civ-select>

<civ-conditional when="employment" equals="employed">
  <civ-text-input label="Employer name" name="employer"></civ-text-input>
</civ-conditional>

<civ-conditional when="employment" equals="self-employed">
  <civ-text-input label="Business name" name="business"></civ-text-input>
</civ-conditional>
```

## Nested conditions

Conditions can nest. The inner trigger is only watched while its outer parent is visible:

<StoryEmbed id="forms-form-conditional--multiple-conditions" />

```html
<civ-yes-no legend="Are you a veteran?" name="veteran"></civ-yes-no>

<civ-conditional when="veteran" equals="yes">
  <civ-text-input label="Branch of service" name="branch"></civ-text-input>

  <civ-select label="Discharge type" name="discharge" options='[...]'></civ-select>

  <civ-conditional when="discharge" equals="other">
    <civ-text-input label="Please describe your discharge type" name="discharge-other"></civ-text-input>
  </civ-conditional>
</civ-conditional>
```

## Operators

`civ-conditional` supports multiple matching operators:

| Attribute | Behavior |
|-----------|----------|
| `equals` | Show when field value equals this string |
| `not-equals` | Show when field value does NOT equal this string |
| `includes` | Show when field value is in a comma-separated list |
| `has-value` | Show when field has any non-empty value |
| `matches` | Show when field value matches a regex pattern |

## Behavior

- The slotted content is kept in the DOM and toggled with `display: none` plus `aria-hidden="true"` when the condition isn't met. Screen readers skip hidden content and focus order excludes it.
- Event scoping: civ-conditional listens for `civ-input` and `civ-change` on the nearest `civ-form` / `<form>` / `civ-fieldset` ancestor, so hundreds of conditionals on one page don't fight over global listeners.
- No animation: the wrapper just flips `display`, so there's no transition to disable for `prefers-reduced-motion`.
- **`civ-form` skips hidden conditional content** when validating and when collecting form data. Required fields inside a hidden branch don't block submit or appear in the error summary, and their stale values aren't sent to the server.

## Components used

- [`civ-conditional`](/components/form/conditional). Conditional container
- [`civ-yes-no`](/components/inputs/yes-no). Common trigger for binary conditions
- Any input component can be a trigger via its `name` attribute
