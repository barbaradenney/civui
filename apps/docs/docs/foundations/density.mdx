# Density Scale Guide

<StoryEmbed id="foundations-density-examples--side-by-side" />

CivUI's density system lets you tune spacing, font sizes, and line heights across an entire page or section with a single attribute. Three scales cover the full range of government digital experiences — from spacious public-facing content to compact staff dashboards.

## The Three Scales

| Scale | Attribute | Spacing | Base Font | Line Height | Best For |
|-------|-----------|---------|-----------|-------------|----------|
| Spacious | `data-civ-scale="spacious"` | 1.5x | 18–22px (fluid) | 1.75 | Homepages, kiosks, content pages |
| Default | _(none needed)_ | 1x | 16–18px (fluid) | 1.6 | Forms, standard applications |
| Dense | `data-civ-scale="dense"` | 0.75x | 14px (static) | 1.4 | Dashboards, data tables, admin panels |

## When to Use Each Scale

### Spacious — Public-Facing Content

Use the spacious scale for pages where readability and accessibility are the top priority. These are pages where users arrive with little context and need to orient themselves quickly.

**Ideal for:**
- Agency homepages and landing pages
- Public kiosk terminals (libraries, DMVs, post offices)
- Long-form content pages (policies, FAQs, guides)
- Accessibility-focused experiences (large text, generous tap targets)

Spacious uses fluid typography — font sizes scale smoothly between 18px and 22px as the viewport grows from 320px to 1200px. This makes it well-suited to kiosks and tablets where touch targets need to be large.

```html
<body data-civ-scale="spacious">
  <header>
    <h1 class="civ-text-4xl civ-font-bold">Welcome to the Department of Labor</h1>
    <p class="civ-text-lg civ-text-base-dark">Find resources, file claims, and access services.</p>
  </header>

  <main class="civ-mt-8 civ-space-y-6">
    <section>
      <h2 class="civ-text-2xl civ-font-semibold">Popular Services</h2>
      <ul class="civ-space-y-3 civ-text-base">
        <li><a href="/claims" class="civ-text-primary">File an unemployment claim</a></li>
        <li><a href="/wages" class="civ-text-primary">Look up prevailing wages</a></li>
        <li><a href="/safety" class="civ-text-primary">Report a workplace safety concern</a></li>
      </ul>
    </section>
  </main>
</body>
```

### Default — Forms and Applications

The default scale is the baseline. You don't need to set any attribute — it's what CivUI components use out of the box. It strikes the right balance between information density and readability for form-heavy workflows.

**Ideal for:**
- Multi-step form applications (benefits, permits, registrations)
- Standard data entry pages
- Account settings and profile pages
- Any page where users are filling in fields

```html
<!-- No data-civ-scale attribute needed — default is automatic -->
<main>
  <civ-form action="/api/apply">
    <civ-form-step heading="Personal Information">
      <civ-name-field name="fullName" required></civ-name-field>
      <civ-text-input label="Email address" name="email" type="email" required></civ-text-input>
      <civ-memorable-date label="Date of birth" name="dob" hint="For example: January 15 1990" required></civ-memorable-date>
    </civ-form-step>
  </civ-form>
</main>
```

### Dense — Dashboards and Admin Interfaces

Use the dense scale for internal tools and data-heavy views where trained staff need to see a lot of information at once. Dense reduces spacing to 0.75x and uses a fixed 14px base font — no fluid scaling, since these layouts prioritize information density over large-screen comfort.

**Ideal for:**
- Case management dashboards
- Admin panels and back-office tools
- Data tables and list views
- Status monitoring screens

```html
<div data-civ-scale="dense">
  <h2 class="civ-text-xl civ-font-semibold civ-mb-2">Open Cases</h2>

  <table class="civ-w-full civ-text-sm">
    <thead>
      <tr class="civ-border-b civ-border-base-light">
        <th class="civ-py-1 civ-text-start">Case ID</th>
        <th class="civ-py-1 civ-text-start">Applicant</th>
        <th class="civ-py-1 civ-text-start">Status</th>
        <th class="civ-py-1 civ-text-start">Updated</th>
      </tr>
    </thead>
    <tbody>
      <tr class="civ-border-b civ-border-base-lighter">
        <td class="civ-py-1">DOL-2026-04821</td>
        <td class="civ-py-1">Martinez, Elena</td>
        <td class="civ-py-1"><span class="civ-text-warning-dark">Under Review</span></td>
        <td class="civ-py-1">Apr 22, 2026</td>
      </tr>
    </tbody>
  </table>
</div>
```

## Mixing Scales on a Page

The `data-civ-scale` attribute cascades to all children, so you can mix scales within a single page. This is useful when a page has both content and a compact data section.

```html
<body data-civ-scale="spacious">
  <!-- Hero area uses spacious scale from body -->
  <header>
    <h1 class="civ-text-4xl civ-font-bold">Case Dashboard</h1>
    <p class="civ-text-lg">Welcome back, reviewer.</p>
  </header>

  <!-- Table section overrides to dense -->
  <section data-civ-scale="dense">
    <table><!-- compact case list --></table>
  </section>

  <!-- Form section uses default for comfortable data entry -->
  <aside data-civ-scale="default">
    <civ-text-input label="Add a note" name="note"></civ-text-input>
  </aside>
</body>
```

## How It Works

Density is powered by CSS custom properties generated from `scales.tokens.json`. When you set `data-civ-scale` on a container, all spacing and typography variables update for that subtree:

- **Spacing variables** (`--civ-spacing-*`) scale by the spacing factor (1.5x, 1x, or 0.75x)
- **Font size variables** (`--civ-typography-fontSize-*`) use modular-scale arithmetic (`base * ratio^step`)
- **Line height variables** (`--civ-typography-lineHeight-*`) adjust for readability at each density

All CivUI Tailwind utilities (`civ-p-4`, `civ-text-lg`, `civ-gap-2`, etc.) reference these variables, so they automatically respond to the active scale. No class changes needed.

## Accessibility Considerations

- **Spacious** exceeds WCAG touch-target minimums (44x44px) at all breakpoints — prefer it for public-facing and kiosk UIs where users may have motor impairments.
- **Default** meets WCAG touch-target minimums and provides comfortable reading line lengths.
- **Dense** can fall below touch-target minimums on small screens. Reserve it for desktop-only staff tools where users are trained and using a mouse. If a dense view must work on mobile, test tap targets carefully.
- Spacious and default use **fluid typography** (`clamp()`), so text scales smoothly across viewports. Dense uses a **static** 14px base — it does not grow on larger screens.

## Quick Decision Table

| You're building... | Use |
|---------------------|-----|
| A public .gov homepage | `spacious` |
| A kiosk at a government office | `spacious` |
| A benefits application form | default (no attribute) |
| A multi-step permit application | default (no attribute) |
| A caseworker dashboard | `dense` |
| An admin data table | `dense` |
| A content page with an embedded form | `spacious` on body, default on form section |
