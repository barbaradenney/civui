Cross-Platform Components

Define once, generate for Web, iOS, Android, and React Native. Consistent behavior and design across every platform.

Schema-Driven Approach

CivUI uses a platform-neutral schema as the single source of truth for every component. Each schema captures the component's properties, events, accessibility requirements, and validation rules. The codegen pipeline then generates idiomatic implementations for each target platform, ensuring consistent APIs and behavior without manual synchronization.

This means a component defined in one schema file produces a Lit web component, a SwiftUI view, a Jetpack Compose composable, and a React Native component, all with matching props, events, and accessibility semantics. When the schema changes, every platform updates in lockstep.

One Component, Four Platforms

The same text input, written idiomatically for each platform. All four are generated from a single text-input.schema.json definition.

WEB (Lit)
<civ-text-input
  label="Email"
  type="email"
  required
></civ-text-input>
iOS (SwiftUI)
CivTextInput(
    label: "Email",
    type: .email,
    isRequired: true
)
ANDROID (Jetpack Compose)
CivTextInput(
    label = "Email",
    type = TextInputType.Email,
    required = true
)
REACT NATIVE
<CivTextInput
  label="Email"
  type="email"
  required
/>

Shared Design Tokens

Design tokens are defined once in W3C DTCG format and compiled to platform-native formats. Every platform gets the exact same values.

Token CSS SwiftUI Compose
Primary var(--civ-color-primary) Color.civPrimary CivTheme.colors.primary
Spacing 4 var(--civ-spacing-4) CivSpacing.s4 CivTheme.spacing.s4
Font Body var(--civ-font-body) CivFont.body CivTheme.typography.body
Error var(--civ-color-error) Color.civError CivTheme.colors.error
Focus Ring var(--civ-focus-ring) CivStyle.focusRing CivTheme.focusRing

Package Structure

@civui/schema

Platform-neutral component schemas. The single source of truth for props, events, accessibility, and validation across all platforms.

@civui/codegen

Code generator that reads schemas and produces idiomatic Lit, SwiftUI, Compose, and React Native components.

@civui/tokens

W3C DTCG tokens compiled to CSS custom properties, Tailwind preset, JS constants, SwiftUI color/spacing assets, and Compose theme values.

@civui/forms

Web components built with Lit 3 and Light DOM. Text input, select, checkbox, radio, combobox, date picker, file upload, and more.

CivUI for iOS

SwiftUI views for iOS 16+ and macOS 13+. Native accessibility, Dynamic Type, and VoiceOver support built in.

CivUI for Android

Jetpack Compose components for Android 8+. Material 3 foundation with CivUI design tokens and accessibility semantics.

@civui/react-native

React Native companion components sharing the same prop APIs and design tokens as the web components.
