# Bulk Actions

<!-- contract-link:start -->

> **Contract reference:** [`civ-bulk-actions`](pathname:///civui/storybook/?path=/docs/contract-civ-bulk-actions--docs) — auto-generated structured contract for cross-platform implementation.

<!-- contract-link:end -->

`<civ-bulk-actions>`

Selection-aware action bar that appears above or below a `civ-data-grid` (or any other component with a selection model) when one or more rows are selected. Shows `"{count} {itemName(s)} selected"` on the leading edge, action buttons in the default slot, and a Clear button on the trailing edge.

Hidden via `display: none` when `count === 0` so the default-slot action buttons stay in the DOM across selection transitions — they don't get re-created every time the user selects/deselects.

## Composing with civ-data-grid

```ts
const grid = document.querySelector('civ-data-grid')!;
const bulk = document.querySelector('civ-bulk-actions')!;

grid.addEventListener('civ-selection-change', (e) => {
  const ids = (e as CustomEvent).detail.selectedRowIds;
  grid.selectedRowIds = ids;
  bulk.count = ids.length;
});

bulk.addEventListener('civ-clear-selection', () => {
  grid.selectedRowIds = [];
  bulk.count = 0;
});

// Wire each action button to its own handler — bulk-actions doesn't
// intercept clicks on its slotted children.
document.querySelector('[data-action="delete"]')!.addEventListener('click', () => {
  api.delete(grid.selectedRowIds).then(() => {
    grid.rows = grid.rows.filter((r) => !grid.selectedRowIds.includes(r.id));
    grid.selectedRowIds = [];
    bulk.count = 0;
  });
});
```

## Pluralization

The component appends `"s"` to `itemName` when `count !== 1`. For nouns that don't pluralize that simply, set `itemNamePlural`:

```html
<civ-bulk-actions count="3" item-name="entry" item-name-plural="entries">
  …
</civ-bulk-actions>
```

## Accessibility

- The bar is `role="region"` with `aria-live="polite"` so the selection
  count is announced when it changes — but only on transitions from
  hidden→visible (since `display: none` content is not announced).
- Action buttons inside the default slot keep their own ARIA semantics
  (use `civ-button` with descriptive labels — "Delete application", not
  just "Delete", per the row-action label guideline).
- The Clear button uses the global focus ring (no extra styling needed).
