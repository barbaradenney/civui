# Date

`<civ-memorable-date>` `<civ-date-picker>` `<civ-date-range-picker>`

Three date components for different use cases:

| Use case | Component | Wrapper | Example |
|----------|-----------|---------|---------|
| **Known dates** (birthday, document dates) | `<civ-memorable-date>` | `<civ-form-fieldset>` | Date of birth, discharge date |
| **Scheduling** (appointments, deadlines) | `<civ-date-picker>` | `<civ-form-field>` | Appointment date, filing deadline |
| **Date ranges** (start/end pairs) | `<civ-date-range-picker>` | `<civ-form-fieldset>` | Leave dates, service period |

:::caution Never use civ-date-input
`civ-date-input` is deprecated due to accessibility issues with Dragon NaturallySpeaking, VoiceOver, and TalkBack.
:::

## Memorable Date

`<civ-memorable-date>`

Three-field date input (month select + day + year) for dates the user knows from memory. Wrapped in `<civ-form-fieldset>` because it contains multiple sub-inputs.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `locale` | `string` | `'en-US'` | Locale for month names |
| `month-label` | `string` | `'Month'` | Label for the month select |
| `day-label` | `string` | `'Day'` | Label for the day input |
| `year-label` | `string` | `'Year'` | Label for the year input |
| `month-empty-label` | `string` | `'- Month -'` | Empty option text for month select |

### Value Format

ISO `YYYY-MM-DD`. Invalid dates (e.g., February 30) produce an empty value.

### Events

| Event | Detail | Description |
|-------|--------|-------------|
| `civ-input` | `{ value, month, day, year }` | Fires on any field change |
| `civ-change` | `{ value, month, day, year }` | Fires on committed change |

### Usage

```html
<civ-form-fieldset legend="Date of birth" hint="For example: January 15 1990" required>
  <civ-memorable-date name="dob" required></civ-memorable-date>
</civ-form-fieldset>
```

## Date Picker

`<civ-date-picker>`

Calendar dialog implementing the W3C APG Dialog + Grid pattern. Combines text input for manual entry with a calendar popup for browsing.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `min` | `string` | `''` | Minimum date (`YYYY-MM-DD`) |
| `max` | `string` | `''` | Maximum date (`YYYY-MM-DD`) |
| `placeholder` | `string` | `'mm/dd/yyyy'` | Input placeholder |
| `locale` | `string` | `'en-US'` | Locale for date formatting |
| `week-starts-on` | `number` | `0` | Day week starts on (0 = Sunday) |
| `disabled-dates` | `string` | `''` | JSON array of ISO dates to disable |
| `hide-today-button` | `boolean` | `false` | Suppress the "Today" button |

### Calendar Keyboard Navigation

| Key | Action |
|-----|--------|
| Arrow keys | Move by day/week |
| Page Down/Up | Next/previous month |
| Shift + Page Down/Up | Next/previous year |
| Home/End | Start/end of week |
| Enter/Space | Select date |
| T | Jump to today (without selecting) |
| Escape | Close calendar |

### Value Format

ISO `YYYY-MM-DD` regardless of display locale.

### Events

| Event | Detail | Description |
|-------|--------|-------------|
| `civ-input` | `{ value: string }` | Fires on text input change |
| `civ-change` | `{ value: string }` | Fires when date is selected or typed |

### Usage

```html
<civ-form-field label="Appointment date" hint="Choose an available date" required>
  <civ-date-picker name="appointment" min="2026-01-01" max="2026-12-31" required></civ-date-picker>
</civ-form-field>
```

## Date Range Picker

`<civ-date-range-picker>`

Two linked date pickers that form a start/end range. Cross-validates so end can never precede start. Wrapped in `<civ-form-fieldset>` because it contains two sub-pickers.

### Examples

<StoryEmbed id="forms-inputs-date-range-picker--default" minHeight={450} />
<StoryEmbed id="forms-inputs-date-range-picker--with-outer-bounds" minHeight={450} />
<StoryEmbed id="forms-inputs-date-range-picker--min-range-days" minHeight={450} />
<StoryEmbed id="forms-inputs-date-range-picker--government-leave-request" minHeight={450} />

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `min` | `string` | `''` | Outer lower bound |
| `max` | `string` | `''` | Outer upper bound |
| `min-range-days` | `number` | `0` | Minimum duration (0 = no check) |
| `max-range-days` | `number` | `0` | Maximum duration (0 = no check) |
| `start-label` | `string` | `'Start date'` | Label on start picker |
| `end-label` | `string` | `'End date'` | Label on end picker |
| `locale` | `string` | `'en-US'` | Forwarded to both pickers |

### Form Value

Writes two FormData fields: `${name}.start` and `${name}.end` in ISO format.

### Cross-Field Validation

Automatically checks: end ≥ start, duration ≥ min-range-days, duration ≤ max-range-days.

### Events

| Event | Detail | Description |
|-------|--------|-------------|
| `civ-input` | `{ value: { start, end } }` | Fires on any change |
| `civ-change` | `{ value: { start, end } }` | Fires on committed change |

### Usage

```html
<civ-form-fieldset legend="Leave dates" hint="First and last day of leave" required>
  <civ-date-range-picker
    name="leave"
    min="2026-01-01"
    max="2026-12-31"
    min-range-days="1"
    max-range-days="30"
    required
  ></civ-date-range-picker>
</civ-form-fieldset>
```
