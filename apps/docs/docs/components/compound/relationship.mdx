# Relationship

`<civ-relationship>`

Compound component for capturing a person and their relationship to the applicant. Includes conditional follow-up fields based on the relationship category â€” spousal types show marriage date, child types show date of birth, etc.

Use the `preset` prop for agency-specific option lists, or pass custom `options` to override.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `legend` | `string` | `'About this person'` | Fieldset legend text |

| `preset` | `'general' \| 'va-dependent' \| 'va-survivor' \| 'ssa-survivor' \| 'immigration' \| 'tax'` | `'general'` | Agency-specific relationship options |
| `options` | `RelationshipTypeConfig[]` | â€” | Custom options (overrides preset) |

## Presets

Each preset provides a curated list of relationship types for a specific agency context:

| Preset | Options | Use case |
|--------|---------|----------|
| `general` | Spouse, Child, Parent, Sibling, Other | Generic forms |
| `va-dependent` | Spouse, Biological child, Adopted child, Stepchild, Parent | VA dependency claims (21-686c) |
| `va-survivor` | Spouse, Child, Parent, Executor, Funeral director, Other | VA burial benefits (21P-530) |
| `ssa-survivor` | Spouse, Former spouse, Child, Grandchild, Parent | SSA survivor benefits |
| `immigration` | Spouse, Biological child, Adopted child, Stepchild, Parent, Sibling | Family-based petitions (I-130) |
| `tax` | Spouse, Biological/Adopted/Step/Foster child, Sibling, Parent | Tax dependents (1040) |

## Conditional fields

Fields appear automatically based on the selected relationship category:

| Category | Relationship types | Fields shown |
|----------|-------------------|--------------|
| Spousal | Spouse, Former spouse, Domestic partner | Marriage date, divorce date (if `show-divorce-date`) |
| Child | Child, Biological/Adopted/Step/Foster child, Grandchild | Date of birth, adoption date (if `show-adoption-date`) |
| Other | Other | Free-text description |
| None | Parent, Sibling, Executor, Funeral director | No conditional fields |

When the user switches relationship types and the category changes, stale conditional field values are automatically cleared.

## Usage

### Basic

```html
<civ-relationship
  legend="About the dependent"
  name="dependent"
  preset="general"
  required
></civ-relationship>
```

### VA survivor with deceased fields

```html
<civ-relationship
  legend="About the person who died"
  name="deceased"
  preset="va-survivor"
  show-deceased
  required
></civ-relationship>
```

### Immigration with adoption date

```html
<civ-relationship
  legend="About the person you are petitioning for"
  name="beneficiary"
  preset="immigration"
  show-adoption-date
  required
></civ-relationship>
```

### Without name fields

Use `show-name="false"` when the name is captured in a separate component:

```html
<civ-name legend="Spouse's name" name="spouseName" required></civ-name>
<civ-relationship
  legend="Relationship details"
  name="spouse"
  preset="general"
  show-name="false"
></civ-relationship>
```
